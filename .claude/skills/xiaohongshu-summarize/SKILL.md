# 小红书报告生成组件

读取小红书抓取数据（raw.json），使用 AI 分析内容并生成结构化报告（_index.md）。

---

## 🎯 执行流程

当用户调用此 skill 时，请按以下步骤执行：

### 步骤 1：读取数据

从指定目录下读取 `raw.json` 文件，提取以下数据：

| 字段 | 说明 |
|------|------|
| `search_time` | 执行时间 |
| `posts` | 帖子数组（每个帖子包含 title, content, author, date, likes, collects, comments_count, comments）|
| `search_strategy` | 搜索策略（固定模式，可选）|
| `divergence_path` | 发散路径（发散模式，可选）|

**计算总评论数**：累加所有帖子的 `comments_count` 字段。

### 步骤 2：判断模式

根据 raw.json 内容判断报告模式：

| 条件 | 模式 | 板块数 |
|------|------|--------|
| 包含 `divergence_path` 字段且非空 | 发散模式 | 9 个板块 |
| 不包含 `divergence_path` 或为空 | 固定关键词模式 | 8 个板块 |

### 步骤 3：生成报告

使用以下模板生成 `_index.md` 报告。

## 📋 核心要求

### 引用格式规范

引用格式取决于 `hyperlinks` 配置：

| 内容类型 | 超链接关闭（默认） | 超链接启用 | 说明 |
|---------|------------------|-----------|------|
| AI 总结内容 | 正常字体（粗体强调重点） | 同左 | 不受超链接配置影响 |
| 帖子标题引用 | `*《标题》*` | `[*《标题》*](id:{post_id})` | 启用时使用 ID 占位符 |
| 用户评论引用 | `_"评论内容"_` | 同左 | 不受超链接配置影响 |
| 来源标注 | `来源：*《标题》*` | `来源：[*《标题》*](id:{post_id})` | 启用时使用 ID 占位符 |
| 追溯来源 | `追溯来源：*《标题》*` | `追溯来源：[*《标题》*](id:{post_id})` | 启用时使用 ID 占位符 |

**注意**：超链接启用时，使用 `id:{post_id}` 作为占位符，实际的 URL 替换由 formatter 阶段完成。

### 向后兼容性

当 raw.json 中的帖子不包含 `post_id` 字段（旧数据）时：
- **必须** 降级为纯文本格式（超链接关闭格式）
- **禁止** 生成 ID 占位符引用
- 继续正常生成报告，仅引用格式不同

### Emoji 使用

**[核心要求]** 以下位置必须使用 emoji 增强可读性：

- 报告标题
- 搜索概览
- 搜索发散路径（仅发散模式）
- 品牌/主题声量分析
- 评论区高频共识
- 场景化决策建议
- 价格/规格参考
- 避雷指南
- 关键洞察
- 数据来源说明

具体 emoji 由 Agent 根据内容上下文自行判断，无需遵循固定映射规则。

### 禁止使用脚本

**[核心要求]** 生成 `_index.md` 时，禁止调用任何外部脚本进行**文本分析或内容提取**（包括但不限于：Python `re` 正则匹配、`collections.Counter` 词频统计、实体识别、情感分析脚本等）。

所有关于内容的理解、分析、总结（如品牌识别、优缺点提炼、情感分析、观点归纳）必须基于 LLM 的自然语言理解能力完成。允许基本数值统计（如帖子数量、评论总数、数值计算）。

### 可追溯性

所有结论**必须可追溯到具体帖子或评论**：
- 必须包含帖子标题、作者、日期、关键引用
- 数据不足时明确标注"样本有限"
- 避免主观判断，以评论区观点为依据

## 输出要求

2. 报告应当以 Markdown 格式完整呈现
3. 禁止仅发送文件路径而不展示内容
