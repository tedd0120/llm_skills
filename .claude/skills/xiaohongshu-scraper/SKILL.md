# 小红书内容抓取 Skill

通过自动化浏览器（Playwright）抓取小红书上的帖子正文和评论区内容。作为 Agent，你需要调用此 Skill 提供的脚本获取原始数据，然后将数据进行 AI 总结并整理输出。

---

## 🎯 两阶段执行流程

当用户调用此命令时，请按以下两个阶段执行：

### 阶段一：澄清阶段（必须执行）

此阶段的目标是与用户确认搜索关键词，确保爬虫抓取的内容符合用户意图。**确认前禁止进入执行阶段**。

#### 1. 解析用户输入

- **将用户输入视为整体文本**，忽略内部所有分隔符（逗号、空格、分号等）
- 理解用户建议的主旨，识别核心要素：主题、地域、场景、意图
- **示例**：输入 "广州装修公司, 对比, 避坑" 应理解为 "广州装修公司相关" 这一整体建议

#### 2. 衍生搜索关键词

**默认配置**：
- **默认帖子上限**：30 篇（可通过 `--max-posts` 参数修改）
- **关键词数量**：默认衍生 3 个搜索关键词，**用户可调整为 1-10 个**
- **帖数分配规则**：均匀分配，无法均分的余数归入第一关键词

```
分配算法（支持可变关键词数量 K）：
总上限 N ÷ K = 每个关键词基础分配
余数 N % K 加到第一关键词

示例（K=3，默认）：
- 30 篇 → [10篇, 10篇, 10篇]
- 50 篇 → [18篇, 16篇, 16篇]

示例（K=5）：
- 50 篇 → [10篇, 10篇, 10篇, 10篇, 10篇]
- 30 篇 → [6篇, 6篇, 6篇, 6篇, 6篇]
```

- 所有衍生词必须符合用户建议的主旨，保留关键要素（如地域、场景）
- **必须为每个关键词提供理由**，说明该词的搜索目的
- **建议从不同角度衍生**（但不强制）：
  - 直译/直接搜索（最接近原意）→ 理由："获取[主题]的整体推荐趋势"
  - 对比/评价（获取他人经验）→ 理由："获取[主题]的对比评价和用户体验"
  - 避雷/负面（获取风险信息）→ 理由："了解[主题]的常见问题和避坑指南"

#### 3. 展示并征求反馈

使用以下格式向用户展示衍生关键词：

```
我基于你的建议衍生了以下 3 个搜索词：

| 序号 | 关键词 | 预计篇数 | 搜索理由 |
|:----:|:------|:--------:|:---------|
| 1 | [关键词一] | 10篇 | [理由一] |
| 2 | [关键词二] | 10篇 | [理由二] |
| 3 | [关键词三] | 10篇 | [理由三] |
| **总计** | — | **30篇** | 默认上限 30 篇 |

你可以：
• 修改关键词数量 (如: "增加到5个" 或 "减少到2个"，范围1-10个)
• 修改单个关键词 (如: "把第1个改成国产电视推荐")
• 重新生成关键词 (如: "换一批关键词")
• 提出关键词建议 (如: "加上'小米电视'这个品牌")
• 修改搜索上限 (如: "增加到50篇" 或 "减少到20篇")
• 确认开始爬取

回复你的选择：
```

**说明**：
- **必须**使用 Markdown 表格格式展示关键词
- 表格列名固定：序号、关键词、预计篇数、搜索理由
- 序号列居中对齐，关键词列左对齐，预计篇数列居中对齐，搜索理由列左对齐
- 最后一行必须为合并列的总计行
- 表格行数动态调整：行数 = 关键词数量 K + 1（K 范围为 1-10）
- 每个关键词必须附带搜索理由，说明该关键词的搜索目的

#### 4. 循环调整

- 根据用户反馈修改关键词
- 重新展示更新后的关键词
- **主动询问**："确认开始爬取吗？"
- 直到用户明确确认（回复"确认"、"开始"、"好的"、"就这样"等）

**澄清阶段完成标志**：用户明确确认关键词，进入阶段二。

**[核心要求] 确认后必须立即创建搜索目录**：
- 用户确认关键词后，**必须立即**在调用 fetch_xhs.py 之前创建搜索目录
- 目录格式：`YYYYMMDD_HHmmSS_搜索主旨`（使用占位符）
- **时间戳必须使用当前系统时间，禁止使用示例、模板或虚构时间**
  - 例如：`20260303_182300_北京粤菜推荐`（18:23 执行时使用 182300，而非 143000）
- 后续所有产出文件（raw.json、_index.md）必须在此目录下
- 禁止在搜索目录外创建任何产出文件

## 🎯 优先级定义与冲突处理

### 优先级标签体系

| 优先级 | 标签 | 含义 | 违反后果 |
|-------|-----|------|---------|
| **P0** | [核心要求] | 绝对必须，违反则任务失败 | 无法生成报告 |
| **P1** | [标准操作] | 强烈建议，违反降低质量 | 可靠性降低 |
| **P2** | [灵活选项] | 可选方案，提供替代路径 | 无 |
| **P3** | [参考信息] | 提示信息，无约束力 | 无 |

### 优先级冲突处理规则

- P0 > P1 > P2 > P3（优先级降序）
- P0 受阻时：必须停止任务并报告失败
- P1 受阻时：可降级使用 P2 方案，并标注降级原因
- P2/P3 受阻时：可忽略该指令并继续任务

### 核心要求概览

以下为核心要求清单，违反任何一项将导致任务失败：

1. **[核心要求]** 必须完成澄清阶段后才能进入执行阶段
2. **[核心要求]** 必须衍生恰好 3 个关键词，且符合用户主旨
3. **[核心要求]** 必须在 `data/xiaohongshu/` 下创建专属结果文件夹
4. **[核心要求]** 必须直接从 JSON 数据生成增强版汇总报告 `_index.md`
5. **[核心要求]** 报告必须包含 8 个分析板块，禁止跳过或简化
6. **[核心要求]** 报告结论必须可追溯到具体帖子或评论
7. **[核心要求]** 生成报告后，必须将报告原文直接发送到用户对话框
8. **[核心要求]** 生成 `_index.md` 时，禁止调用任何外部脚本进行**文本分析或内容提取**（包括但不限于：Python `re` 正则匹配、`collections.Counter` 词频统计、实体识别、情感分析脚本等）。所有关于内容的理解、分析、总结（如品牌识别、优缺点提炼、情感分析、观点归纳）必须基于 LLM 的自然语言理解能力完成。允许基本数值统计（如帖子数量、评论总数、数值计算）。
9. **[核心要求]** 创建搜索目录时，时间戳必须使用当前系统时间，禁止使用示例、模板或虚构时间（如模板中的 `143000`）。

## 🎯 前置依赖配置

### 环境变量配置

请确保用户在 `.env` 中添加了此配置，用于保存登录的持久化 state（Cookie 等）：

```env
XHS_AUTH_STATE=.claude/skills/xiaohongshu-scraper/scripts/xhs_auth.json
```

### 依赖安装

```bash
pip install -r .claude/skills/xiaohongshu-scraper/scripts/requirements.txt
```

### Windows 环境

脚本会优先使用系统 Edge (`channel="msedge"`) 以减少风控特征。

### Linux 无屏幕环境配置（Xvfb 虚拟显示器）

在 Linux 服务器（无物理显示器）上运行时，[标准操作] 应当使用 Xvfb 创建虚拟屏幕以启动**有头模式浏览器**，这比 headless 模式更不容易被小红书检测：

```bash
# 1. 安装 Xvfb 和 Chromium 依赖
sudo apt-get update
sudo apt-get install -y xvfb libgbm1 libnss3 libatk-bridge2.0-0 libxkbcommon0 \
    libgtk-3-0 libdrm2 libxcomposite1 libxdamage1 libxrandr2 libasound2

# 2. 安装 Playwright 浏览器
playwright install chromium

# 3. 启动虚拟显示器 (1920x1080 分辨率, 24位色深)
Xvfb :99 -screen 0 1920x1080x24 &

# 4. 设置 DISPLAY 环境变量
export DISPLAY=:99

# 5. 正常运行脚本 (不加 --headless，将以有头模式在虚拟屏幕中运行)
python .claude/skills/xiaohongshu-scraper/scripts/fetch_xhs.py --keywords "搜索词"
```

> **[参考信息]**：可将 `export DISPLAY=:99` 写入 `.bashrc` 或 `.env` 中持久化。如果使用 systemd 管理 agent 服务，在 service 文件的 `[Service]` 段添加 `Environment=DISPLAY=:99`。

> **[灵活选项]**：如无法安装 Xvfb，可降级使用 `--headless` 参数（风险提示：风控检测概率更高）。

## 🤖 Agent 执行详细步骤

### 步骤 1：创建搜索目录（确认后立即执行）

**[核心要求]** 用户确认关键词后，**必须立即**执行以下命令创建搜索目录。

**步骤 1.1：执行目录创建命令**
```bash
# 替换 [搜索主旨] 为基于关键词主题提取的名称
mkdir -p "data/xiaohongshu/$(date +%Y%m%d_%H%M%S)_[搜索主旨]/"
```

**执行示例：**
```bash
# 假设搜索主旨是"北京咖啡厅"，执行：
mkdir -p "data/xiaohongshu/$(date +%Y%m%d_%H%M%S)_北京咖啡厅/"

# 实际创建的目录（假设当前系统时间为 2026-03-03 21:15:30）：
# data/xiaohongshu/20260303_211530_北京咖啡厅/
```

**步骤 1.2：保存目录路径供后续使用**
```bash
# 将目录路径保存为变量，后续 fetch_xhs.py 和报告生成都会用到
OUTPUT_DIR="data/xiaohongshu/$(date +%Y%m%d_%H%M%S)_[搜索主旨]/"
```

**说明：**
- `$(date +%Y%m%d_%H%M%S)` 会自动获取当前系统时间并格式化
- 禁止手动替换时间戳或使用硬编码值（如 `143000`、`182300`）
- 搜索主旨基于用户确认的关键词主题提取
- 清理主旨中的文件系统非法字符（如 `/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`）
- 如目录已存在，警告用户并询问是否覆盖或添加序号后缀

**[核心要求] 同时保存 search_strategy**：
- 用户确认关键词后，Agent 需要在内存中保存 `search_strategy` 数据结构
- 数据格式：`[{"keyword": "关键词一", "posts_count": 10, "intent": "搜索理由"}, ...]`
- 此数据将在执行阶段传递给 `fetch_xhs.py` 脚本

### 步骤 1.5：确保已登录（新增，必须在抓取前执行）

先调用独立登录 skill 检查 Cookie 状态：

```bash
python .claude/skills/xiaohongshu-login/scripts/login_xhs.py --check-only
```

处理规则：

1. 输出 `LOGIN_OK`：继续步骤 2，执行抓取。
2. 输出非 `LOGIN_OK`：执行完整登录流程：

```bash
python .claude/skills/xiaohongshu-login/scripts/login_xhs.py
```

3. 如果输出 `NEED_LOGIN:<abs_path>`：读取 `<abs_path>` 二维码并展示给用户扫码。
4. 如果输出 `LOGIN_SUCCESS`：继续步骤 2。
5. 如果输出 `LOGIN_TIMEOUT` 或 `LOGIN_FAILED`：告知用户登录失败并停止本轮抓取，等待用户重试。

### 步骤 2：执行抓取脚本（后台运行）

调用 `fetch_xhs.py` 获取原始数据。**必须使用后台运行模式**。

**2.1 启动后台抓取任务**

```bash
# 使用后台模式启动，保存 task_id
# 命令会立即返回 task_id，不会阻塞
TASK_ID=$(python .claude/skills/xiaohongshu-scraper/scripts/fetch_xhs.py \
    --keywords "关键词1,关键词2,关键词3" \
    --max-posts 30 \
    --search-strategy '[{"keyword":"关键词1","posts_count":10,"intent":"获取整体推荐趋势"},{"keyword":"关键词2","posts_count":10,"intent":"获取探店体验"},{"keyword":"关键词3","posts_count":10,"intent":"了解避坑指南"}]' \
    --output "data/xiaohongshu/YYYYMMDD_HHmmSS_搜索主旨/raw.json")
```

**2.2 等待完成并处理结果**

使用 `TaskOutput` 等待后台任务完成并获取完整输出。

```
final_output = TaskOutput(TASK_ID, block=true)  # 等待完成

if exit_code != 0:
    if "检测到未登录" in final_output:
        提示用户执行 xiaohongshu-login skill
    else:
        展示错误信息并停止任务
else:
    # 任务成功完成，向用户展示简短完成消息
    告知用户抓取完成，即将生成报告
```

**进度输出格式**（由脚本输出，供参考）：
```
[*] 搜索关键词: xxx (配额 N, 总剩余 M)
  找到 X 篇（去重后），抓前 Y 篇
  [i/total] URL...
[✓] 共 N 篇 → output_file
```

**参数说明：**
- `--max-posts N`：**脚本必传参数**，Agent 调用时用户未修改篇数则传 30（硬上限 100）
- `--output PATH`：**必须**指定输出到搜索目录下的 raw.json
- `--search-strategy JSON`：搜索策略元数据（包含 keyword、posts_count、intent 的数组）
- `--headless`：强制无头模式（可选）

**[核心要求]** 必须使用 `run_in_background=true` 启动任务，禁止阻塞调用。

### 步骤 3：生成增强版汇总报告

读取刚才的 JSON 结果，直接生成完整的 `_index.md` 增强版汇总报告。

**数据读取**：
从 raw.json 中读取以下数据用于报告生成：
- `search_time`：执行时间
- `posts` 数组：所有帖子的完整数据
- `search_strategy` 数组：搜索策略元数据（如存在）
- 总评论数：累加所有帖子的 `comments_count` 字段

#### 搜索策略渲染逻辑

**[标准操作] 处理 search_strategy**：

1. **检测 search_strategy 是否存在**
   - 如果 raw.json 包含 `search_strategy` 且非空 → 渲染搜索策略子表格
   - 如果 raw.json 不包含 `search_strategy` 或为空 → 仅渲染基本信息表格

2. **Emoji 使用**

   **[核心要求]** 报告标题和以下 8 个主要板块必须使用 emoji 增强可读性：
   - 报告标题
   - 搜索概览
   - 品牌/主题声量分析
   - 评论区高频共识
   - 场景化决策建议
   - 价格/规格参考
   - 避雷指南
   - 关键洞察

   具体使用的 emoji 由 Agent 根据内容上下文自行判断，无需遵循固定映射规则。

3. **渲染搜索策略表格**

   ```markdown
   **搜索策略**

   | 关键词 | 篇数 | 搜索意图 |
   |:-------|-----:|:---------|
   | [关键词一] | 10篇 | [搜索理由一] |
   | [关键词二] | 10篇 | [搜索理由二] |
   | [关键词三] | 10篇 | [搜索理由三] |
   ```

#### 报告必需板块（8个）

| 板块 | 说明 |
|-----|------|
| 1. 搜索任务摘要 | 时间、关键词、数据量 |
| 2. 品牌/主题声量分析 | 统计主要品牌/话题的讨论热度 |
| 3. 多维对比矩阵 | 多维度对比表（至少3个维度） |
| 4. 评论区高频共识 | Top 5 优点 & Top 5 痛点 |
| 5. 场景化决策建议 | 决策树/场景推荐 |
| 6. 价格/规格参考 | 从评论中提取（如适用） |
| 7. 避雷指南 | 高频问题汇总 |
| 8. 关键洞察 | 跨帖洞察总结（至少3条） |

#### 数据来源规则

- **品牌/主题声量**：统计所有帖子中的品牌/主题提及
- **对比矩阵**：基于主贴内容和评论区对比讨论
- **高频共识**：分析所有评论（优先高赞评论）
- **决策建议**：提炼"什么情况选哪个"模式
- **价格信息**：从评论中提取实际成交价

**引用格式要求**：
- 帖子标题使用 `*《标题》*` 格式
- 用户评论使用 `_"评论内容"_` 格式
- 追溯来源使用 `> 来源：*《帖子标题》*` 格式
- AI 总结分析使用正常字体（非斜体）

#### 质量要求

- 所有结论**必须可追溯到具体帖子或评论**
- **必须包含**：帖子标题、作者、日期、关键引用
- 数据不足时明确标注"样本有限"
- 避免主观判断，以评论区观点为依据

**引用格式规范**（区分 AI 总结与原始引用）：

| 内容类型 | Markdown 格式 | 示例 |
|---------|--------------|------|
| AI 总结内容 | 正常字体（粗体强调重点） | 用户普遍反馈索尼过保即坏 |
| 帖子标题引用 | 斜体书名号 | *《想买索尼电视，我劝你冷静》* |
| 用户评论引用 | 斜体引号 | _"三年质保刚过就没声音"_ |
| 来源标注 | 斜体书名号 | 来源：*《电视别买75寸啊！！！》* |

> **重要**：所有引用内容（帖子标题、正文片段、评论）必须使用斜体样式 `*...*`，使 AI 分析总结与原始引用内容视觉可区分。

#### 输出要求

- **[核心要求]** 生成 `_index.md` 后，**必须将报告原文直接发送到用户对话框**
- 禁止仅发送文件路径而不展示内容
- 报告应当以 Markdown 格式完整呈现

#### 数据不足处理

- 当帖子数 < 15 篇时：标注"样本有限"，但不跳过任何板块
- 当某品牌/主题 < 3 篇时：标注"数据不足"，不强行对比
