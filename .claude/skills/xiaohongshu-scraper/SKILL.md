# 小红书内容抓取 Skill

一站式小红书内容抓取与分析入口。作为编排层，协调各个子 skills 完成从数据抓取到报告生成的完整流程。

**内部架构**：
- `scripts/login_xhs.py` - 单次调用内检查登录状态，并在需要时处理扫码登录
- `xiaohongshu-fetch` - 执行浏览器自动化抓取，输出 raw.json（详见其 SKILL.md）
- `xiaohongshu-summarize` - 分析数据并生成结构化报告（描述性 skill）
- `xiaohongshu-formatter` - 美化报告格式，增强 emoji（描述性 skill）

---

## 两阶段执行流程

当用户调用此命令时，请按以下两个阶段执行：

### 阶段一：澄清阶段（必须执行）

此阶段的目标是与用户确认搜索关键词，确保爬虫抓取的内容符合用户意图。**确认前禁止进入执行阶段**。

#### 1. 解析用户输入

- **将用户输入视为整体文本**，忽略内部所有分隔符（逗号、空格、分号等）
- 理解用户建议的主旨，识别核心要素：主题、地域、场景、意图
- **示例**：输入 "广州装修公司, 对比, 避坑" 应理解为 "广州装修公司相关" 这一整体建议

#### 2. 选择搜索模式

在解析用户输入后、衍生关键词前，**必须**先展示模式选择：

```markdown
请先选择搜索模式：

| 选项 | 模式 | 说明 |
|:----:|:-----|:-----|
| A | 固定关键词模式 | 你先确认关键词，系统按关键词一次性执行搜索 |
| B | 发散模式 | AI 从主题出发自动多轮搜索，每轮动态决定下一关键词 |

回复 `A` 或 `B`。
```

#### 3. 模式 A（固定关键词）: 衍生搜索关键词

**默认配置**：
- **默认帖子上限**：30 篇（可通过 `--max-posts` 参数修改）
- **关键词数量**：默认衍生 3 个搜索关键词，**用户可调整为 1-10 个**
- **帖数分配规则**：均匀分配，无法均分的余数归入第一关键词

```
分配算法（支持可变关键词数量 K）：
总上限 N ÷ K = 每个关键词基础分配
余数 N % K 加到第一关键词

示例（K=3，默认）：
- 30 篇 → [10篇, 10篇, 10篇]
- 50 篇 → [18篇, 16篇, 16篇]
```

- 所有衍生词必须符合用户建议的主旨，保留关键要素（如地域、场景）
- **必须为每个关键词提供理由**，说明该词的搜索目的
- **建议从不同角度衍生**（但不强制）：
  - 直译/直接搜索（最接近原意）→ 理由："获取[主题]的整体推荐趋势"
  - 对比/评价（获取他人经验）→ 理由："获取[主题]的对比评价和用户体验"
  - 避雷/负面（获取风险信息）→ 理由："了解[主题]的常见问题和避坑指南"

#### 4. 模式 A（固定关键词）: 展示并征求反馈

使用以下格式向用户展示衍生关键词：

```
我基于你的建议衍生了以下 3 个搜索词：

| 序号 | 关键词 | 预计篇数 | 搜索理由 |
|:----:|:------|:--------:|:---------|
| 1 | [关键词一] | 10篇 | [理由一] |
| 2 | [关键词二] | 10篇 | [理由二] |
| 3 | [关键词三] | 10篇 | [理由三] |
| **总计** | — | **30篇** | 默认上限 30 篇 |

你可以：
• 修改关键词数量 (如: "增加到5个" 或 "减少到2个"，范围1-10个)
• 修改单个关键词 (如: "把第1个改成国产电视推荐")
• 重新生成关键词 (如: "换一批关键词")
• 提出关键词建议 (如: "加上'小米电视'这个品牌")
• 修改搜索上限 (如: "增加到50篇" 或 "减少到20篇")
• 确认开始爬取

回复你的选择：
```

**说明**：
- **必须**使用 Markdown 表格格式展示关键词
- 表格列名固定：序号、关键词、预计篇数、搜索理由
- 最后一行必须为合并列的总计行
- 每个关键词必须附带搜索理由

#### 5. 模式 A（固定关键词）: 循环调整

- 根据用户反馈修改关键词
- 重新展示更新后的关键词
- **主动询问**："确认开始爬取吗？"
- 直到用户明确确认（回复"确认"、"开始"、"好的"、"就这样"等）

#### 6. 超链接格式选择（模式 A 和 B 共通）

用户确认关键词或发散参数后，必须询问超链接格式：

```markdown
报告中超链接格式：

| 选项 | 格式   | 说明                              |
|:----:|:------|:----------------------------------|
| A    | 纯文本  | 原帖引用为纯文本，无超链接（默认）     |
| B    | 超链接  | 原帖引用可直接点击跳转到小红书       |

回复 `A` 或 `B`（默认 A）。
```

#### 7. 模式 B（发散模式）: 参数设定与确认

用户选择 `B` 后，禁止执行固定关键词衍生流程，直接进入参数收集：

- 篇数上限：默认 30，硬上限 100
- 发散轮数：默认 3，范围 2-10

**参数收集模板：**

```markdown
已切换到发散模式（AI 自动多轮搜索）。

请确认参数：
- 篇数上限：30（可改，最大 100）
- 发散轮数：3（可改，范围 2-10）

回复示例：
- 使用默认
- 改为 50 篇，5 轮
```

确认参数后，必须展示配额预览：

```markdown
配额分配预览：
- 总上限 50 篇，发散 3 轮
- 每轮配额 = 总上限 ÷ 轮数，余数加到第一轮
- 本次分配: [18, 16, 16]

确认按该参数开始发散搜索吗？
```

**澄清阶段完成标志**：
- 模式 A：用户明确确认关键词 + 选择超链接格式，进入阶段二
- 模式 B：用户明确确认参数 + 选择超链接格式，进入阶段二

**[核心要求] 确认后必须立即创建搜索目录**：
- 用户确认关键词或发散参数后，**必须立即**在调用子 skill 前创建搜索目录
- 目录格式：`YYYYMMDD_HHmmSS_搜索主旨`
- **时间戳必须使用当前系统时间，禁止使用示例、模板或虚构时间**
- 后续所有产出文件必须在此目录下

---

### 阶段二：执行阶段（确认后进入）

此阶段编排各个子 skills 完成数据抓取和报告生成。

**编排层职责**：确保登录 → 抓取数据 → 生成报告 → 格式化报告 → 发送报告

#### 执行任务清单

进入阶段二后，必须立即将以下内容写入 `{OUTPUT_DIR}/tasks.md`：

```markdown
## 执行任务清单

- [ ] 确保登录
- [ ] 抓取数据
- [ ] 生成报告
- [ ] 格式化报告
- [ ] 发送报告
```

**核心要求**：
- 每完成一项任务，必须立即更新 tasks.md 将对应的 `[ ]` 改为 `[x]`
- 阶段二结束前，必须执行 `python scripts/verify_tasks.py <tasks_file_path>` 验证全部完成
- 验证失败必须报错中止，禁止发送未完成的报告

#### 登录轮询配置

`确保登录`步骤使用以下配置：

- `LOGIN_POLL_INTERVAL_SEC`：登录输出轮询间隔（秒）
  - 默认值：`2`
  - 建议范围：`1-10`
  - 若配置值非法（非数字、<=0 或超出建议范围），**必须**回退到默认值 `2`，并在日志中说明回退原因

#### 上下文传递

scraper 通过以下参数在子 skills 间传递上下文：

## 核心要求（编排层）

以下为核心要求清单，违反任何一项将导致任务失败：

1. **[核心要求]** 必须完成澄清阶段后才能进入执行阶段
2. **[核心要求]** 澄清阶段必须先执行模式选择（A 固定关键词 / B 发散模式）
3. **[核心要求]** 模式 A 必须衍生关键词并获得用户确认；模式 B 必须收集参数并获得用户确认
4. **[核心要求]** 模式 B 必须展示每轮配额预览（余数加到第一轮）
5. **[核心要求]** 用户确认后必须立即创建搜索目录，路径格式为 `data/xiaohongshu/YYYYMMDD_HHmmSS_主题/`
6. **[核心要求]** 目录时间戳必须使用当前系统时间，禁止使用示例或虚构时间
7. **[核心要求]** 进入阶段二后必须立即创建 `{OUTPUT_DIR}/tasks.md`，包含 5 项未勾选任务
8. **[核心要求]** 每完成一项任务必须立即更新 tasks.md，将 `[ ]` 改为 `[x]`
9. **[核心要求]** 确保登录步骤必须使用后台轮询模式：
   - 必须后台启动 `scripts/login_xhs.py`，禁止阻塞等待到脚本结束后再处理输出
   - 必须按 `LOGIN_POLL_INTERVAL_SEC` 间隔轮询新增输出（默认 2 秒；非法值回退到 2 秒）
   - 检测到 `NEED_LOGIN:<abs_path>` 时，必须立即提示用户扫码并展示二维码绝对路径
   - 检测到 `LOGIN_OK` 或 `LOGIN_SUCCESS` 后，才能打勾并进入下一步
   - 检测到 `LOGIN_TIMEOUT` 或 `LOGIN_FAILED` 时，必须报错中止，禁止继续抓取
10. **[核心要求]** 执行阶段必须按顺序完成所有步骤，禁止跳过任意步骤：
   - 确保登录 → 完成后台轮询收敛后打勾
   - 抓取数据 → 产出 `raw.json` 和 `id_url_map.json` 后打勾
   - 生成报告 → 阅读 `xiaohongshu-summarize/SKILL.md` 并产出 `_index.md` 后打勾
   - 格式化报告 → 阅读 `xiaohongshu-formatter/SKILL.md` 并更新 `_index.md` 后打勾
   - 发送报告 → 将最终报告发送到用户对话框后打勾
11. **[核心要求]** 阶段结束前必须执行 `python scripts/verify_tasks.py <tasks_file_path>` 验证
    - 返回 `TASKS_COMPLETE`：成功结束
    - 返回 `TASKS_INCOMPLETE`：报错中止，禁止发送报告

## 前置依赖配置

### 环境变量配置

请确保用户在 `.env` 中添加了此配置：

```env
XHS_AUTH_STATE=.claude/skills/xiaohongshu-scraper/scripts/xhs_auth.json
```

### 依赖安装

```bash
pip install -r .claude/skills/xiaohongshu-scraper/scripts/requirements.txt
```

### Windows 环境

脚本会优先使用系统 Edge (`channel="msedge"`) 以减少风控特征。

### Linux 无屏幕环境配置（Xvfb 虚拟显示器）⚠️ 必须

**[核心要求]** 在 Linux/WSL 服务器（无物理显示器）上运行时，必须使用 Xvfb 创建虚拟屏幕。脚本强制使用有头模式以确保登录成功率，无 `DISPLAY` 环境变量时将报错退出。

```bash
# 安装 Xvfb 和依赖
sudo apt-get install -y xvfb libgbm1 libnss3 libatk-bridge2.0-0

# 启动虚拟显示器
Xvfb :99 -screen 0 1920x1080x24 &
export DISPLAY=:99
```

> ⚠️ **无虚拟显示器将无法运行**：小红书登录在有头模式下成功率更高，headless 模式已被移除。

## 常见限制与故障排除

| 问题 | 原因 | 处理方式 |
|-----|------|---------|
| 检测到未登录 | Cookie 过期或未登录 | 执行 `scripts/login_xhs.py`，在当前会话中完成扫码登录 |
| 风控封禁 | 小红书检测到自动化 | 脚本已内置延时，**禁止取消延时** |
| 元素选择器失效 | 小红书页面改版 | 检查 `scripts/xhs_selectors.py` 并更新选择器 |
| 二维码过期或超时 | 登录等待时间过长 | 使用 `--timeout N` 调整超时时间 |
