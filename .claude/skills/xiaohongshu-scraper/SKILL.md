# 小红书内容抓取 Skill

一站式小红书内容抓取与分析入口。作为编排层，协调各个子 skills 完成从数据抓取到报告生成的完整流程。

**内部架构**：
- `scripts/login_xhs.py` - 检查登录状态并处理扫码登录
- `xiaohongshu-fetch` - 执行浏览器自动化抓取，输出 raw.json（详见其 SKILL.md）
- `xiaohongshu-summarize` - 分析数据并生成结构化报告（描述性 skill）
- `xiaohongshu-formatter` - 美化报告格式，增强 emoji（描述性 skill）

---

## 两阶段执行流程

当用户调用此命令时，请按以下两个阶段执行：

### 阶段一：澄清阶段（必须执行）

此阶段的目标是与用户确认搜索关键词，确保爬虫抓取的内容符合用户意图。**确认前禁止进入执行阶段**。

#### 1. 解析用户输入

- **将用户输入视为整体文本**，忽略内部所有分隔符（逗号、空格、分号等）
- 理解用户建议的主旨，识别核心要素：主题、地域、场景、意图
- **示例**：输入 "广州装修公司, 对比, 避坑" 应理解为 "广州装修公司相关" 这一整体建议

#### 2. 选择搜索模式

在解析用户输入后、衍生关键词前，**必须**先展示模式选择：

```markdown
请先选择搜索模式：

| 选项 | 模式 | 说明 |
|:----:|:-----|:-----|
| A | 固定关键词模式 | 你先确认关键词，系统按关键词一次性执行搜索 |
| B | 发散模式 | AI 从主题出发自动多轮搜索，每轮动态决定下一关键词 |

回复 `A` 或 `B`。
```

#### 3. 模式 A（固定关键词）: 衍生搜索关键词

**默认配置**：
- **默认帖子上限**：30 篇（可通过 `--max-posts` 参数修改）
- **关键词数量**：默认衍生 3 个搜索关键词，**用户可调整为 1-10 个**
- **帖数分配规则**：均匀分配，无法均分的余数归入第一关键词

```
分配算法（支持可变关键词数量 K）：
总上限 N ÷ K = 每个关键词基础分配
余数 N % K 加到第一关键词

示例（K=3，默认）：
- 30 篇 → [10篇, 10篇, 10篇]
- 50 篇 → [18篇, 16篇, 16篇]
```

- 所有衍生词必须符合用户建议的主旨，保留关键要素（如地域、场景）
- **必须为每个关键词提供理由**，说明该词的搜索目的
- **建议从不同角度衍生**（但不强制）：
  - 直译/直接搜索（最接近原意）→ 理由："获取[主题]的整体推荐趋势"
  - 对比/评价（获取他人经验）→ 理由："获取[主题]的对比评价和用户体验"
  - 避雷/负面（获取风险信息）→ 理由："了解[主题]的常见问题和避坑指南"

#### 4. 模式 A（固定关键词）: 展示并征求反馈

使用以下格式向用户展示衍生关键词：

```
我基于你的建议衍生了以下 3 个搜索词：

| 序号 | 关键词 | 预计篇数 | 搜索理由 |
|:----:|:------|:--------:|:---------|
| 1 | [关键词一] | 10篇 | [理由一] |
| 2 | [关键词二] | 10篇 | [理由二] |
| 3 | [关键词三] | 10篇 | [理由三] |
| **总计** | — | **30篇** | 默认上限 30 篇 |

你可以：
• 修改关键词数量 (如: "增加到5个" 或 "减少到2个"，范围1-10个)
• 修改单个关键词 (如: "把第1个改成国产电视推荐")
• 重新生成关键词 (如: "换一批关键词")
• 提出关键词建议 (如: "加上'小米电视'这个品牌")
• 修改搜索上限 (如: "增加到50篇" 或 "减少到20篇")
• 确认开始爬取

回复你的选择：
```

**说明**：
- **必须**使用 Markdown 表格格式展示关键词
- 表格列名固定：序号、关键词、预计篇数、搜索理由
- 最后一行必须为合并列的总计行
- 每个关键词必须附带搜索理由

#### 5. 模式 A（固定关键词）: 循环调整

- 根据用户反馈修改关键词
- 重新展示更新后的关键词
- **主动询问**："确认开始爬取吗？"
- 直到用户明确确认（回复"确认"、"开始"、"好的"、"就这样"等）

#### 6. 超链接格式选择（模式 A 和 B 共通）

用户确认关键词或发散参数后，必须询问超链接格式：

```markdown
报告中超链接格式：

| 选项 | 格式   | 说明                              |
|:----:|:------|:----------------------------------|
| A    | 纯文本  | 原帖引用为纯文本，无超链接（默认）     |
| B    | 超链接  | 原帖引用可直接点击跳转到小红书       |

回复 `A` 或 `B`（默认 A）。
```

#### 7. 模式 B（发散模式）: 参数设定与确认

用户选择 `B` 后，禁止执行固定关键词衍生流程，直接进入参数收集：

- 篇数上限：默认 30，硬上限 100
- 发散轮数：默认 3，范围 2-10

**参数收集模板：**

```markdown
已切换到发散模式（AI 自动多轮搜索）。

请确认参数：
- 篇数上限：30（可改，最大 100）
- 发散轮数：3（可改，范围 2-10）

回复示例：
- 使用默认
- 改为 50 篇，5 轮
```

确认参数后，必须展示配额预览：

```markdown
配额分配预览：
- 总上限 50 篇，发散 3 轮
- 每轮配额 = 总上限 ÷ 轮数，余数加到第一轮
- 本次分配: [18, 16, 16]

确认按该参数开始发散搜索吗？
```

**澄清阶段完成标志**：
- 模式 A：用户明确确认关键词 + 选择超链接格式，进入阶段二
- 模式 B：用户明确确认参数 + 选择超链接格式，进入阶段二

**[核心要求] 确认后必须立即创建搜索目录**：
- 用户确认关键词或发散参数后，**必须立即**在调用子 skill 前创建搜索目录
- 目录格式：`YYYYMMDD_HHmmSS_搜索主旨`
- **时间戳必须使用当前系统时间，禁止使用示例、模板或虚构时间**
- 后续所有产出文件必须在此目录下

### 阶段二：执行阶段（确认后进入）

此阶段编排各个子 skills 完成数据抓取和报告生成。

#### 完整流程

```
┌─────────────────────────────────────────────────────────────┐
│                    xiaohongshu-scraper (编排层)               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 澄清阶段（固定/发散模式）                                    │
│     → 用户确认关键词或参数                                      │
│     → 用户选择超链接格式                                        │
│     → 创建 OUTPUT_DIR                                        │
│       路径模板：data/xiaohongshu/YYYYMMDD_HHmmSS_主题/        │
│                                                             │
│  2. 检查登录                                                  │
│     → 调用 scripts/login_xhs.py --check-only                │
│     → 返回 LOGIN_OK：继续执行                                 │
│     → 返回其他状态：执行登录流程（显示二维码让用户扫码）          │
│                                                             │
│  3. 抓取数据 → xiaohongshu-fetch（详见其 SKILL.md）            │
│     → 固定模式：单次调用 → raw.json                            │
│     → 发散模式：多轮循环 → 合并 → raw.json                      │
│     → 生成 id_url_map.json（从 raw.json 提取映射）            │
│                                                             │
│  4. 生成报告（描述性 skill）                                   │
│     → 阅读 xiaohongshu-summarize/SKILL.md 了解报告模板         │
│     → 读取 raw.json，按模板生成 _index.md                     │
│                                                             │
│  5. 格式化（描述性 skill）                                    │
│     → 阅读 xiaohongshu-formatter/SKILL.md 了解格式要求         │
│     → 增强 emoji，替换超链接占位符                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 上下文传递

scraper 通过以下参数在子 skills 间传递上下文：

| 参数 | 类型 | 说明 | 传递给 |
|------|------|------|--------|
| `OUTPUT_DIR` | string | 搜索目录路径，格式为 `data/xiaohongshu/YYYYMMDD_HHmmSS_主题/` | fetch, summarize, formatter |
| `search_strategy` | array | 固定模式搜索策略 | fetch |
| `divergence_params` | object | 发散模式参数配置 | fetch |
| `hyperlinks` | boolean | 是否启用超链接功能 | fetch, summarize, formatter |

## 核心要求（编排层）

以下为核心要求清单，违反任何一项将导致任务失败：

1. **[核心要求]** 必须完成澄清阶段后才能进入执行阶段
2. **[核心要求]** 澄清阶段必须先执行模式选择（A 固定关键词 / B 发散模式）
3. **[核心要求]** 执行阶段开始前必须检查登录状态：
   - 调用 `python .claude/skills/xiaohongshu-scraper/scripts/login_xhs.py --check-only`
   - 等待并解析返回结果
   - 返回 `LOGIN_OK`：继续执行
   - 返回 `NEED_LOGIN:<path>`：执行登录流程（显示二维码路径让用户扫码），成功后继续
   - 返回 `LOGIN_TIMEOUT` 或 `LOGIN_FAILED`：报告错误并等待用户处理
4. **[核心要求]** 模式 A 必须衍生关键词并获得用户确认；模式 B 必须收集参数并获得用户确认
5. **[核心要求]** 模式 B 必须展示每轮配额预览（余数加到第一轮）
6. **[核心要求]** 用户确认后必须立即创建搜索目录，路径格式为 `data/xiaohongshu/YYYYMMDD_HHmmSS_主题/`
7. **[核心要求]** 目录时间戳必须使用当前系统时间，禁止使用示例或虚构时间
8. **[核心要求]** 生成报告后必须将报告原文直接发送到用户对话框
9. **[核心要求]** 执行阶段必须按顺序完成所有步骤，禁止跳过任意步骤：
   - 步骤 2：检查登录状态 → 必须返回 `LOGIN_OK`
   - 步骤 3：抓取数据 → 必须产出 `raw.json`
   - 步骤 4：生成报告 → 必须阅读 `xiaohongshu-summarize/SKILL.md` 并产出 `_index.md`
   - 步骤 5：格式化报告 → 必须阅读 `xiaohongshu-formatter/SKILL.md` 并更新 `_index.md`
   - 步骤 6：发送报告 → 必须将最终报告发送到用户对话框

   **每一步完成后必须验证产出文件存在，再进入下一步**

## 前置依赖配置

### 环境变量配置

请确保用户在 `.env` 中添加了此配置：

```env
XHS_AUTH_STATE=.claude/skills/xiaohongshu-scraper/scripts/xhs_auth.json
```

### 依赖安装

```bash
pip install -r .claude/skills/xiaohongshu-scraper/scripts/requirements.txt
```

### Windows 环境

脚本会优先使用系统 Edge (`channel="msedge"`) 以减少风控特征。

### Linux 无屏幕环境配置（Xvfb 虚拟显示器）

在 Linux 服务器（无物理显示器）上运行时，[标准操作] 应当使用 Xvfb 创建虚拟屏幕：

```bash
# 安装 Xvfb 和依赖
sudo apt-get install -y xvfb libgbm1 libnss3 libatk-bridge2.0-0

# 启动虚拟显示器
Xvfb :99 -screen 0 1920x1080x24 &
export DISPLAY=:99
```

> **[灵活选项]**：如无法安装 Xvfb，可降级使用 `--headless` 参数（风险提示：风控检测概率更高）。

## 常见限制与故障排除

| 问题 | 原因 | 处理方式 |
|-----|------|---------|
| 检测到未登录 | Cookie 过期或未登录 | 执行 `scripts/login_xhs.py` 完成扫码登录 |
| 风控封禁 | 小红书检测到自动化 | 脚本已内置延时，**禁止取消延时** |
| 元素选择器失效 | 小红书页面改版 | 检查 `scripts/selectors.py` 并更新选择器 |
| 二维码过期或超时 | 登录等待时间过长 | 使用 `--timeout N` 调整超时时间 |
