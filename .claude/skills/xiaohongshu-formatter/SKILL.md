# 小红书报告格式化 Skill

对 `xiaohongshu-scraper` 生成的 `_index.md` 报告进行后处理，增强 emoji 使用，提升报告的趣味性和可读性。

---

## 🎯 使用场景

在 `xiaohongshu-scraper` 完成数据抓取并生成初步 `_index.md` 后，调用此 skill 进行格式美化：

```
1. xiaohongshu-scraper 生成 raw.json 和初步 _index.md
2. 调用 xiaohongshu-formatter
3. 输出格式增强后的 _index.md
```

---

## 📋 Emoji 使用规范（Moderate 风格）

### 原则
- **适度使用**：增强可读性和趣味性，不过度装饰
- **语义相关**：emoji 与内容相关，不随机堆砌
- **保持专业**：报告依然保持数据分析的可信度

### 必须使用 Emoji 的位置

| 位置 | Emoji 示例 | 说明 |
|:-----|:-----------|:-----|
| 报告标题 | ☕️ 北京咖啡厅探店报告 ☕️ | 前后各一个 |
| 8个主要板块标题 | 📊 搜索概览、🔥 声量分析、💬 高频共识 | 标题前加一个 |
| 场景推荐表格 | ☕ Dirty首选、📚 自习办公 | 场景列加 emoji |
| 避雷问题类型 | 🚫 空间问题、💸 价格问题 | 问题分类标识 |
| 优缺点列表前缀 | ✨ 核心优点、😤 核心痛点 | 列表标题 |
| 关键洞察编号后 | 1. ☕ Dirty是...、2. 🏙️ 胡同咖啡... | 每条洞察一个 |
| 表格内的评级 | ⭐⭐⭐⭐⭐ | 质量评分 |
| 热度指示 | 🔥🔥🔥 | 3-5个火焰表示热度 |

### 可选使用 Emoji 的位置

| 位置 | 使用建议 |
|:-----|:---------|
| 拥挤程度 | 😊/😌/😰 表达感受 |

### 不使用 Emoji 的位置

- 引用的用户评论内容（保持原始感）
- 来源标注行
- 长段落正文
- 价格档位（保持清晰简洁）

---

## 🛠️ 执行流程

1. **读取现有报告**
   - 定位到搜索目录下的 `_index.md`
   - 读取全部内容

2. **应用格式化**
   - 按照上述 emoji 规范增强格式
   - 确保各主要章节之间使用 Markdown 分割线（`---`）
   - 清理用户评论中的小红书格式标记（见下方"清理规则"）
   - 保持原有内容和结构不变

3. **覆盖原文件**
   - 将格式化后的内容写回 `_index.md`
   - 或者先输出到终端供用户预览

---

## 📏 分割线规范

### 必须添加分割线的位置

在报告的以下主要章节之间**必须**添加 Markdown 分割线（`---`）：

| 位置 | 说明 |
|:-----|:-----|
| 搜索概览 → 品牌/主题声量分析 | 第一章节后 |
| 品牌/主题声量分析 → 评论区高频共识 | 第二章节后 |
| 评论区高频共识 → 场景化决策建议 | 第三章节后 |
| 场景化决策建议 → 价格/规格参考 | 第四章节后 |
| 价格/规格参考 → 避雷指南 | 第五章节后 |
| 避雷指南 → 关键洞察 | 第六章节后 |

**分割线格式：**
```markdown
---
```

**注意：** 子章节（如"讨论热度分布"、"核心优点"等）之间不需要分割线，只在 8 个主要板块之间使用。

## 🧹 清理规则

在格式化过程中，同时清理小红书评论系统的格式标记：

### 回复标记清理

小红书评论中，回复他人的评论会带有 `":"` 前缀，需要移除。

**清理前：**
```markdown
用户反馈：_":"店在新街口平安里附近，在胡同里，是真的不方便停车"_
```

**清理后：**
```markdown
用户反馈：_"店在新街口平安里附近，在胡同里，是真的不方便停车"_
```

**规则：**
- 匹配模式：`_(":"` → 替换为 `_("`
- 只在用户评论（斜体引号包裹的内容）中应用
- 不影响其他位置的 `":"` 符号

**清理位置：**
- 用户反馈块：`用户反馈：_(":"XXX")_` → `用户反馈：_("XXX")_`
- 评论引用：`_" ":"XXX"_` → `_("XXX")_`
