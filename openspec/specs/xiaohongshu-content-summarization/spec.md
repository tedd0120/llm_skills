# 小红书内容总结规范

## 目的

定义 AI agent 处理小红书爬虫原始数据并生成总结报告的行为规范。

---

## 新增需求

### 需求:搜索词衍生
AI agent 必须根据用户提供的搜索主旨，自动生成 2-3 个衍生搜索词，以扩大搜索覆盖面。

#### 场景:生成衍生搜索词
- **当** 用户提供搜索主旨（如 "2026年春季穿搭"）
- **那么** agent 必须生成 2-3 个语义相关的衍生搜索词（如 "春装搭配"、"早春outfit"、"春季通勤穿搭"），连同原始主旨共 3-4 个关键词传入抓取脚本

### 需求:帖子内容AI总结
AI agent 必须读取抓取脚本输出的原始文本数据，为每篇帖子生成独立的 markdown 总结文件。总结必须包含主贴内容摘要和评论区要点。

#### 场景:总结单篇帖子
- **当** agent 读取到一篇帖子的原始数据（标题、正文、评论列表）
- **那么** agent 必须生成一个 markdown 文件，包含：帖子元信息（作者、时间、互动数据、原始链接）、主贴内容摘要、评论区要点总结

#### 场景:评论区为空的帖子
- **当** 帖子没有评论
- **那么** agent 必须在总结文件中标注「无评论」，仅包含主贴内容摘要

### 需求:输出目录组织
系统必须按搜索主旨和时间戳创建独立的输出文件夹，每篇帖子的总结存为独立 .md 文件。

#### 场景:创建输出目录
- **当** agent 开始处理一次搜索任务
- **那么** 必须创建目录 `data/xiaohongshu/{搜索主旨}_{YYYYMMDD_HHmmss}/`，文件夹名使用搜索主旨原文（截断至 20 字符）+ 时间戳

#### 场景:帖子文件命名
- **当** agent 写入单篇帖子的总结
- **那么** 文件必须命名为 `{序号}_{标题摘要}.md`，序号从 01 开始零填充，标题摘要截取前 15 个字符

### 需求:搜索概览索引
agent 必须在输出目录中生成一个 `_index.md` 索引文件，概述本次搜索的全局信息。

#### 场景:生成索引文件
- **当** 所有帖子总结完成
- **那么** agent 必须生成 `_index.md`，包含：搜索主旨、使用的搜索关键词列表、抓取帖子总数、各帖子标题与文件链接列表、搜索时间

### 需求:帖子去重
当多个搜索词返回相同帖子时，agent 必须按帖子 URL 去重，避免重复总结。

#### 场景:重复帖子过滤
- **当** 两个搜索词的结果中包含相同 URL 的帖子
- **那么** 该帖子只总结一次，在 `_index.md` 中标注该帖子被多个关键词命中

## 移除需求 (来自 xhs-tasks-enforcement)

### 需求:生成报告后必须继续执行格式化步骤

**Reason**: 格式化职责已迁移到 xiaohongshu-scraper 编排层，summarize 只负责生成 `_index.md`。

**Migration**: 由 xiaohongshu-scraper 在调用 summarize 后，自行调用 xiaohongshu-formatter。

#### 场景:旧行为（已移除）

- ~~当~~ summarize 生成 `_index.md`
- ~~那么~~ 必须继续阅读 formatter/SKILL.md 并执行格式化
- ~~并且~~ 必须发送报告到用户对话框

### 需求:生成报告后必须发送到用户对话框

**Reason**: 发送职责已迁移到 xiaohongshu-scraper 编排层。

**Migration**: 由 xiaohongshu-scraper 在格式化完成后，负责发送最终报告。

#### 场景:旧行为（已移除）

- ~~当~~ summarize 完成报告生成和格式化
- ~~那么~~ 必须将报告发送到用户对话框

## 新增需求 (来自 xhs-tasks-enforcement)

### 需求:summarize 只负责生成 _index.md
xiaohongshu-summarize 的职责边界明确为：读取 raw.json，生成 `_index.md`，结束。

#### 场景:生成报告后结束
- **当** summarize 成功生成 `_index.md`
- **那么** 任务完成
- **禁止** 自动执行格式化或发送操作
