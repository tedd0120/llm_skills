# 小红书登录功能规范

独立处理小红书登录与 Cookie 校验，供内容抓取功能前置调用。

---

## 新增需求

### 需求: 登录状态检查
系统必须提供 `--check-only` 模式，在不触发登录流程的情况下验证当前 Cookie 是否有效。

#### 场景: Cookie 有效
- **当** 用户使用 `--check-only` 模式且 Cookie 文件存在并加载后访问搜索结果页检测到用户已登录
- **那么** 输出 `LOGIN_OK` 并以退出码 0 退出

#### 场景: Cookie 无效或不存在
- **当** 用户使用 `--check-only` 模式且 Cookie 文件不存在，或加载后检测到未登录状态
- **那么** 输出表示需要登录的非成功状态
- **并且** 禁止触发二维码登录流程

### 需求: 扫码登录
系统默认执行登录命令时，必须在单次浏览器会话内先检查当前登录状态，并仅在需要时继续扫码登录。

#### 场景: 已登录时快速返回
- **当** 用户未使用 `--check-only` 模式启动登录脚本
- **并且** 脚本加载现有 Cookie 后访问搜索结果页检测到已登录
- **那么** 必须直接输出 `LOGIN_OK`
- **并且** 禁止重新打开第二个浏览器会话
- **并且** 禁止进入二维码登录流程

#### 场景: 首次登录
- **当** 用户未使用 `--check-only` 模式启动登录脚本
- **并且** Cookie 文件不存在或检测到未登录状态
- **那么** 系统必须在当前浏览器会话中继续打开登录界面
- **那么** 截取二维码并保存到项目根目录
- **那么** 输出 `NEED_LOGIN:xhs_qr_login.png`（包含二维码文件绝对路径）
- **那么** 等待用户扫码完成
- **那么** 扫码成功后保存 Cookie 到环境变量 `XHS_AUTH_STATE` 指定路径
- **那么** 输出 `LOGIN_SUCCESS` 并删除二维码图片
- **并且** 禁止通过退出脚本再重新启动第二次登录命令来继续该流程

#### 场景: Cookie 过期重新登录
- **当** 用户未使用 `--check-only` 模式启动登录脚本
- **并且** Cookie 文件存在但加载后检测到未登录状态
- **那么** 执行与首次登录相同的单会话扫码流程

#### 场景: 首次登录输出时序
- **当** 用户未使用 `--check-only` 模式启动登录脚本
- **并且** Cookie 文件不存在或检测到未登录状态
- **那么** 系统必须先输出 `NEED_LOGIN:<qr_path>`
- **并且** `<qr_path>` 必须是二维码图片的绝对路径
- **然后** 才进入等待用户扫码完成阶段

#### 场景: Cookie 过期重新登录输出时序
- **当** 用户未使用 `--check-only` 模式启动登录脚本
- **并且** Cookie 文件存在但加载后检测到未登录状态
- **那么** 系统必须先输出 `NEED_LOGIN:<qr_path>`
- **并且** `<qr_path>` 必须是二维码图片的绝对路径
- **然后** 才进入等待用户扫码完成阶段

### 需求: 登录超时处理
系统必须支持可配置的扫码等待超时时间，默认 120 秒。

#### 场景: 扫码超时
- **当** 用户在超时时间内未完成扫码
- **那么** 输出 `LOGIN_TIMEOUT` 并以非零退出码退出
- **那么** 清理二维码图片文件

### 需求: 登录异常处理
系统必须处理登录过程中的各类异常情况。

#### 场景: 浏览器异常
- **当** 浏览器启动失败或页面加载异常
- **那么** 输出 `LOGIN_FAILED` 并以非零退出码退出

#### 场景: 二维码加载失败
- **当** 登录页二维码元素无法找到或加载超时
- **那么** 输出 `LOGIN_FAILED` 并以非零退出码退出

### 需求: 超时配置
系统必须支持通过命令行参数配置扫码等待超时时间。

#### 场景: 自定义超时
- **当** 用户指定 `--timeout N` 参数（N 为秒数）
- **那么** 扫码等待时间使用指定的 N 秒而非默认的 120 秒

## 新增需求 (来自 minimize-browser-window-on-windows)

### 需求: Windows 环境下浏览器必须最小化启动

login 组件在 Windows 环境下启动浏览器时，必须使用 `--start-minimized` 参数使窗口最小化。

#### 场景: Windows 环境启动

- **当** 在 Windows 环境下启动 Edge 浏览器
- **那么** 必须添加 `--start-minimized` 参数
- **并且** 窗口应以最小化状态启动

#### 场景: 非 Windows 环境启动

- **当** 在 Linux 或其他非 Windows 环境下启动浏览器
- **那么** 不添加 `--start-minimized` 参数
- **并且** 保持现有启动行为
