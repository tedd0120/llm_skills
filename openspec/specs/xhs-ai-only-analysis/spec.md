# 小红书 AI 纯分析规范

小红书报告生成必须使用 AI 自然语言理解，禁止自动化语义分析脚本。

## 新增需求

### 需求: 禁止语义分析脚本

Agent 在生成 `_index.md` 报告时，**必须禁止**调用任何外部脚本或程序进行文本分析或内容提取。

禁止的脚本类型包括但不限于：
- Python `re` 模块进行正则表达式匹配
- Python `collections.Counter` 进行词频统计
- 任何实体识别脚本
- 任何情感分析脚本
- 任何关键词提取脚本
- 任何文本分类脚本

#### 场景: 报告生成时禁止调用语义分析脚本

- **当** Agent 正在生成 `_index.md` 报告
- **那么** Agent **禁止**调用任何外部脚本进行文本分析
- **并且** 所有内容理解、分析、总结**必须**基于 LLM 的自然语言理解能力完成

### 需求: 允许基本数值统计

Agent **允许**读取 raw.json 并进行基本数值统计。

允许的操作包括：
- 获取数组长度（如帖子数量）
- 数值求和（如总评论数）
- 百分比计算
- 阈值比较（如判断是否小于 15 篇）

#### 场景: 允许基本数值统计

- **当** Agent 需要获取数据规模信息
- **那么** Agent **可以**进行基本数值统计（如 len、sum）
- **但** **不得**使用脚本进行语义分析

### 需求: Emoji 使用自由化

Agent 在生成报告时使用的 emoji **不**受固定映射规则约束。

#### 场景: Emoji 自由选择

- **当** Agent 生成报告时
- **那么** Agent 根据上下文自行判断使用何种 emoji 及其位置
- **并且** **无需**遵循任何固定的 emoji 映射规则

### 需求: 定性描述替代精确计数

报告中的提及次数**应当**使用定性描述而非精确计数。

推荐的定性描述包括：
- "多位用户提到"
- "广泛讨论"
- "高频反馈"
- "少数用户提及"
- "普遍反映"

#### 场景: 使用定性描述

- **当** Agent 需要描述某个观点的提及频率
- **那么** Agent **应当**使用定性描述（如"多位用户提到"）
- **并且** **避免**使用"提及 ~X 次"这种精确计数格式

#### 场景: 粗略估计

- **当** Agent 认为数量信息对用户有参考价值
- **那么** Agent **可以**使用粗略估计（如"约 5-10 位用户"）
- **但** **禁止**使用脚本进行精确词频统计
