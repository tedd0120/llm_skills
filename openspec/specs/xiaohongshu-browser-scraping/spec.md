# 小红书浏览器爬虫规范

## 目的

定义使用 Playwright 自动化抓取小红书搜索结果和帖子内容的行为规范。

---

## 新增需求

### 需求:浏览器自动化启动
系统必须使用 Playwright 启动浏览器实例。Windows 环境下必须优先使用 `channel="msedge"`，Linux 环境下必须支持 headless 模式降级。

#### 场景:Windows 有头模式启动
- **当** 系统在 Windows 环境运行
- **那么** 必须以 `headless=False, channel="msedge"` 启动浏览器

#### 场景:Linux 无 DISPLAY 降级
- **当** 系统在 Linux 环境运行且未检测到 DISPLAY 环境变量
- **那么** 必须以 `headless=True` 启动浏览器，并应用 stealth 插件

### 需求:QR码登录
系统必须支持通过 QR 码扫码登录小红书。系统必须在登录页面截取 QR 码区域并保存为图片文件，将图片路径输出到终端。

#### 场景:首次登录
- **当** 不存在有效的 Cookie 持久化文件
- **那么** 系统必须打开小红书登录页面，截取 QR 码图片，输出图片路径，等待用户扫码完成登录

#### 场景:已有 GUI 环境首次登录
- **当** 系统运行在有 GUI 的环境且不存在有效 Cookie
- **那么** 系统必须弹出浏览器窗口展示 QR 码，同时将 QR 码截图保存为文件

### 需求:Cookie持久化
系统必须在登录成功后将浏览器状态（cookies + localStorage）持久化为 JSON 文件。持久化路径必须从 `.env` 的 `XHS_AUTH_STATE` 变量读取。

#### 场景:登录后保存状态
- **当** 用户成功扫码登录
- **那么** 系统必须调用 `context.storage_state()` 将浏览器状态保存到 `XHS_AUTH_STATE` 指定的路径

#### 场景:后续启动加载状态
- **当** `XHS_AUTH_STATE` 指定的文件存在且有效
- **那么** 系统必须使用 `storage_state` 参数加载已保存的浏览器状态，跳过登录流程

#### 场景:Cookie过期
- **当** 加载已保存状态后页面检测到未登录
- **那么** 系统必须自动回退到 QR 码登录流程

### 需求:关键词搜索
系统必须接受一个搜索关键词参数，在小红书搜索框中输入并执行搜索。搜索结果必须按「综合」排序。

#### 场景:执行搜索
- **当** 提供搜索关键词（如 "春季穿搭"）
- **那么** 系统必须导航到小红书探索页面，在搜索框输入关键词，执行搜索，等待结果加载

#### 场景:多关键词搜索
- **当** 提供多个搜索关键词
- **那么** 系统必须依次搜索每个关键词，对结果按帖子 URL 去重

### 需求:帖子内容抓取
系统必须打开搜索结果中的帖子，提取帖子正文文字内容。系统禁止下载图片或视频。

#### 场景:抓取图文帖
- **当** 打开一个图文类型帖子
- **那么** 系统必须提取帖子标题、正文文字、作者名称、点赞数、收藏数、评论数、帖子 URL

#### 场景:抓取视频帖
- **当** 打开一个视频类型帖子
- **那么** 系统必须提取帖子标题、描述文字、作者名称、互动数据、帖子 URL，禁止下载视频

### 需求:评论区抓取
系统必须抓取帖子评论区的文字内容。评论加载禁止超过 2 页（或约 30-50 条评论），以避免触发风控。

#### 场景:加载评论
- **当** 打开一个帖子详情页
- **那么** 系统必须滚动加载评论区，最多点击 1-2 次「展开更多评论」，提取可见评论的文字内容

#### 场景:评论区为空
- **当** 帖子没有评论
- **那么** 系统必须记录评论数为 0，继续处理下一篇帖子

### 需求:反风控策略
系统必须实施反风控措施以避免触发小红书的自动化检测。

#### 场景:帖子间延迟
- **当** 完成一篇帖子的抓取后
- **那么** 系统必须随机等待 3-8 秒后再打开下一篇帖子

#### 场景:帖子数量上限
- **当** 抓取帖子数达到配置上限（默认 10，最大 20）
- **那么** 系统必须停止抓取并输出已抓取的数据

### 需求:结构化输出
系统必须将抓取的原始文本以 JSON 格式输出到 stdout 或临时文件，供 agent 层读取。

#### 场景:输出格式
- **当** 完成所有帖子的抓取
- **那么** 系统必须输出包含帖子列表的 JSON，每个帖子包含：标题、正文、作者、互动数据、评论列表、原始 URL

#### 场景:输出到文件
- **当** 指定 `--output` 参数
- **那么** 系统必须将 JSON 输出写入指定文件路径

### 需求:CLI接口
脚本必须提供命令行接口，支持通过参数控制搜索关键词、帖子上限、输出路径等。

#### 场景:CLI调用
- **当** 用户在终端执行 `python fetch_xhs.py --keywords "春季穿搭" --max-posts 10 --output result.json`
- **那么** 系统必须使用指定参数执行抓取并将结果写入 `result.json`

#### 场景:默认参数
- **当** 用户未指定 `--max-posts`
- **那么** 系统必须使用默认值 10
