# group-members-org-tree-skill 规范

## 目的
定义“成员 JSON -> 组织架构树 HTML”能力的行为规范。
该能力最初由归档变更 `add-teams-org-tree-html-skill` 引入，并于 2026-02-10 合并到 `.agent/skills/teams-group-members` 统一维护。
## 需求
### 需求：从成员 JSON 生成组织架构树 HTML

系统**必须**支持读取已抓取的成员 JSON 列表，并生成可本地查看的组织架构树 HTML 文件。
系统**必须**在群成员抓取流程中，基于本次抓取得到的最新成员数据生成 HTML。
系统**必须**通过抓取入口统一触发该流程，**禁止**暴露独立 HTML 生成 CLI 入口。

#### 场景：单群抓取后自动生成
- 当用户执行单群抓取
- 系统**必须**使用本次抓取成员数据生成 HTML
- 系统**禁止**要求用户再执行独立 HTML 生成命令

#### 场景：批量抓取后自动生成
- 当用户执行批量抓取
- 系统**必须**使用去重后的最终成员数据生成 HTML
- 系统**必须**保证该 HTML 反映最新抓取结果

#### 场景：读取已有 JSON 生成 HTML
- 当用户提供成员 JSON 输入文件路径
- 系统**必须**解析成员列表并输出 HTML 文件
- 系统**禁止**在该流程中调用群成员抓取 API

### 需求：组织树按 superior 关系构建

系统**必须**以成员字段 `superior` 作为父子关系依据构建组织树，并**必须**保留多个根节点展示。

#### 场景：按 superior 关联上下级

- 当成员 A 的 `superior` 等于成员 B 的 `name`
- 系统**必须**将 A 作为 B 的下级节点展示

#### 场景：无上级或无法匹配上级时保留多根

- 当成员 `superior` 为空，或未匹配到任何成员 `name`
- 系统**必须**将该成员作为根层节点展示
- 系统**禁止**强制合并为单一虚拟根节点

### 需求：页面支持搜索与高亮

HTML 页面**必须**提供简单搜索能力，并对命中节点高亮显示。

#### 场景：搜索命中姓名/工号/部门

- 当用户输入关键字
- 系统**必须**在姓名、工号、部门字段中进行包含匹配
- 命中节点**必须**高亮，且其祖先路径**必须**可见

### 需求：页面展示抓取日期与总人数

HTML 页面**必须**展示数据抓取日期与总人数。

#### 场景：显式传入抓取日期

- 当 CLI 提供抓取日期参数
- 页面**必须**显示该日期
- 页面**必须**显示 JSON 解析后的总人数

#### 场景：未显式传入抓取日期

- 当 CLI 未提供抓取日期参数
- 页面**必须**回退显示输入 JSON 文件的最后修改日期

### 需求：根节点点击仅展开一层下级

HTML 页面**必须**以画布式决策树展示组织结构，并支持节点点击展开直属下级。

#### 场景：首屏仅显示根节点

- 当页面首次加载
- 系统**必须**仅显示根节点（`superior` 为空或无法匹配上级）并横向排布

#### 场景：点击节点展开直属下级

- 当用户点击某个可展开节点
- 系统**必须**在该节点下方展示其直属下级
- 系统**必须**保持更深层级默认折叠，除非用户继续点击展开

### 需求：画布支持拖动与缩放

组织树画布**必须**支持用户拖动和平滑缩放。

#### 场景：鼠标滚轮缩放

- 当用户在画布上滚轮操作
- 系统**必须**按缩放比例放大/缩小画布内容

#### 场景：鼠标拖动画布
- 当用户按住画布拖动
- 系统**必须**平移画布视图

### 需求：组织树默认布局必须紧凑可读

组织树页面**必须**采用紧凑布局参数，在保持文本可读性的前提下减少横向占用。

#### 场景：默认视图紧凑展示
- 当页面首次渲染组织树
- 系统**必须**使用不大于 `150px` 的节点宽度
- 系统**必须**使用不大于 `180px` 的层级横向间距
- 系统**必须**保证节点姓名与部门文本仍可读（可截断但不可为空）

### 需求：节点悬停 1 秒展示祖先链

页面**必须**在鼠标悬停节点超过 1 秒后显示该节点上游祖先链信息，内容格式为“姓名-部门”。

#### 场景：悬停到达阈值显示提示
- 当用户将鼠标停留在节点上达到 1 秒
- 系统**必须**显示从根节点到父节点的祖先链
- 系统**必须**按“根 -> ... -> 父级”顺序展示每一项“姓名-部门”

#### 场景：悬停不足阈值不显示
- 当用户在节点上停留不足 1 秒即移开
- 系统**禁止**显示祖先链提示

### 需求：节点文本可选择且不触发画布平移

页面**必须**允许用户在节点中进行文本框选，并避免与画布拖动行为冲突。

#### 场景：节点文本框选
- 当用户在节点文本区域执行鼠标拖动框选
- 系统**必须**允许文本被选中
- 系统**禁止**在该操作中触发画布平移

#### 场景：画布背景拖动
- 当用户在非节点背景区域按住拖动
- 系统**必须**继续支持画布平移

### 需求：页面视觉必须采用 iOS 极简风格

组织树页面**必须**采用 iOS 极简视觉风格，并使用 Apple 系优先字体栈。

#### 场景：字体与视觉风格一致
- 当页面加载完成
- 系统**必须**使用包含 `-apple-system` 的字体栈
- 系统**必须**使用低饱和配色、轻阴影和简洁圆角样式

### 需求：姓名搜索命中后自动居中

组织树页面**必须**在姓名搜索命中时自动将命中节点定位到视图中心。

#### 场景：姓名搜索命中自动聚焦
- 当用户在搜索框输入姓名关键字并命中至少一个节点
- 系统**必须**自动平移画布，使当前命中节点位于可视区域中心
- 系统**必须**保留该节点及其他命中节点的高亮状态

#### 场景：姓名无命中不触发聚焦
- 当用户输入姓名关键字但没有命中节点
- 系统**禁止**触发自动居中跳转

### 需求：同名命中支持 pre/next 切换

当姓名搜索命中多个同名节点时，页面**必须**提供 `pre` / `next` 控件用于切换当前聚焦节点。

#### 场景：next 切换命中节点
- 当存在多个同名命中且用户点击 `next`
- 系统**必须**切换到下一个命中节点并自动居中
- 当当前已是最后一个命中时，系统**必须**循环切换到第一个命中

#### 场景：pre 切换命中节点
- 当存在多个同名命中且用户点击 `pre`
- 系统**必须**切换到上一个命中节点并自动居中
- 当当前已是第一个命中时，系统**必须**循环切换到最后一个命中

### 需求：姓名匹配优先级规则

姓名搜索命中集合**必须**遵循精确匹配优先、包含匹配回退的顺序。

#### 场景：存在精确姓名匹配
- 当关键字与至少一个节点姓名完全一致
- 系统**必须**仅在精确匹配集合中进行聚焦与 pre/next 切换

#### 场景：不存在精确姓名匹配
- 当关键字与所有节点姓名都不完全一致
- 系统**必须**回退到姓名包含匹配集合进行聚焦与切换

### 需求：保留搜索高亮与头部信息

系统**必须**保留搜索高亮、抓取日期、总人数展示。

#### 场景：搜索命中并高亮

- 当用户输入关键字
- 系统**必须**按姓名/工号/部门匹配并高亮命中节点
- 系统**必须**自动展开命中节点的祖先路径

#### 场景：头部信息展示

- 当页面渲染完成
- 系统**必须**展示数据抓取日期与总人数
