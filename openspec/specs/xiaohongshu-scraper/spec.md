# 小红书抓取 Skill 规范

## 目的

定义 xiaohongshu-scraper Skill 的行为规范，确保 Agent 在执行小红书内容抓取和分析任务时严格遵循分层约束体系，使用明确的优先级标签，避免使用模糊语言导致任务失败或质量下降。

## 需求

### 需求:Agent 执行规范必须使用三层约束标签
SKILL.md 和 workflow.md 中的所有操作指令必须使用明确的优先级标签标识约束等级。

#### 场景:核心要求标签
- **当** 指令描述任务成功必须完成的步骤
- **那么** 必须使用 `[核心要求]` 标签
- **并且** 关键词使用"必须"、"禁止"、"绝对"

#### 场景:标准操作标签
- **当** 指令描述强烈建议但可在特殊情况下灵活调整的步骤
- **那么** 必须使用 `[标准操作]` 标签
- **并且** 关键词使用"应当"、"要求"

#### 场景:灵活选项标签
- **当** 指令描述可选的替代路径或特殊情况的备选方案
- **那么** 必须使用 `[灵活选项]` 标签
- **并且** 关键词使用"可"、"可以"、"可选"

#### 场景:参考信息标签
- **当** 指令仅提供提示信息，不约束 Agent 行为
- **那么** 必须使用 `[参考信息]` 标签
- **并且** 关键词使用"注意"、"提示"、"⏱️"

### 需求:必须定义优先级冲突处理规则
文档必须包含清晰的优先级定义章节，说明各标签的优先级关系和冲突处理逻辑。

#### 场景:优先级降序
- **当** Agent 面临多个冲突指令
- **那么** 必须按照 P0 > P1 > P2 > P3 的优先级降序执行

#### 场景:P0 受阻处理
- **当** P0 核心要求无法执行
- **那么** Agent 必须停止任务并报告失败

#### 场景:P1 降级到 P2
- **当** P1 标准操作受阻但存在 P2 灵活选项
- **那么** Agent 可降级使用 P2 方案
- **并且** 必须在输出中标注降级原因

#### 场景:P2/P3 忽略
- **当** P2 或 P3 指令无法执行
- **那么** Agent 可忽略该指令并继续任务

### 需求:SKILL.md 必须移除模糊语言
SKILL.md 中所有使用"建议"、"推荐"等模糊词汇的指令必须替换为对应优先级的明确表述。

#### 场景:Xvfb 配置指令
- **当** SKILL.md 描述 Linux 环境使用 Xvfb
- **那么** 必须标记为 `[标准操作]` 应当使用 Xvfb
- **并且** headless 降级方案标记为 `[灵活选项]`

#### 场景:数据保存指令
- **当** SKILL.md 描述抓取结果保存
- **那么** 必须标记为 `[核心要求]` 必须保存到临时 JSON

#### 场景:时间提示信息
- **当** SKILL.md 提示抓取过程耗时
- **那么** 必须标记为 `[参考信息]` ⏱️ 过程可能需要几分钟

#### 场景:故障排除指令
- **当** SKILL.md 描述元素选择器失效时的处理
- **那么** 必须标记为 `[核心要求]` 必须检查 scripts/selectors.py

### 需求:workflow.md 必须同步约束标签
workflow.md 中的步骤描述必须使用与 SKILL.md 一致的优先级标签体系。

#### 场景:强制输出流程步骤
- **当** workflow.md 步骤 4 描述输出要求
- **那么** 每个子步骤必须添加 `[核心要求]` 标签前缀
- **并且** 保持"必须"、"禁止"等关键词

#### 场景:新增分层步骤
- **当** workflow.md 添加标准操作和灵活选项步骤
- **那么** 必须使用对应的 `[标准操作]` 和 `[灵活选项]` 标签
