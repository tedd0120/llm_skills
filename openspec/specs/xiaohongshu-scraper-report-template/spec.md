## 修改需求

### 需求: 报告概览必须采用嵌入式格式
报告概览部分必须使用方案A的嵌入式设计，包含基本信息表格和搜索策略子表格。

#### 场景: 生成带有搜索策略的概览
- **当** raw.json 包含 `search_strategy` 字段且非空
- **那么** 报告概览必须包含基本信息表格（搜索任务、执行时间、数据规模）
- **并且** 必须包含独立的"🔍 搜索策略"子表格
- **并且** 搜索策略表格必须包含"关键词"、"篇数"、"搜索意图"三列

#### 场景: 生成缺少搜索策略的概览
- **当** raw.json 不包含 `search_strategy` 字段或字段为空
- **那么** 报告概览必须仅显示基本信息表格
- **并且** 必须省略搜索策略子表格

### 需求: 表头列名必须使用友好的 Emoji 前缀描述
报告概览表格的列名必须使用 Emoji 前缀，而不是"字段"/"值"等生硬描述。

#### 场景: 验证表头格式
- **当** 渲染报告概览表格
- **那么** 第一列必须使用 "🎯 搜索任务"、"⏰ 执行时间"、"📊 数据规模" 等格式
- **并且** 禁止使用 `<b>` 标签（Markdown 表头自动加粗）

### 需求: 搜索意图必须映射到对应 Emoji
搜索策略表格中的关键词必须根据其意图类型添加对应 Emoji 前缀。

#### 场景: 映射推荐类意图
- **当** 搜索意图包含"推荐"、"趋势"等关键词
- **那么** 关键词必须添加 `📍` 前缀

#### 场景: 映射探店类意图
- **当** 搜索意图包含"探店"、"体验"等关键词
- **那么** 关键词必须添加 `🏪` 前缀

#### 场景: 映射避坑类意图
- **当** 搜索意图包含"避坑"、"负面"等关键词
- **那么** 关键词必须添加 `⚠️` 前缀

#### 场景: 映射对比类意图
- **当** 搜索意图包含"对比"、"评价"等关键词
- **那么** 关键词必须添加 `⚖️` 前缀

#### 场景: 映射教程类意图
- **当** 搜索意图包含"教程"、"攻略"等关键词
- **那么** 关键词必须添加 `📖` 前缀

#### 场景: 映射价格类意图
- **当** 搜索意图包含"价格"、"优惠"等关键词
- **那么** 关键词必须添加 `💰` 前缀

#### 场景: 未知意图类型
- **当** 搜索意图不匹配任何已知类型
- **那么** 关键词必须使用默认的 `🔍` 前缀
