# 小红书内容总结输出格式要求

## 适用范围

本文档对小红书内容搜索和总结流程中的输出环节增加强约束要求。

---

## 新增需求

### 需求: 抓取数据必须不包含 URL 字段
`fetch_xhs.py` 脚本输出的 JSON 数据必须不包含 `url` 字段，只保留标题、内容、作者、日期、点赞数、收藏数、评论数和评论列表。

#### 场景: 数据抓取
- **当** 脚本抓取小红书帖子数据
- **那么** 输出的 JSON 必须包含 8 个字段：title, content, author, date, likes, collects, comments_count, comments
- **并且** 禁止包含 url 字段

#### 场景: 数据兼容性
- **当** 读取已有的 raw.json 数据
- **那么** Agent 应当正常处理，不依赖 url 字段的存在

---

## 修改需求

### 需求: Agent 输出流程
Agent 在执行小红书内容搜索和总结时，必须遵守以下流程：

1. **必须创建结果目录**：必须在 `data/xiaohongshu/` 下创建一个以主题和时间戳命名的独立目录。
2. **必须生成增强版汇总报告**：必须直接从 JSON 数据生成 `_index.md` 增强版汇总报告。
   * 禁止跳过任何分析板块。
   * 禁止使用"导航链接 + 一句简短判语"代替详尽分析。
3. **必须保留可追溯性**：报告中的结论必须可追溯到具体帖子或评论。
   * 必须包含帖子标题、作者和日期。
   * 必须包含关键引用（用户反馈、典型评论）。

#### 场景: 成功生成增强版报告
- **当** Agent 完成小红书内容抓取，获得 JSON 数据
- **那么** Agent 必须直接读取 JSON，生成包含以下所有板块的 `_index.md`：
  - 搜索任务摘要
  - 品牌/主题声量分析
  - 多维对比矩阵
  - 评论区高频共识（Top 5 优点 & Top 5 痛点）
  - 场景化决策建议
  - 价格/规格参考（如适用）
  - 避雷指南
  - 关键洞察

#### 场景: 数据不足处理
- **当** 抓取的帖子数量少于 15 篇
- **那么** Agent 必须在相关分析板块标注"样本有限"，但不跳过任何板块

#### 场景: 板块不适用处理
- **当** 某主题不适合特定分析板块（如"教程类"主题不适合产品对比矩阵）
- **那么** Agent 可在报告中标注"此板块不适用于当前主题"，并说明理由

### 需求: 增强版汇总报告内容
`_index.md` 报告必须包含以下板块和内容要求：

#### 1. 基础信息板块（必选）

**搜索任务摘要**
- 必须包含：执行时间、使用搜索词、共抓取帖子数

#### 2. 深度分析板块（全部包含）

**品牌/主题声量分析**
- 必须统计并展示主要品牌/主题的讨论热度
- 必须包含：讨论占比、主要提及内容

**多维对比矩阵**
- 必须创建多维度对比表格（如产品对比、方案对比）
- 必须基于主贴内容和评论区对比讨论
- 维度至少包含 3 个（如：价格、性能、用户体验等）

**评论区高频共识**
- 必须统计所有评论，提取 Top 5 核心优点和 Top 5 核心痛点
- 必须包含：提及次数、主要涉及的品牌/主题、典型反馈

**场景化决策建议**
- 必须提炼评论区中"什么情况选哪个/怎么做"的模式
- 必须包含：决策树或场景推荐列表
- 推荐必须有理由支撑

**价格/规格参考**（如适用）
- 如评论中包含价格或规格信息，必须提取并汇总
- 必须包含：项目、信息/价格、备注

**避雷指南**
- 必须汇总各品牌/主题的高频问题和用户反馈
- 必须引用具体评论或用户反馈

**关键洞察**
- 必须提炼 3 条以上跨帖洞察
- 洞察必须基于数据，不能是主观猜测

#### 场景: 产品对比类报告
- **当** 用户搜索主题为产品推荐/对比类（如"扫拖机器人推荐"）
- **那么** 报告必须包含完整的品牌对比矩阵和场景化决策建议

#### 场景: 信息获取类报告
- **当** 用户搜索主题为教程/资讯类（如"咖啡拉花教程"）
- **那么** 报告可将"品牌对比"调整为"内容类型对比"或"技巧汇总"
- **那么** 报告可将"场景化决策"调整为"难度级别建议"或"工具选择建议"

#### 场景: 数据追溯要求
- **当** 报告中引用具体观点或数据
- **那么** 必须包含来源帖子或评论的引用
- **并且** 引用格式应包含帖子标题、作者和日期

---

## 移除需求

### 需求: 单篇帖子文章生成
**Reason**: 单篇 MD 文件使用率低，造成 token 和时间浪费。用户主要价值集中在汇总报告中。
**Migration**: 直接从 JSON 数据生成增强版汇总报告，不再生成单篇帖子 MD 文件。

原需求内容：
> 2. **必须生成单篇帖子文章**：对于抓取到的每一篇有效帖子，**必须**在该目录下生成一份独立的 Markdown 文档。
>     * **禁止**将多篇帖子的内容合并到一个文件中。
>     * **禁止**跳过包含单篇主帖摘要和评论区要点的总结环节。

此需求已移除。
