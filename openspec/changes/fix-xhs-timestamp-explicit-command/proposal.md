## 为什么

当前 `xiaohongshu-scraper` skill 在创建搜索目录时，SKILL.md 中的时间戳示例（如 `20260303_182300`）被 Agent 误认为是可复用的模板，导致目录时间戳使用硬编码值而非当前系统时间。这违反了"必须使用当前系统时间"的核心要求，使得目录无法按真实时间正确排序。

## 变更内容

- 修改 `xiaohongshu-scraper/SKILL.md` 中"步骤 1：创建搜索目录"的指令格式
- 从意图声明（"使用当前系统时间"）改为显式命令（直接提供可执行的 Bash 命令）
- 移除可能诱导模仿的具体数字示例，使用命令模板代替

## 功能 (Capabilities)

### 修改功能

- `xiaohongshu-scraper`: SKILL.md 中目录创建指令的格式变更，从描述性约束改为可执行命令模板

## 影响

- **SKILL.md**: 更新"步骤 1：创建搜索目录"部分
- **Agent 执行流程**: 不再需要推断如何获取时间，直接执行提供的命令
- **目录命名**: 确保所有搜索目录使用真实的当前时间戳
