# 功能：group-members-org-tree-skill

## 新增需求
### 需求：姓名搜索命中后自动居中

组织树页面**必须**在姓名搜索命中时自动将命中节点定位到视图中心。

#### 场景：姓名搜索命中自动聚焦
- 当用户在搜索框输入姓名关键字并命中至少一个节点
- 系统**必须**自动平移画布，使当前命中节点位于可视区域中心
- 系统**必须**保留该节点及其他命中节点的高亮状态

#### 场景：姓名无命中不触发聚焦
- 当用户输入姓名关键字但没有命中节点
- 系统**禁止**触发自动居中跳转

### 需求：同名命中支持 pre/next 切换

当姓名搜索命中多个同名节点时，页面**必须**提供 `pre` / `next` 控件用于切换当前聚焦节点。

#### 场景：next 切换命中节点
- 当存在多个同名命中且用户点击 `next`
- 系统**必须**切换到下一个命中节点并自动居中
- 当当前已是最后一个命中时，系统**必须**循环切换到第一个命中

#### 场景：pre 切换命中节点
- 当存在多个同名命中且用户点击 `pre`
- 系统**必须**切换到上一个命中节点并自动居中
- 当当前已是第一个命中时，系统**必须**循环切换到最后一个命中

### 需求：姓名匹配优先级规则

姓名搜索命中集合**必须**遵循精确匹配优先、包含匹配回退的顺序。

#### 场景：存在精确姓名匹配
- 当关键字与至少一个节点姓名完全一致
- 系统**必须**仅在精确匹配集合中进行聚焦与 pre/next 切换

#### 场景：不存在精确姓名匹配
- 当关键字与所有节点姓名都不完全一致
- 系统**必须**回退到姓名包含匹配集合进行聚焦与切换
