## 上下文

**当前状态**：小红书抓取技能的两阶段工作流（澄清→执行）中，澄清阶段生成的关键词分配策略（关键词、篇数、搜索意图）仅用于与用户确认，执行后即丢弃。报告生成的概览部分仅显示原始关键词列表，无法追溯分配策略。

**约束**：
- Agent 执行流程中，澄清阶段和执行阶段是分离的
- `fetch_xhs.py` 脚本当前只接收关键词字符串数组
- 报告模板硬编码在 SKILL.md 中，由 Agent 手动渲染

**利益相关者**：使用该技能的用户（需要可读的搜索策略说明）、技能维护者（需要清晰的模板）

## 目标 / 非目标

**目标：**
- 保存澄清阶段的搜索策略到 raw.json，供报告生成使用
- 改进报告概览部分的可读性和信息密度
- 统一 Emoji 使用风格

**非目标：**
- 不改变两阶段工作流本身
- 不修改抓取逻辑（仅添加元数据保存）
- 不强制已存在的 raw.json 必须包含新字段（向后兼容）

## 决策

### 1. search_strategy 数据结构

**决策**：在 `raw.json` 顶层添加 `search_strategy` 数组

```json
{
  "search_strategy": [
    {
      "keyword": "北京咖啡厅推荐",
      "posts_count": 10,
      "intent": "获取整体推荐趋势"
    }
  ]
}
```

**理由**：
- 数组格式保持关键词顺序（与分配顺序一致）
- 嵌入式对象结构清晰，易于渲染
- 向后兼容：旧文件不存在此字段时，可降级显示关键词列表

**替代方案**：使用对象以 keyword 为 key
- 反对：失去顺序信息，JSON 对象键无序

### 2. fetch_xhs.py 参数传递

**决策**：添加 `--search-strategy` 参数，接收 JSON 字符串

```bash
python fetch_xhs.py --keywords "k1,k2,k3" \
    --search-strategy '[{"keyword":"k1","posts_count":10,"intent":"..."}]'
```

**理由**：
- 最小侵入性，不改变现有参数
- JSON 字符串易于 Agent 构造和脚本解析
- 可选参数：未提供时不影响现有调用

**替代方案**：修改 `--keywords` 格式
- 反对：破坏向后兼容性，增加解析复杂度

### 3. 报告概览格式（方案A）

**决策**：采用嵌入式设计

```
┌─────────────────────────────────────────┐
│ 📋 搜索概览                              │
├─────────────────────────────────────────┤
│ | 🎯 搜索任务 | 北京咖啡厅推荐 |         │
│ | ⏰ 执行时间 | 2026-03-03 14:30 |       │
│ | 📊 数据规模 | 30篇帖子 · 1935条评论 |   │
│                                         │
│ **🔍 搜索策略**                          │
│ | 关键词 | 篇数 | 搜索意图 |             │
│ | 📍 ... | 10篇 | ... |                 │
└─────────────────────────────────────────┘
```

**理由**：
- 信息分层：基本信息 + 详细策略
- Emoji 前缀提供视觉锚点
- 表格对齐整齐

**替代方案**：纯列表格式
- 反对：篇数/意图对齐不美观

### 4. Emoji 映射规则

**决策**：根据搜索意图类型分配 Emoji

| 意图类型 | Emoji |
|---------|-------|
| 推荐/趋势 | 📍 |
| 探店/体验 | 🏪 |
| 避坑/负面 | ⚠️ |
| 对比/评价 | ⚖️ |
| 教程/攻略 | 📖 |
| 价格/优惠 | 💰 |

**理由**：语义关联，用户直观理解

## 风险 / 权衡

| 风险 | 缓解措施 |
|-----|---------|
| `search_strategy` 缺失时报告失败 | 降级处理：显示关键词列表，警告提示 |
| JSON 参数转义问题 | Agent 使用 Python dict 转换，避免手动构造 |
| 表格渲染在不同客户端不一致 | 使用标准 Markdown 表格语法 |
| 新旧 raw.json 混用 | 添加版本标识字段 |

## 迁移计划

1. 修改 `fetch_xhs.py`：添加 `--search-strategy` 参数解析和保存
2. 更新 SKILL.md 模板：使用新的概览格式
3. Agent 逻辑调整：澄清阶段保存策略，执行阶段传递参数
4. 报告生成降级逻辑：检测 `search_strategy` 是否存在

**回滚策略**：保留 `keywords` 字段，确保旧数据可读
