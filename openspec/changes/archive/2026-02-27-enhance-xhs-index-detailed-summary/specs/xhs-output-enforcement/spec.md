## 修改需求

### 需求:Agent 强制输出流程
Agent 在执行小红书内容搜索和总结时，必须严格遵守以下顺序流程，禁止跳过任何步骤：

1. 必须在 `data/xiaohongshu/` 下创建以主题与时间戳命名的独立目录。
2. 对抓取到的每一篇有效帖子，必须在该目录下生成一份独立 Markdown 文档。
   - 禁止将多篇帖子内容合并到一个文件中。
   - 禁止跳过单篇主帖摘要与评论区要点。
3. 必须在所有单篇 Markdown 完成后，才生成 `_index.md` 索引文件。
   - 索引文件必须包含指向每篇独立 Markdown 的有效链接。
   - 禁止将 `_index.md` 作为唯一格式化输出。
4. `_index.md` 必须作为“深度汇总报告”输出，不得仅为导航列表或榜单。
   - 对每篇帖子，必须在索引内给出结构化详尽摘要，至少覆盖：核心结论、关键证据、评论区共识或分歧、适用边界/场景、可执行启示。
   - 对全量帖子，必须给出跨帖子汇总，至少覆盖：共识结论、主要分歧、按人群或场景的建议、风险与信息缺口。
   - 禁止以“一句简短判语”替代详尽分析；禁止仅输出品牌/型号排名而缺少依据说明。

#### 场景:生成完整且详尽的索引报告
- **当** Agent 完成一次小红书主题抓取并已生成全部单篇 Markdown
- **那么** Agent 必须在 `_index.md` 中为每篇帖子提供结构化详尽摘要，并输出跨帖子共识、分歧、建议与风险缺口，同时保留到单篇文件的有效链接

#### 场景:防止索引内容过度简化
- **当** Agent 试图在 `_index.md` 中使用“导航 + 一句话”或“纯榜单”形式结束任务
- **那么** 该输出必须被视为不合规，Agent 必须补全每篇与跨帖子的深度分析后再完成任务
