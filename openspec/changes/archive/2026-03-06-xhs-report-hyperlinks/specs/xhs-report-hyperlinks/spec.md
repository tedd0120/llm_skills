# 小红书报告超链接功能规范

## 目的

定义小红书报告中为原帖引用添加可点击超链接的功能规范。用户可在澄清阶段选择是否启用此功能。

---

## 新增需求

### 需求:超链接功能可选
系统必须在澄清阶段提供超链接格式选项，用户可选择启用或禁用。默认状态必须为禁用（纯文本格式）。

#### 场景:用户选择纯文本格式
- **当** 用户在澄清阶段选择超链接格式选项 A
- **那么** 报告中的原帖引用必须使用纯文本格式 `*《帖子标题》*`

#### 场景:用户选择超链接格式
- **当** 用户在澄清阶段选择超链接格式选项 B
- **那么** 报告中的原帖引用必须使用超链接格式 `[*《帖子标题》*](https://www.xiaohongshu.com/explore/xxxxx)`

#### 场景:默认行为
- **当** 用户未明确选择超链接格式
- **那么** 系统必须默认使用纯文本格式

### 需求:澄清阶段交互
系统必须在模式选择和关键词确认之后，新增超链接格式选择步骤。

#### 场景:展示超链接选项
- **当** 用户完成关键词确认
- **那么** 系统必须展示超链接格式选择表格，包含纯文本和超链接两个选项

#### 场景:用户确认超链接选项
- **当** 用户回复 A 或 B
- **那么** 系统必须记录用户选择并继续执行流程

### 需需:帖子 ID 提取
系统必须从小红书帖子 URL 中提取帖子 ID。

#### 场景:标准 URL 提取
- **当** URL 格式为 `https://www.xiaohongshu.com/explore/{note_id}`
- **那么** 系统必须提取 `{note_id}` 作为帖子 ID

#### 场景:带参数 URL 提取
- **当** URL 包含查询参数（如 `?xsec_token=...`）
- **那么** 系统必须在提取 ID 前去除查询参数

### 需求:ID-URL 映射文件
系统必须在超链接启用时生成 `id_url_map.json` 文件，包含帖子 ID 到完整 URL 的映射关系。

#### 场景:生成映射文件
- **当** 用户启用超链接功能且完成数据抓取
- **那么** 系统必须在输出目录生成 `id_url_map.json` 文件，格式为 `{"post_id": "full_url", ...}`

#### 场景:超链接禁用时不生成
- **当** 用户未启用超链接功能
- **那么** 系统禁止生成 `id_url_map.json` 文件

### 需求:报告引用格式（超链接启用）
系统必须在超链接启用时，使用 ID 占位符格式生成报告引用。

#### 场景:来源引用
- **当** 报告中引用帖子来源且超链接已启用
- **那么** 必须使用格式 `来源：[*《帖子标题》*](id:{post_id})`

#### 场景:追溯来源
- **当** 报告中追溯来源且超链接已启用
- **那么** 必须使用格式 `追溯来源：[*《帖子标题》*](id:{post_id})`

### 需求:报告引用格式（超链接禁用）
系统必须在超链接禁用时，使用纯文本格式生成报告引用。

#### 场景:来源引用（纯文本）
- **当** 报告中引用帖子来源且超链接未启用
- **那么** 必须使用格式 `来源：*《帖子标题》*`

#### 场景:追溯来源（纯文本）
- **当** 报告中追溯来源且超链接未启用
- **那么** 必须使用格式 `追溯来源：*《帖子标题》*`

### 需求:Formatter URL 替换
系统必须在 formatter 阶段将 ID 占位符替换为实际 URL。

#### 场景:存在映射文件时替换
- **当** `id_url_map.json` 文件存在
- **那么** formatter 必须读取映射文件，将所有 `(id:{post_id})` 替换为 `(https://www.xiaohongshu.com/explore/{post_id})`

#### 场景:映射文件不存在时跳过
- **当** `id_url_map.json` 文件不存在
- **那么** formatter 必须跳过 URL 替换，仅执行原有的 emoji 美化

### 需求:向后兼容
系统必须能够处理没有 post_id 字段的旧数据。

#### 场景:旧 raw.json 处理
- **当** raw.json 中的帖子不包含 post_id 字段
- **那么** summarize 阶段必须降级为纯文本格式，禁止生成超链接引用

#### 场景:旧报告格式化
- **当** _index.md 中不包含 ID 占位符
- **那么** formatter 阶段必须跳过 URL 替换

### 需求:数据结构扩展
系统必须在 raw.json 中为每篇帖子添加 post_id 字段。

#### 场景:新数据包含 post_id
- **当** 系统生成新的 raw.json
- **那么** 每篇帖子必须包含 post_id 字段，值为从小红书 URL 提取的帖子 ID

#### 场景:保留原始 URL
- **当** 系统生成新的 raw.json
- **那么** 每篇帖子必须保留完整的 url 字段
