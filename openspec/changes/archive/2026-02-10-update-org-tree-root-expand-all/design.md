## 上下文

`teams-org-tree-html` 已支持按 `superior -> name` 构树与基础折叠展示，但“点击后展开范围”未显式约束。用户期望是点击根节点时只展开一层下级，避免一次性展开过深层级。

## 目标 / 非目标

- 目标：
  - 明确保留多个根节点（无上级人员）展示。
  - 点击某个根节点时，仅展开该根节点一层下级。
- 非目标：
  - 不引入新的字段推断“职级”。
  - 不改动抓取 Skill 或数据格式。
  - 不新增后端服务。

## 决策

- 决策 1：根节点判定保持数据驱动。
  - `superior` 为空或无法匹配成员 `name` 时，归为根节点。
- 决策 2：根节点展开采用“一层展开”。
  - 用户点击根节点时，仅展开根节点本身；其直接下级可见，但下级的下级保持折叠。
- 决策 3：非根节点保持现有逐级展开行为，减少交互变化范围。

## 风险 / 权衡

- 仅展开一层可能需要用户多次点击浏览深层结构；作为最小实现先接受该交互权衡。
- “职级最大”未在数据中提供结构化字段，本次按“无上级即根节点”解释并在文档中声明。
