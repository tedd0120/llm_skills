# 小红书爬虫执行规范 - 增量变更

## 新增需求

### 需求:确认后必须立即创建搜索目录
系统必须在用户确认关键词后立即创建搜索目录，所有后续产出文件必须在此目录下，避免搜索阶段出现bug时重复创建文件夹。

#### 场景:目录创建时机
- **当** 用户在澄清阶段明确确认搜索关键词
- **那么** 系统必须在调用 fetch_xhs.py 之前立即创建搜索目录
- **禁止** 等待爬虫脚本执行完成后再创建目录

#### 场景:目录命名格式
- **当** 创建搜索目录
- **那么** 目录格式必须为 `yyyyMMdd_hhMMss_搜索主旨`
- **那么** 时间戳必须使用当前时间，精确到秒
- **那么** 搜索主旨必须基于用户确认的关键词主题提取
- **示例**：`20260303_143000_北大口腔拔牙`

#### 场景:文件输出约束
- **当** 调用 fetch_xhs.py 脚本
- **那么** 必须使用 --output 参数指定输出到搜索目录下的 raw.json
- **当** 生成 _index.md 报告
- **那么** 必须输出到搜索目录下
- **禁止** 在搜索目录外创建任何产出文件

#### 场景:目录冲突防护
- **当** 搜索目录已存在
- **那么** 系统必须警告用户并询问是否覆盖或使用新目录
- **或者** 自动在目录名后添加序号后缀（如 _1、_2）避免冲突
