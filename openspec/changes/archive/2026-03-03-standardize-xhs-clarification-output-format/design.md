## 上下文

当前小红书 skill 的关键词澄清阶段使用简单文本列表展示关键词，格式不统一且可读性较差。目录创建时机在执行阶段，可能导致搜索阶段出错时重复创建目录的问题。

## 目标 / 非目标

**目标：**
- 统一关键词展示格式为标准 Markdown 表格
- 规范目录创建时机，确保澄清结束后立即创建唯一目录

**非目标：**
- 不修改爬虫脚本本身的行为
- 不改变搜索逻辑和帖子分配算法

## 决策

### 1. 表格格式设计

**决策**：使用 4 列表格 + 合并总计行

| 列名 | 说明 | 对齐方式 |
|-----|------|---------|
| 序号 | 1, 2, 3... | 居中 |
| 关键词 | 搜索关键词 | 左对齐 |
| 预计篇数 | 如 "10篇" | 居中 |
| 搜索理由 | 说明搜索目的 | 左对齐 |

**最后一行**：`| **总计** | — | **N篇** | 默认上限 N 篇 |`

**替代方案**：单独的总计行 vs 合并行
- 选择合并行：更紧凑，视觉上更统一

### 2. 目录格式设计

**决策**：`yyyyMMdd_hhMMss_搜索主旨`

- 时间戳在前：便于按时间排序
- 主旨在后：便于人工识别
- 使用下划线分隔：兼容所有文件系统

**创建时机**：用户确认关键词后，调用爬虫脚本之前

**替代方案**：主旨在前 vs 时间戳在前
- 选择时间戳在前：目录列表自动按时间排序

### 3. 优先级标记

两项需求均标记为 **[核心要求]**：
- 违反表格格式：用户无法清晰确认关键词
- 违反目录创建时机：可能产生重复目录

## 风险 / 权衡

| 风险 | 缓解措施 |
|-----|---------|
| 表格格式可能被 AI 忽略 | 在 spec.md 中明确标记为 [核心要求]，在 SKILL.md 中提供明确示例 |
| 时间戳精度导致目录冲突 | 使用秒级精度 (hhMMss)，单次会话中不太可能产生冲突 |
| 搜索主旨包含非法字符 | Agent 负责清理主旨中的文件系统非法字符 |
