## 为什么

当前小红书 skill 的关键词澄清阶段输出格式不统一，使用的是简单文本列表，不易阅读且不规范。同时，搜索目录的创建时机不明确，可能导致重复创建文件夹的问题。需要统一输出格式为 Markdown 表格，并强制规定目录创建时机。

## 变更内容

1. **规范关键词展示格式**：使用 Markdown 表格展示候选关键词+篇数+理由，表格行数动态可调（1-10个关键词+总计行）

2. **强制目录创建时机**：澄清阶段结束后立即创建搜索目录，格式为 `yyyyMMdd_hhMMss_搜索主旨`，后续所有产出文件必须在此目录下

## 功能 (Capabilities)

### 修改功能

- **xiaohongshu-search-clarification**:
  - 新增需求：必须使用 Markdown 表格格式展示关键词
  - 表格列名固定：序号 | 关键词 | 预计篇数 | 搜索理由
  - 最后一行合并列显示总计
  - 表格行数 = 关键词数量 K + 1（K范围1-10）
  - 标记为 [核心要求]

- **xiaohongshu-scraper**:
  - 新增需求：澄清阶段确认后必须立即创建搜索目录
  - 目录格式：`yyyyMMdd_hhMMss_搜索主旨`
  - 后续所有文件（raw.json, _index.md）必须在此目录下
  - 避免搜索阶段出现bug时重复创建文件夹
  - 标记为 [核心要求]

## 影响

- **SKILL.md**：更新关键词展示格式示例和目录创建说明
- **Agent 行为**：Agent 必须按照新格式输出表格，并在确认后立即创建目录
