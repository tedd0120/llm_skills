## 为什么

当前 `login_xhs.py` 的登录验证逻辑不可靠：只检查"是否有登录元素出现"，采用反向验证方式。这无法确保搜索功能真正可用，Cookie 可能存在但已过期，或页面加载异常但未触发登录弹窗。

用户需要的是正向验证：执行一次 test 搜索，验证搜索结果正常加载，才算登录成功。

## 变更内容

修改 `login_xhs.py` 的 `_is_logged_in()` 方法：

- **之前**：检查登录弹窗/二维码/登录按钮是否出现，都不出现则认为已登录
- **之后**：必须同时满足两个条件才算登录成功：
  1. 无登录弹窗/二维码/登录按钮出现
  2. 搜索结果区域有帖子卡片（`section.note-item`）存在
