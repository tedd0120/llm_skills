# 小红书登录验证修复增量规范

## 目的

修复 `login_xhs.py` 的登录验证逻辑，必须通过实际搜索验证登录状态。

---

## 新增需求

### 需求:登录验证必须通过搜索结果验证

`login_xhs.py` 的 `_is_logged_in()` 方法必须验证搜索结果区域有帖子卡片存在，禁止仅用"无登录元素"判断登录成功。

#### 场景:登录验证成功

- **当** `_is_logged_in()` 执行登录检查
- **那么** 必须访问搜索页面 `search_result?keyword=test`
- **并且** 必须检查登录弹窗、二维码、登录按钮是否出现
- **并且** 必须检查帖子卡片 `section.note-item` 数量是否大于 0
- **那么** 当且仅当无登录元素且帖子卡片数量 > 0 时返回 True

#### 场景:登录验证失败-有登录弹窗

- **当** `_is_logged_in()` 执行登录检查
- **并且** 检测到登录弹窗、二维码或登录按钮出现
- **那么** 必须返回 False

#### 场景:登录验证失败-无搜索结果

- **当** `_is_logged_in()` 执行登录检查
- **并且** 无登录元素出现
- **并且** 帖子卡片数量为 0
- **那么** 必须返回 False

#### 场景:禁止仅用文件存在判断

- **当** `_is_logged_in()` 执行登录检查
- **禁止** 仅通过 auth 文件是否存在判断登录状态
- **禁止** 仅通过是否能打开小红书页面判断登录状态
- **必须** 通过实际搜索结果验证登录状态

---

## 移除需求

### 需求:仅检查登录元素的验证方式

**Reason**: 反向验证不可靠，无法确保搜索功能真正可用。

**Migration**: 使用新的搜索结果验证方式。
