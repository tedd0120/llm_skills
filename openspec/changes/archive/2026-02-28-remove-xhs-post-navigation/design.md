## 上下文

当前 xiaohongshu-scraper 的数据格式包含 9 个字段，其中 `url` 字段用于生成报告中的"帖子导航目录"。经过实际使用发现：
- 用户很少点击原始帖子链接
- 报告的核心价值在于汇总分析，而非导航到原始帖子
- URL 字段占用约 10-15% 的 JSON 数据空间

## 目标 / 非目标

**目标：**
- 简化 JSON 数据结构（从 9 字段减少到 8 字段）
- 删除报告中的"帖子导航目录"板块（从 9 个板块减少到 8 个）
- 调整"可追溯性"要求以适应新的数据格式

**非目标：**
- 不改变抓取脚本的其他功能
- 不影响报告的其他分析板块
- 不改变数据提取逻辑

## 决策

### 1. JSON 字段删除策略

**决策：** 完全删除 `url` 字段，不保留替代字段

**理由：**
- 帖子标题 + 作者 + 日期已足够标识唯一帖子
- 不需要使用帖子 ID（`675fded1000000000b014dc0`）作为替代
- 简单直接，无冗余数据

**替代方案：** 保留 `post_id` 字段 - 因用途有限被否决

### 2. 报告板块调整

**决策：** 删除"帖子导航目录"，其他板块重新编号（2→2, 3→3, ...）

**理由：**
- 保持板块顺序的逻辑性
- 避免"板块 1、板块 3-9"的混乱编号

**替代方案：** 保留编号空缺 - 因不美观被否决

### 3. 可追溯性要求调整

**决策：** 从"必须包含链接"改为"必须包含标题、作者、日期"

**理由：**
- 保留信息来源标识能力
- 引用时可以格式化为：`(作者: xxx, 日期: xxx)`

**替代方案：** 完全删除可追溯性要求 - 因影响报告可信度被否决

### 4. 应用范围

**必须修改的文件：**
1. `.agent/skills/xiaohongshu-scraper/scripts/fetch_xhs.py`
   - 第 248 行：删除 `"url": url,`

2. `.agent/skills/xiaohongshu-scraper/SKILL.md`
   - 第 97 行：调整可追溯性要求描述
   - 第 139、173 行：删除"帖子导航目录"提及
   - 第 171-180 行：更新必需板块列表（9→8）
   - 第 197-200 行：删除模板中的目录示例

3. `.agent/workflows/xiaohongshu-scraper.md`
   - 更新板块描述（与 SKILL.md 保持一致）

## 风险 / 权衡

**风险：** 删除 URL 后无法验证原始帖子上下文
**缓解措施：** 通过帖子标题、作者、日期和评论引用来提供足够的上下文信息

**风险：** 已有的 raw.json 文件仍包含 URL 字段
**缓解措施：** 不影响现有数据，新抓取的数据不包含 URL

**风险：** 报告引用格式需要调整
**缓解措施：** 在模板中提供统一的引用格式示例
