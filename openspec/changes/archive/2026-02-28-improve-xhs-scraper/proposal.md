## 为什么

当前 xiaohongshu-scraper skill 的澄清阶段交互不够透明，用户不清楚每个关键词会抓取多少帖子；报告输出中的引用内容与 AI 总结内容格式混同，难以区分。

**问题 1**：关键词衍生阶段缺少帖子上限和分配信息，用户不知道每个关键词会抓取多少篇帖子，也无法方便地修改上限。

**问题 2**：报告中引用的帖子标题、正文、评论与 AI 分析总结使用相同格式，阅读时难以快速区分原始引用内容和分析结论。

## 变更内容

1. **关键词衍生展示增强**
   - 每个关键词显示预计抓取篇数
   - 显示总帖子上限（默认 30 篇，可修改）
   - 关键词间均匀分配帖子数，无法均分的余数归入第一关键词

2. **用户交互选项扩展**
   - 修改单个关键词
   - 重新生成关键词
   - 提出关键词建议
   - 修改搜索帖子上限

3. **报告引用格式优化**
   - 所有引用内容（帖子标题、正文片段、评论）使用斜体标注
   - AI 总结分析内容使用正常字体
   - 在 Markdown 中使用 `*文本*` 实现斜体效果

## 功能 (Capabilities)

### 新增功能
- `xhs-keyword-phase-ui`: 关键词衍生阶段的用户交互增强

### 修改功能
- `xiaohongshu-content-summarization`: 报告输出格式变更，引用内容使用斜体样式

## 影响

- **Skill 文档**: `.claude/skills/xiaohongshu-scraper/skill.md` 需要更新
- **报告模板**: 报告生成的 Markdown 模板需要更新引用格式
- **用户交互流程**: 澄清阶段的提示语和反馈选项需要扩展
- **无代码变更**: 纯文档和流程描述变更，不涉及 Python 脚本修改
