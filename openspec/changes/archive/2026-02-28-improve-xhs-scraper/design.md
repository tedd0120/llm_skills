## 决策

### 关键词帖数分配算法

```
分配算法:
N = 总帖子上限（默认 30）
K = 关键词数量（固定 3）

每个关键词基础分配 = N // K
余数 = N % K

第一关键词分配 = 基础分配 + 余数
其他关键词分配 = 基础分配
```

**示例**:
- N=30, K=3: [10, 10, 10]
- N=50, K=3: [18, 16, 16]
- N=40, K=3: [14, 13, 13]

### Markdown 引用格式规范

| 内容类型 | Markdown 语法 | 示例 |
|---------|--------------|------|
| 帖子标题 | `*《标题》*` | *《想买索尼电视，我劝你冷静》* |
| 用户评论 | `_"内容"_` | _"三年质保刚过就没声音"_ |
| AI 总结 | 正常字体 | 用户普遍反馈索尼过保即坏 |

**为什么使用斜体而非代码块？**
- 斜体在渲染后更自然阅读
- 代码块 (` `"content" `) 在移动端显示效果不佳
- 斜体是标准的 Markdown 引用格式

## 风险 / 权衡

**风险**: 斜体样式在某些 Markdown 渲染器中可能不明显
- **缓解**: 在 skill 文档中明确说明格式规范，AI agent 遵循执行

**风险**: 用户可能不理解帖子上限的含义
- **缓解**: 在提示语中说明"每个关键词分配的预计抓取篇数"

**权衡**: 固定 3 个关键词的限制
- 用户可能想要更多关键词，但保持 3 个可以简化分配逻辑
- 如需更多关键词，用户可以通过"提出关键词建议"方式实现
