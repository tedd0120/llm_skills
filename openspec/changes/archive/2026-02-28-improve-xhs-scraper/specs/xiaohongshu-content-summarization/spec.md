## 修改需求

### 需求:报告引用格式区分
AI agent 生成的报告中，所有引用内容（帖子标题、正文片段、评论）必须使用斜体样式与 AI 总结内容区分。

#### 场景:帖子标题引用格式
- **当** 报告中引用帖子标题
- **那么** 必须使用 Markdown 斜体语法包裹（格式：`*《帖子标题》*`）

#### 场景:帖子正文引用格式
- **当** 报告中引用帖子正文片段
- **那么** 必须使用 Markdown 斜体语法包裹引用内容

#### 场景:用户评论引用格式
- **当** 报告中引用用户评论
- **那么** 必须使用 Markdown 斜体语法包裹评论内容（格式：`_"用户评论内容"_`）

#### 场景:来源信息引用格式
- **当** 报告中标注引用来源
- **那么** 来源帖子标题必须使用斜体（格式：`来源：*《帖子标题》*`）

#### 场景:AI 总结内容格式
- **当** 报告中展示 AI 分析总结
- **那么** 必须使用正常字体（非斜体）
- **禁止** 对 AI 总结内容使用斜体样式

### 需求:声量分析引用格式
品牌/主题声量分析板块中的引用内容必须遵循斜体格式规范。

#### 场景:品牌提及来源
- **当** 声量分析中引用某品牌讨论的来源
- **那么** 来源帖子标题必须使用斜体

#### 场景:用户反馈引用
- **当** 声量分析中引用用户反馈
- **那么** 评论内容必须使用斜体

### 需求:避雷指南引用格式
避雷指南板块中的高频问题和用户反馈必须遵循斜体格式规范。

#### 场景:高频问题描述
- **当** 避雷指南中引用用户报告的问题
- **那么** 问题描述可使用正常字体（AI 总结）
- **但** 用户原话引用必须使用斜体

#### 场景:用户反馈引用
- **当** 避雷指南中引用用户反馈
- **那么** 用户反馈内容必须使用斜体（格式：`_"用户反馈内容"_`）

### 需求:关键洞察引用格式
关键洞察板块中追溯来源的引用必须遵循斜体格式规范。

#### 场景:洞察来源追溯
- **当** 关键洞察需要追溯到具体帖子或评论
- **那么** 来源帖子标题必须使用斜体（格式：`> 追溯来源：*《帖子标题》*`）
