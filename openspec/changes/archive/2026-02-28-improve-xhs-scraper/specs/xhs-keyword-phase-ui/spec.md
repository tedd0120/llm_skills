## 新增需求

### 需求:关键词帖数分配说明
AI agent 必须在展示衍生关键词时，显示每个关键词预计抓取的帖子数量，以及总帖子上限。

#### 场景:默认分配展示
- **当** agent 向用户展示 3 个衍生关键词
- **那么** 必须在每个关键词后显示预计抓取篇数（格式：`关键词 (N篇)`）
- **并且** 必须在关键词列表后显示总计信息（格式：`总计: N 篇 (默认上限)`）

#### 场景:均匀分配计算
- **当** 总帖子上限为 N，关键词数量为 K
- **那么** 每个关键词分配 `N // K` 篇
- **并且** 余数 `N % K` 必须分配给第一关键词

#### 场景:上限 30 篇的默认分配
- **当** 默认上限 30 篇，3 个关键词
- **那么** 关键词1 分配 10 篇，关键词2 分配 10 篇，关键词3 分配 10 篇

#### 场景:上限 50 篇的分配
- **当** 上限 50 篇，3 个关键词
- **那么** 关键词1 分配 18 篇，关键词2 分配 16 篇，关键词3 分配 16 篇

### 需求:关键词调整选项扩展
agent 必须向用户提供明确的调整选项，包括修改单个关键词、重新生成、提出建议和修改上限。

#### 场景:展示调整选项
- **当** agent 向用户展示衍生关键词
- **那么** 必须同时展示以下调整选项列表：
  - 修改单个关键词
  - 重新生成关键词
  - 提出关键词建议
  - 修改搜索帖子上限
  - 确认开始爬取

#### 场景:用户修改单个关键词
- **当** 用户请求修改某个关键词（如 "把第1个改成国产电视推荐"）
- **那么** agent 必须仅更新指定关键词
- **并且** 重新展示完整的 3 个关键词及分配信息
- **并且** 询问用户是否确认

#### 场景:用户提出关键词建议
- **当** 用户提出额外的关键词（如 "加上'小米电视'这个品牌"）
- **那么** agent 必须将用户建议融入衍生关键词列表
- **或者** 询问用户希望替换哪个现有关键词

#### 场景:用户修改帖子上限
- **当** 用户请求修改帖子上限（如 "增加到50篇"）
- **那么** agent 必须重新计算每个关键词的分配数量
- **并且** 重新展示更新后的关键词及分配信息
- **并且** 第一关键词获得余数分配

### 需求:关键词理由说明
agent 必须为每个衍生关键词提供简短的理由说明。

#### 场景:展示关键词理由
- **当** agent 展示衍生关键词
- **那么** 每个关键词必须附带理由说明（格式：`关键词 - 理由`）
- **并且** 理由必须说明该关键词的搜索目的（如：获取主流品牌推荐 / 对比具体型号 / 了解避坑信息）

#### 场景:理由模板
- **直译/直接搜索词**: 理由为 "获取[主题]的整体推荐趋势"
- **对比/评价词**: 理由为 "获取[主题]的对比评价和用户体验"
- **避雷/负面词**: 理由为 "了解[主题]的常见问题和避坑指南"
