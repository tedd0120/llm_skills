## 修改需求

### 需求:可变关键词数量
AI agent 必须支持用户自由选择关键词数量，而非强制使用恰好 3 个。

#### 场景:默认3个关键词
- **当** 用户未指定关键词数量
- **那么** agent 必须默认衍生 3 个关键词

#### 场景:用户指定5个关键词
- **当** 用户明确请求 5 个关键词（如："增加到5个关键词"）
- **那么** agent 必须衍生 5 个关键词
- **并且** 重新计算帖数分配

#### 场景:用户指定关键词数量范围
- **关键词数量限制**: 1 到 10 个关键词
- **当** 用户请求超过 10 个关键词
- **那么** agent 必须提示最多支持 10 个，并建议用户调整

### 需求:帖数分配支持可变数量
帖数分配算法必须支持任意数量的关键词 K。

#### 场景:通用分配算法
- **当** 总帖子上限为 N，关键词数量为 K（1 ≤ K ≤ 10）
- **那么** 每个关键词分配 `N // K` 篇
- **并且** 余数 `N % K` 必须分配给第一关键词

#### 场景:5个关键词50篇的分配
- **当** 上限 50 篇，5 个关键词
- **那么** 分配为 [10篇, 10篇, 10篇, 10篇, 10篇]

#### 场景:4个关键词50篇的分配
- **当** 上限 50 篇，4 个关键词
- **那么** 分配为 [14篇, 12篇, 12篇, 12篇]（余数2归第一关键词）

### 需求:关键词数量调整选项
agent 必须在调整选项列表中添加"修改关键词数量"选项。

#### 场景:展示完整调整选项
- **当** agent 向用户展示衍生关键词
- **那么** 必须同时展示以下调整选项列表：
  - 修改单个关键词
  - 修改关键词数量 (如: "增加到5个" 或 "减少到2个")
  - 重新生成关键词
  - 提出关键词建议
  - 修改搜索帖子上限
  - 确认开始爬取
