## 为什么

当前 xiaohongshu-scraper skill 强制要求使用恰好 3 个关键词，这限制了用户的灵活性。用户可能有更具体或更广泛的需求，希望使用更多或更少的关键词来搜索。

## 变更内容

- **修改**：将关键词数量从"强制 3 个"改为"默认 3 个"
- **新增**：支持用户自由调整关键词数量（1-10个）
- **修改**：帖数分配算法支持可变的关键词数量

## 功能 (Capabilities)

### 修改功能
- `xhs-keyword-phase-ui`: 关键词衍生阶段 UI 增强，支持可变关键词数量

## 影响

- **Skill 文档**: `.claude/skills/xiaohongshu-scraper/skill.md` 需要更新
- **用户交互**: 澄清阶段需要支持关键词数量的调整
- **帖数分配算法**: 分配公式需要支持可变数量 K（而非固定 3）
- **无代码变更**: 纯文档和流程描述变更
