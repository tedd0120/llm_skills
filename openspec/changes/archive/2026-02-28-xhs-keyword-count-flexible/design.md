## 上下文

当前 skill.md 强制要求"必须衍生恰好 3 个搜索关键词"，这是 v3.0 版本引入的限制。用户反馈希望有更灵活的关键词数量选择。

**约束**:
- 纯文档变更，不修改 Python 脚本
- 必须向后兼容（默认仍是 3 个）
- 关键词数量需要合理上限（1-10）

## 目标 / 非目标

**目标：**
1. 将关键词数量从"强制 3 个"改为"默认 3 个，可调整
2. 更新帖数分配算法说明，支持通用 K（而非固定 3）
3. 在调整选项中添加"修改关键词数量"

**非目标：**
- 不修改脚本代码
- 不改变报告结构

## 决策

### 关键词数量上限

选择 **1-10** 个关键词作为合理范围：
- 下限 1：用户可能只想用一个非常精确的关键词
- 上限 10：避免用户请求过多导致抓取时间过长（风控风险）

### 通用分配算法

```
分配算法（支持任意 K）：
N = 总帖子上限（默认 30）
K = 关键词数量（默认 3，可调整 1-10）

每个关键词基础分配 = N // K
余数 = N % K

第一关键词分配 = 基础分配 + 余数
其他关键词分配 = 基础分配
```

**示例**:
- K=1, N=30: [30]
- K=3, N=50: [18, 16, 16]
- K=4, N=50: [14, 12, 12, 12]
- K=5, N=50: [10, 10, 10, 10, 10]
- K=5, N=30: [6, 6, 6, 6, 6]

### 用户交互模板更新

```
你可以：
• 修改关键词数量 (如: "增加到5个" 或 "减少到2个")
• 修改单个关键词
• 重新生成关键词
• 提出关键词建议
• 修改搜索上限
• 确认开始爬取
```
