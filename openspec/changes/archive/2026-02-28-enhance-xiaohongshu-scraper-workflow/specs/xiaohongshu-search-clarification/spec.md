## 新增需求

### 需求:用户输入必须作为整体处理
系统必须将用户输入的搜索建议视为一个整体文本，忽略内部所有分隔符（逗号、空格、分号等），AI 需要理解文本的主旨而非机械分割。

#### 场景:包含逗号的输入
- **当** 用户输入 "广州装修公司, 对比, 避坑"
- **那么** 系统必须将整个输入理解为"广州装修公司相关"的建议
- **那么** AI 必须基于整体主旨衍生关键词

#### 场景:自然语言输入
- **当** 用户输入 "我想了解新手买电脑需要注意什么"
- **那么** 系统必须识别主题为"新手买电脑"
- **那么** AI 必须基于"新手"和"买电脑"两个要素衍生关键词

### 需求:必须衍生恰好三个搜索词
系统必须基于用户建议的主旨衍生恰好 3 个搜索关键词，不得多于 3 个，不得少于 3 个。

#### 场景:标准衍生
- **当** 用户输入 "装修公司"
- **那么** 系统必须衍生恰好 3 个关键词
- **例如** ["装修公司推荐", "装修避坑", "装修预算"]

#### 场景:具体输入的衍生
- **当** 用户输入 "广州天河区哪家装修公司好"
- **那么** 系统必须衍生 3 个有关联但角度不同的关键词
- **例如** ["广州天河装修公司", "装修公司评价", "装修公司对比"]

### 需求:衍生关键词必须符合用户主旨
系统衍生的所有搜索词必须符合用户输入建议的核心意图和主旨，不得偏离用户意图。

#### 场景:地域保留
- **当** 用户输入包含地域信息 "广州装修"
- **那么** 衍生的关键词必须保留地域要素
- **反例** 不能衍生为"装修公司推荐"（丢失了"广州"）

#### 场景:场景保留
- **当** 用户输入包含场景信息 "新手买电脑"
- **那么** 衍生的关键词必须考虑"新手"场景
- **反例** 不能衍生为"高端游戏电脑推荐"（偏离了"新手"场景）

### 需求:必须执行澄清阶段
系统必须在首次调用命令时执行澄清阶段，与用户进行多轮问答确认搜索关键词，确认前不得执行爬虫操作。

#### 场景:首次调用流程
- **当** 用户首次调用 xiaohongshu-scraper 命令
- **那么** 系统必须解析用户输入
- **那么** 系统必须衍生 3 个关键词
- **那么** 系统必须以格式C展示给用户
- **那么** 系统必须等待用户反馈
- **禁止** 在确认前调用 fetch_xhs.py

#### 场景:多轮澄清
- **当** 用户要求修改关键词
- **那么** 系统必须根据用户反馈更新关键词
- **那么** 系统必须重新展示更新后的关键词
- **那么** 系统必须再次请求确认

### 需求:必须使用格式C展示关键词
系统必须使用格式C向用户展示衍生关键词，包含关键词列表和明确的回复选项说明。

#### 场景:格式C展示
- **当** 系统向用户展示衍生关键词
- **那么** 必须使用以下格式：
  ```
  我基于你的建议衍生了以下 3 个搜索词：

  1. [关键词一]
  2. [关键词二]
  3. [关键词三]

  回复：
  • "确认" - 开始爬取
  • 其他内容 - 继续调整
  ```

### 需求:必须主动请求确认
系统必须在每次展示关键词后主动询问用户是否确认开始爬取，不得依赖模糊信号判断。

#### 场景:明确确认请求
- **当** 系统展示衍生关键词后
- **那么** 必须明确询问 "确认开始爬取吗？"
- **禁止** 仅依赖用户说"行"、"可以"等模糊表达

#### 场景:用户明确确认
- **当** 用户回复 "确认"、"开始"、"好的"、"就这样"
- **那么** 系统必须进入执行阶段
- **那么** 系统必须调用 fetch_xhs.py

### 需求:确认后必须执行爬虫
系统必须在用户确认关键词后执行爬虫操作，调用 fetch_xhs.py 传入确认的 3 个关键词。

#### 场景:执行爬虫
- **当** 用户确认搜索关键词
- **那么** 系统必须调用 fetch_xhs.py
- **那么** 系统必须传入确认的 3 个关键词作为 --keywords 参数
- **那么** 系统必须继续执行报告生成流程

### 需求:SKILL.md必须明确标记阶段
SKILL.md 文档必须使用明确的章节标记区分"澄清阶段"和"执行阶段"，确保 AI 能够识别当前所处阶段。

#### 场景:阶段标记
- **当** AI 阅读 SKILL.md
- **那么** 必须能够清晰识别"阶段一：澄清阶段"和"阶段二：执行阶段"
- **那么** 必须在澄清阶段完成后才进入执行阶段

### 需求:建议多角度衍生策略
系统建议（但不强制）从不同角度衍生关键词，包括：直译/直接搜索、对比/评价、避雷/负面等角度。

#### 场景:产品推荐类输入
- **当** 用户输入关于产品推荐的搜索建议
- **那么** 建议从以下角度衍生：
  - 直译角度：直接相关的搜索词
  - 对比角度：获取对比/评价信息
  - 避雷角度：获取风险/问题信息

#### 场景:技术问题类输入
- **当** 用户输入关于技术问题的搜索建议
- **那么** 建议从以下角度衍生：
  - 问题分类：不同类型的相关问题
  - 解决方案：不同解决途径
  - 同义词：相似表述的关键词
