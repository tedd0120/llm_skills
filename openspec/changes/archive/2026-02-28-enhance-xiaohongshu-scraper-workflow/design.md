**约束**：
- 不能修改 fetch_xhs.py（仅通过 SKILL.md 调用）
- 必须保持与现有报告生成流程的兼容性

## 目标 / 非目标

**目标：**
1. 将关键词衍生数量固定为恰好 3 个
2. 增加澄清阶段，确保搜索词符合用户意图
3. 在 SKILL.md 中明确区分"澄清阶段"和"执行阶段"

**非目标：**
- 不修改 fetch_xhs.py 脚本
- 不改变报告生成逻辑

## 决策

### 1. 阶段划分

**决策**：在 SKILL.md 中明确标记两个阶段

```
阶段一：澄清阶段（必须执行）
阶段二：执行阶段（确认后进入）
```

**理由**：
- AI 需要清晰的指令来知道自己在哪个阶段
- 防止跳过澄清直接执行爬虫

**替代方案及不选原因**：
- 依赖对话上下文 → AI 可能"忘记"状态
- 询问用户确认每个过渡 → 可能过于啰嗦

### 2. 确认机制

**决策**：使用格式C展示 + 主动询问确认

```
我基于你的建议衍生了以下 3 个搜索词：

1. [关键词一]
2. [关键词二]
3. [关键词三]

回复：
• "确认" - 开始爬取
• 其他内容 - 继续调整
```

**理由**：
- 用户明确知道如何回复
- 降低模糊确认（如"行"、"可以"）带来的误判

### 3. 用户输入处理

**决策**：将用户输入视为整体，忽略内部所有分隔符

**理由**：
- 用户可能使用自然语言表达，逗号或空格不一定代表分隔意图
- AI 需要理解主旨，而非机械分割

**示例**：
- 输入："广州装修公司, 避坑, 推荐" → 理解为"广州装修公司避坑推荐"这一整体建议
- AI 衍生：["广州装修公司", "装修避坑", "装修公司推荐"]

### 4. 关键词衍生策略

**决策**：在 SKILL.md 中建议多角度衍生，但不强制具体角度

```
建议从不同角度衍生：
• 直译/直接搜索（最接近原意）
• 对比/评价（获取他人经验）
• 避雷/负面（获取风险信息）
```

**理由**：
- 保持灵活性，让 AI 根据具体主题选择最佳角度
- 不同主题（如产品推荐 vs 技术问题）需要不同的衍生策略

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| 用户不愿参与澄清，只想快速结果 | 在展示格式中明确告知回复"确认"即可开始 |
| AI 衍生的关键词不符合用户主旨 | 明确要求"所有衍生词必须符合用户建议的主旨"，并在澄清阶段允许用户调整 |
| 澄清轮次过多导致时间浪费 | 每次展示后主动询问"确认开始爬取吗？"，给用户快速退出的路径 |
| 用户输入包含多个不相关主题 | 澄清阶段 AI 识别并询问用户"我注意到多个主题，请确认你最关注哪一个" |
