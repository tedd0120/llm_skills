## 上下文

### 当前状态
xiaohongshu-scraper skill 的工作流程（在 SKILL.md 中定义）包含 5 个步骤：
1. 扩充搜索词
2. 执行抓取脚本 → 输出 JSON
3. 创建结果目录
4. **数据总结写入** → 为每篇帖子生成独立 MD（20 篇 = 20 个文件）
5. **生成索引** → 读取所有单篇 MD，生成 _index.md

### 问题
- Step 4 生成的 20 个单篇 MD 文件使用率极低（用户反馈基本不看）
- AI 需要大量 token 生成这些文件，然后再次消耗 token 读取它们
- 最终价值集中在 _index.md 汇总报告中

### 约束
- 不能修改爬虫脚本（fetch_xhs.py, selectors.py）
- 需要保持与现有工作流的兼容性（用户仍调用 `/xiaohongshu-scraper` 命令）
- 增强版报告必须提供足够深度，弥补失去单篇 MD 的损失

---

## 目标 / 非目标

**目标：**
1. 简化工作流程：删除低价值的单篇 MD 生成环节
2. 提升报告质量：将 _index.md 从导航列表升级为深度分析报告
3. 提高效率：减少 token 消耗和执行时间
4. 保持可追溯性：通过原小红书链接引用具体帖子

**非目标：**
- 不修改爬虫功能（JSON 输出格式不变）
- 不改变命令调用方式（用户仍使用 `/xiaohongshu-scraper`）
- 不支持自定义报告模板（固定增强版模板）

---

## 决策

### 决策 1：直接从 JSON 生成增强版报告

**选择**：删除 Step 4（单篇 MD 生成），改为 Step 4 直接读取 JSON 生成 _index.md

**理由**：
- 消除中间环节，减少 40%+ 的 token 消耗（不再生成和读取单篇 MD）
- AI 可以一次性读取所有数据，做跨帖分析更高效
- 用户直接获得高质量报告，无需翻阅 20 个文件

**替代方案**：
- 方案 A：保留单篇 MD，但只生成 Top 5 重点帖子
  - ❌ 仍需要选择逻辑，且部分用户可能想要查看其他帖子
- 方案 B：保留单篇 MD，但简化内容
  - ❌ 仍然消耗 token，简化后价值更低

### 决策 2：增强版报告模板结构

**选择**：固定 8 个分析板块

1. **基础信息板块**（必选）
   - 搜索任务摘要
   - 帖子导航目录（链接到原小红书）

2. **深度分析板块**（全部包含）
   - 品牌/主题声量分析
   - 多维对比矩阵
   - 评论区高频共识（Top 5 优点 / Top 5 痛点）
   - 场景化决策建议
   - 价格/规格参考
   - 避雷指南
   - 关键洞察

**理由**：
- 基于实际测试经验（20 篇"扫拖机器人推荐"帖子分析）
- 这些板块覆盖了用户决策的核心需求
- 数据可从 JSON 中直接提取，无需额外输入

**替代方案**：
- 方案 A：根据主题动态选择板块
  - ❌ 增加复杂度，且测试中所有主题都能从这些板块受益
- 方案 B：让用户选择需要的板块
  - ❌ 增加交互成本，与"快速获取报告"目标冲突

### 决策 3：数据提取策略

**选择**：AI 直接从 JSON 提取结构化数据，而不是依赖单篇 MD 的结构化字段

**理由**：
- JSON 已包含所有需要的数据（标题、内容、评论）
- 现代 AI 模型可以高效处理 JSON（50K tokens ≈ 20 篇帖子）
- 避免在 JSON 和 MD 之间维护两套数据结构

**提取逻辑**：
- **品牌声量**：统计所有帖子的 `title` 和 `content` 中的品牌提及
- **对比矩阵**：分析评论区的"vs"、"对比"、"区别"等关键词
- **高频共识**：统计所有评论，提取出现频率最高的观点
- **价格信息**：使用正则匹配评论中的价格模式（如"3200元"、"2999"）

### 决策 4：可追溯性设计

**选择**：在 _index.md 中保留原文小红书链接 + 关键引用

**格式示例**：
```markdown
### 核心痛点
1. **刮花地板** - 提及 800 次
   - 典型反馈: "地板全部刮花！几十万装修地板刮得像个狗屎"
   - 来源: [@用户名](https://www.xiaohongshu.com/...)
```

**理由**：
- 用户可以点击链接查看原始帖子
- 关键引用提供了足够的上下文，无需依赖单篇 MD

**替代方案**：
- 方案 A：在报告中嵌入完整原文
  - ❌ 报告过长，失去"汇总"价值
- 方案 B：不提供引用，只给链接
  - ❌ 用户需要跳转才能理解结论依据

## 风险 / 权衡

### 风险 1：AI 处理大量数据可能出错

**风险**：一次性读取 20 篇帖子 + 6000+ 条评论（约 70K tokens），AI 可能遗漏或错误统计

**缓解措施**：
- 在模板中明确要求"数据不足时标注'样本有限'"
- 建议用户"如需精确数据，可减少抓取数量（如 10 篇）"
- 保持 JSON 文件，用户可重新生成报告

### 风险 2：失去单篇 MD 的存档价值

**风险**：某些用户可能想要保存单个帖子的详细分析

**缓解措施**：
- JSON 文件默认保留（包含原始数据）
- 用户可以单独为某篇帖子生成 MD（手动调用 AI）
- _index.md 中的引用已包含关键信息

### 风险 3：不同主题的报告质量可能不一致

**风险**：某些主题（如"教程类"）不适合产品对比矩阵

**缓解措施**：
- 模板中的"品牌/产品"应理解为广义的"主题/类别"
- 如数据不足，AI 可标注"此板块不适用于当前主题"
- 保留基础信息板块，确保报告始终有价值
