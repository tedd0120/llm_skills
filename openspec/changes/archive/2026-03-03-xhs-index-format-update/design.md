## 上下文

小红书抓取 skill 当前生成的 `_index.md` 报告使用列表形式展示搜索概览：

```markdown
# 搜索任务: [主旨]

- **执行时间**: [YYYY-MM-DD HH:MM]
- **使用搜索词**: keyword1, keyword2
- **共抓取帖子**: [N] 篇
```

**约束条件**：
- Markdown 标准不直接支持表头加粗语法（`**` 在表头中被视为普通文本）
- Agent 在生成报告时需要从 JSON 数据中计算评论总数
- 修改需要保持向后兼容，不破坏现有功能

## 目标 / 非目标

**目标：**
- 将搜索概览改为表格格式，提升可读性
- 实现表头加粗效果
- 新增"涉及评论"统计字段
- 更新 SKILL.md 中的报告模板

**非目标：**
- 不修改报告的其他部分（声量分析、评论区共识等）
- 不改变 JSON 数据结构
- 不修改爬虫脚本行为

## 决策

### 1. 表头加粗方案：使用 HTML `<b>` 标签

**选择**：在表头中使用 `<b>字段</b>` 和 `<b>值</b>` 格式

**理由**：
- Markdown 不支持原生表头加粗
- HTML `<b>` 标签在大多数 Markdown 渲染器中都能正确显示
- 比让整个表格内容都加粗更简洁

**替代方案及拒绝理由**：
- 全表格加粗：视觉效果过重，不适合大量文本展示

### 2. 评论数统计方式：运行时聚合

**选择**：在生成报告时遍历所有帖子的 `comments_count` 字段求和

**理由**：
- 不需要修改爬虫脚本的数据结构
- 灵活适应不同数量的帖子

**实现逻辑**（伪代码）：
```
totalComments = posts.reduce((sum, post) => sum + post.comments_count, 0)
```

### 3. 多关键词展示：逗号分隔

**选择**：使用逗号加空格分隔多个搜索词

**理由**：
- 简洁明了，节省空间
- 与澄清阶段的输出格式保持一致

## 风险 / 权衡

| 风险 | 缓解措施 |
|-----|---------|
| HTML 标签在某些 Markdown 渲染器中可能不生效 | 使用标准 `<b>` 标签，广泛支持 |
| 评论数统计可能遗漏嵌套评论 | 文档明确说明统计的是 `comments_count` 字段 |
| 用户可能不适应新格式 | 新格式更具结构性，易于理解 |

## 迁移计划

1. **更新 SKILL.md 模板**：将"增强版汇总报告模板"部分的头部改为表格格式
2. **更新 Agent 执行步骤**：在生成报告时计算并添加评论数
3. **无数据迁移**：此变更仅影响新生成的报告，现有报告无需处理

