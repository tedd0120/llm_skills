## 为什么

当前小红书抓取 skill 生成的 `_index.md` 报告头部使用列表形式展示搜索概览信息，存在以下问题：
- 格式不够结构化，阅读体验欠佳
- 缺少"涉及评论"统计字段，无法直观了解数据规模
- 多关键词展示时可能显得拥挤

现在是优化报告格式的合适时机，以提升输出质量和可读性。

## 变更内容

### 新增
- 报告头部"搜索概览"改为 Markdown 表格形式
- 表格列使用 HTML `<b>` 标签实现表头加粗
- 新增"涉及评论"字段，统计所有帖子的评论总数

### 修改
- `SKILL.md` 中的报告模板更新为新的表格格式
- 报告生成逻辑增加评论数统计

## 功能 (Capabilities)

### 新增功能
- `xhs-report-table-format`: 定义小红书报告头部的表格格式规范，包括字段定义、表格样式、加粗规则和评论数统计方式

### 修改功能
- 无（这是报告输出格式的调整，不影响 skill 的核心行为规范）

## 影响

- **受影响的文件**:
  - `.claude/skills/xiaohongshu-scraper/SKILL.md` - 报告模板部分
  - 未来生成的 `data/xiaohongshu/*/index.md` 报告文件

- **无破坏性变更**: 新格式向后兼容，仅改变展示方式
