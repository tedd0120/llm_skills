## 新增需求

### 需求:报告头部必须使用表格格式
小红书抓取 skill 生成的 `_index.md` 报告头部搜索概览部分必须使用 Markdown 表格形式展示，不得使用列表或其他格式。

#### 场景:生成报告头部
- **当** Agent 生成小红书报告的头部概览部分
- **那么** 必须使用 Markdown 表格格式，包含"字段"和"值"两列

### 需求:表格表头必须加粗
报告概览表格的表头必须使用 HTML `<b>` 标签实现加粗效果。

#### 场景:表头渲染
- **当** 渲染报告概览表格
- **那么** 表头单元格"字段"和"值"必须使用 `<b>字段</b>` 和 `<b>值</b>` 格式

### 需求:概览表格必须包含五个必需字段
报告概览表格必须包含以下五个字段：搜索任务、执行时间、使用搜索词、共抓取帖子、涉及评论。

#### 场景:完整字段展示
- **当** 生成报告概览表格
- **那么** 表格必须包含恰好 5 行数据行，分别对应：搜索任务、执行时间、使用搜索词、共抓取帖子、涉及评论

### 需求:搜索词必须全部展示
当使用多个搜索关键词时，"使用搜索词"字段必须展示所有关键词，使用逗号分隔。

#### 场景:多关键词展示
- **当** 搜索包含多个关键词（如 "咖啡厅推荐, 独立咖啡馆, 社区咖啡"）
- **那么** "使用搜索词"字段必须显示为 "咖啡厅推荐, 独立咖啡馆, 社区咖啡"

### 需求:涉及评论数必须统计所有帖子
"涉及评论"字段必须显示所有抓取帖子的评论总数，即每篇帖子的 `comments_count` 之和。

#### 场景:评论数统计
- **当** 抓取了 3 篇帖子，评论数分别为 10、20、30
- **那么** "涉及评论"字段必须显示 "60 条"

### 需求:帖子和评论数必须带单位
"共抓取帖子"和"涉及评论"字段的值必须包含单位（"篇"或"条"）。

#### 场景:单位格式
- **当** 显示帖子和评论数量
- **那么** 格式必须为 "N 篇" 和 "M 条"

### 需求:表格必须在报告最顶部
搜索概览表格必须位于 `_index.md` 报告文件的最顶部，在所有其他内容之前。

#### 场景:表格位置
- **当** 生成完整的报告文件
- **那么** 搜索概览表格必须是文件的第一个内容块（仅在标题之后）

## 修改需求

无。此变更为新增格式规范，不修改现有行为需求。

## 移除需求

### 需求:列表形式的报告头部概览
**原因**: 被表格格式取代，提供更好的可读性和结构化展示

**迁移方案**: 将原有的列表格式替换为表格格式，保留相同的信息内容
