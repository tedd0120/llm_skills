# 小红书爬虫执行规范（增量）

## 目的

为小红书爬虫 skill 新增发散搜索模式的执行规范。

---

## 新增需求

### 需求: 发散模式执行循环
Agent 在发散模式下必须执行迭代式搜索循环：每轮搜索 → 分析结果 → 决定下一个关键词 → 重复。

#### 场景: 初始关键词生成
- **当** 进入发散模式执行阶段（第 1 轮）
- **那么** Agent 必须基于用户原始主题生成 1 个初始搜索关键词
- **并且** 关键词应当直接反映用户主题的核心含义

#### 场景: 单轮搜索执行
- **当** Agent 执行第 N 轮搜索
- **那么** 必须调用 `fetch_xhs.py`，传入：
  - `--keywords` 为本轮关键词（仅 1 个）
  - `--max-posts` 为本轮配额
  - `--output` 为 `{search_dir}/raw_round_{N}.json`
  - `--seen-ids` 为 `{search_dir}/seen_ids.txt`
- **并且** 必须等待本轮搜索完成后再进入分析阶段

#### 场景: AI 分析与关键词决策
- **当** 第 N 轮搜索完成且 N < 总轮数
- **那么** Agent 必须阅读本轮及之前所有轮次的搜索结果
- **那么** Agent 必须基于 LLM 自然语言理解能力决定下一个关键词
- **那么** Agent 必须记录本次决策的理由（用于发散路径板块）
- **并且** 下一个关键词**必须与用户原始主题保持关联**
- **并且** 应当基于已搜索结果中发现的新方向（如高频品牌、争议点、信息盲区）
- **并且** 应当避免与已搜索过的关键词高度重复

#### 场景: 循环终止
- **当** 当前轮次 = 总轮数（最后一轮搜索完成）
- **那么** Agent 必须停止搜索循环
- **那么** 必须合并所有轮次的数据为最终 `raw.json`

### 需求: 配额平均分配
系统必须将篇数上限均匀分配到每轮搜索中。

#### 场景: 均匀分配
- **当** 篇数上限可被轮数整除
- **那么** 每轮配额 = 篇数上限 ÷ 轮数
- **示例**：50 篇 ÷ 5 轮 = 每轮 10 篇

#### 场景: 不可整除时的分配
- **当** 篇数上限不可被轮数整除
- **那么** 余数必须加到第一轮
- **示例**：50 篇 ÷ 3 轮 = [18, 16, 16]

### 需求: 跨轮去重
`fetch_xhs.py` 必须支持 `--seen-ids` 参数实现跨调用去重。

#### 场景: seen-ids 参数
- **当** `fetch_xhs.py` 接收到 `--seen-ids <filepath>` 参数
- **那么** 启动时必须读取文件中已有的 note_id 集合
- **并且** 搜索时必须跳过已见的 note_id
- **并且** 完成后必须将本次新发现的 note_id 追加写入该文件

#### 场景: seen-ids 文件格式
- **当** 读写 seen-ids 文件
- **那么** 文件格式必须为每行一个 note_id（纯文本）
- **并且** 文件不存在时必须自动创建

#### 场景: 去重后篇数不足
- **当** 去重后实际抓取篇数少于本轮配额
- **那么** 系统必须接受实际篇数，不进行补量
- **并且** 在最终报告中标注实际篇数

### 需求: 数据合并
Agent 必须在所有轮次完成后将分轮数据合并为最终 raw.json。

#### 场景: 合并格式
- **当** 所有轮次搜索完成
- **那么** Agent 必须将所有 `raw_round_N.json` 中的帖子合并为一个 `raw.json`
- **并且** `raw.json` 必须包含 `divergence_path` 字段（发散路径数据）
- **并且** `raw.json` 的 `search_strategy` 字段由发散路径数据替代
- **并且** 必须保留 `search_time`、`keywords`（所有轮次关键词合集）、`posts` 字段

### 需求: 发散模式进度输出
Agent 在发散搜索过程中必须输出进度信息。

#### 场景: 轮次进度
- **当** 开始新一轮搜索
- **那么** 应当输出：`[发散轮次 N/R] 关键词: xxx (配额 M 篇)`
- **当** 一轮搜索完成并决定下一个关键词
- **那么** 应当输出：`[发散决策] 下一关键词: xxx | 理由: xxx`
