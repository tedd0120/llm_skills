# 小红书搜索关键词澄清规范（增量）

## 目的

为搜索关键词澄清阶段新增发散模式选择，用户可在固定关键词模式和发散模式之间选择。

---

## 新增需求

### 需求: 澄清阶段必须提供模式选择
系统在解析用户输入后、衍生关键词之前，必须让用户选择搜索模式。

#### 场景: 展示模式选择
- **当** 用户首次调用 xiaohongshu-scraper 命令
- **那么** 系统必须在解析输入主旨后展示两种模式选项：
  - A. 固定关键词模式（现有流程）
  - B. 发散模式（AI 自动发散搜索）
- **并且** 必须简要说明两种模式的区别

#### 场景: 用户选择固定关键词模式
- **当** 用户选择模式 A
- **那么** 系统必须按照现有澄清流程执行（衍生关键词 → 展示 → 确认）
- **并且** 后续流程与当前完全一致

#### 场景: 用户选择发散模式
- **当** 用户选择模式 B
- **那么** 系统必须进入发散模式参数设定阶段
- **禁止** 执行固定关键词的衍生和展示流程

### 需求: 发散模式必须收集参数
用户选择发散模式后，系统必须收集篇数上限和发散轮数两个参数。

#### 场景: 参数收集
- **当** 用户选择发散模式
- **那么** 系统必须询问用户设定以下参数：
  - 篇数上限（默认 30，硬上限 100）
  - 发散轮数（默认 3，范围 2-10）
- **并且** 必须展示配额分配预览（如 "50篇 ÷ 5轮 = 每轮 10 篇"）

#### 场景: 使用默认参数
- **当** 用户未指定参数而直接确认
- **那么** 系统必须使用默认值：30 篇、3 轮

#### 场景: 参数确认
- **当** 用户确认参数
- **那么** 系统必须立即创建搜索目录（与固定模式相同的命名规则）
- **并且** 进入发散模式执行阶段
