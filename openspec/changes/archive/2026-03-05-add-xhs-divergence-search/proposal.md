## 为什么

当前小红书爬虫 skill 使用固定关键词模式：用户在澄清阶段一次性确认 3 个关键词，然后批量执行搜索。这种模式适合用户已知明确搜索方向的场景，但在面对宏观主题（如"扫地机器人"、"装修经验"）时，用户很难在搜索前就预判最有价值的关键词组合。

实际的人类研究过程是迭代式的——先广泛搜索建立认知，再根据发现深入特定方向。当前的固定模式无法模拟这种自适应的研究行为。

## 变更内容

新增 **发散搜索模式**，与现有固定关键词模式并行，在澄清阶段让用户选择：

### 新增能力

- **模式选择**：澄清阶段新增模式选择（A: 固定关键词 / B: 发散模式），固定模式流程完全不变
- **发散循环**：AI 基于用户宏观主题生成初始关键词 → 搜索 → 分析结果 → 决定下一个关键词 → 重复，直到达到轮数上限
- **参数设定**：用户设定篇数上限和发散轮数，Agent 平均分配每轮配额，搜索过程全自动无需用户参与
- **发散策略**：AI 自由决策下一个关键词（方案 A + 轻约束），仅要求与用户原始主题保持关联
- **跨轮去重**：`fetch_xhs.py` 支持 `--seen-ids` 参数，实现跨调用去重；去重后篇数不足上限时接受实际篇数，在报告中标注
- **发散路径板块**：`_index.md` 报告新增「发散路径」板块，展示每轮关键词选择及 AI 的发散理由
- **格式化同步**：`xiaohongshu-formatter` skill 同步支持发散路径板块的格式美化

### 修改现有能力

- `xiaohongshu-scraper`: SKILL.md 澄清阶段增加模式选择逻辑，执行阶段增加发散循环
- `xiaohongshu-search-clarification` spec: 增加模式选择需求
- `xiaohongshu-scraper` spec: 增加发散模式执行规范
- `xiaohongshu-scraper-report-template` spec: 增加发散路径板块模板
- `fetch_xhs.py`: 支持 `--seen-ids` 参数实现跨调用去重

## 影响

- **SKILL.md**：澄清阶段和执行阶段均需修改
- **fetch_xhs.py**：新增 `--seen-ids` CLI 参数
- **xiaohongshu-formatter SKILL.md**：新增发散路径板块格式化规则
- **4 个 OpenSpec specs**：需新增/修改需求
- **现有固定关键词模式**：无影响，完全向后兼容
