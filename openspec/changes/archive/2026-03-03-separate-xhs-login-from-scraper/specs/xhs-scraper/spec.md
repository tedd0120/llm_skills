## 修改需求

### 需求: 移除内置登录逻辑
搜索脚本必须不再处理登录相关逻辑，假设调用前已确保登录状态。

#### 场景: 执行搜索
- **当** Agent 调用搜索脚本
- **那么** 脚本直接加载已存在的 Cookie 文件
- **那么** 脚本不执行任何登录状态检查或登录流程
- **那么** 脚本直接开始搜索抓取任务

### 需求: 登录失败错误处理
搜索脚本必须在检测到未登录状态时输出明确错误信息并退出。

#### 场景: 检测到未登录
- **当** 搜索过程中检测到用户未登录（如重定向到登录页）
- **那么** 输出错误信息 `[✗] 检测到未登录，请先执行 xiaohongshu-login skill`
- **那么** 以非零退出码退出
- **那么** 不尝试自动登录

## 移除需求

### 需求: 内置登录流程
**Reason**: 登录逻辑已迁移到独立的 `xiaohongshu-login` skill，实现职责分离。

**Migration**: Agent 在调用搜索脚本前必须先调用 `xiaohongshu-login --check-only` 确保已登录。

#### 场景: 旧行为（已移除）
- ~~当~~ Cookie 不存在或无效
- ~~那么~~ 脚本自动启动扫码登录流程
