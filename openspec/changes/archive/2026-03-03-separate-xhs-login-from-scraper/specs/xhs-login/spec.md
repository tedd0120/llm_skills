## 新增需求

### 需求: 登录状态检查
系统必须提供 `--check-only` 模式，在不触发登录流程的情况下验证当前 Cookie 是否有效。

#### 场景: Cookie 有效
- **当** Cookie 文件存在且加载后访问小红书首页检测到用户已登录
- **那么** 输出 `LOGIN_OK` 并以退出码 0 退出

#### 场景: Cookie 无效或不存在
- **当** Cookie 文件不存在或加载后访问首页检测到未登录状态
- **那么** 输出 `LOGIN_OK` 的相反状态（不触发登录流程）并退出

### 需求: 扫码登录
系统必须支持通过二维码方式完成小红书登录。

#### 场景: 首次登录
- **当** Cookie 文件不存在且不使用 `--check-only` 模式
- **那么** 系统启动浏览器，打开小红书登录页，截取二维码并保存到项目根目录
- **那么** 输出 `NEED_LOGIN:xhs_qr_login.png`（包含二维码文件绝对路径）
- **那么** 等待用户扫码完成
- **那么** 扫码成功后保存 Cookie 到环境变量 `XHS_AUTH_STATE` 指定路径
- **那么** 输出 `LOGIN_SUCCESS` 并删除二维码图片
- **那么** 以退出码 0 退出

#### 场景: Cookie 过期重新登录
- **当** Cookie 文件存在但加载后检测到未登录状态
- **那么** 执行与首次登录相同的流程

### 需求: 登录超时处理
系统必须支持可配置的扫码等待超时时间，默认 120 秒。

#### 场景: 扫码超时
- **当** 用户在超时时间内未完成扫码
- **那么** 输出 `LOGIN_TIMEOUT` 并以非零退出码退出
- **那么** 清理二维码图片文件

### 霜求: 登录异常处理
系统必须处理登录过程中的各类异常情况。

#### 场景: 浏览器异常
- **当** 浏览器启动失败或页面加载异常
- **那么** 输出 `LOGIN_FAILED` 并以非零退出码退出

#### 场景: 二维码加载失败
- **当** 登录页二维码元素无法找到或加载超时
- **那么** 输出 `LOGIN_FAILED` 并以非零退出码退出

### 需求: 超时配置
系统必须支持通过命令行参数配置扫码等待超时时间。

#### 场景: 自定义超时
- **当** 用户指定 `--timeout N` 参数（N 为秒数）
- **那么** 扫码等待时间使用指定的 N 秒而非默认的 120 秒
