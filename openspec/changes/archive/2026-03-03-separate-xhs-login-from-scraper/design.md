## 上下文

**当前状态**：
- `fetch_xhs.py` 是一个单体脚本，同时处理登录验证和搜索抓取
- 登录流程中的扫码等待会阻塞 Python 进程
- Agent 只能在脚本执行结束后才能检测到 `NEED_LOGIN` 输出
- 用户经常错过 120 秒扫码窗口导致超时

**约束**：
- 小红书必须登录才能访问内容
- Cookie 有效期有限，需要定期重新登录
- Playwright 浏览器启动和操作是耗时操作

## 目标 / 非目标

**目标：**
- 将登录逻辑从搜索脚本中分离，创建独立的 `xiaohongshu-login` skill
- Agent 能够实时检测登录需求并展示二维码给用户
- 提供可配置的超时机制（默认 120 秒）
- 支持 Cookie 验证模式（`--check-only`）避免不必要的浏览器启动

**非目标：**
- 不改变 Cookie 存储格式和路径（保持 `XHS_AUTH_STATE` 环境变量）
- 不改变搜索抓取的核心逻辑（仅移除登录部分）

## 决策

### 1. 独立 Skill 而非函数分离

**决策**：创建独立的 `xiaohongshu-login` skill，而非在现有 skill 内部拆分函数。

**理由**：
- 清晰的职责边界：登录和搜索是两个独立的用户意图
- Agent 可以选择性调用：先确保登录，再执行搜索
- 便于测试和调试：登录流程可以独立验证

**替代方案**：在 `xiaohongshu-scraper` 内部拆分 `login()` 和 `scrape()` 函数
- **拒绝原因**：仍然存在进程阻塞问题，Agent 无法实时感知

### 2. 输出格式设计

**决策**：使用前缀标记的输出格式，便于 Agent 解析。

```
LOGIN_OK              # Cookie 有效，无需登录
NEED_LOGIN:<path>     # 需要扫码，附带二维码绝对路径
LOGIN_SUCCESS         # 扫码成功，Cookie 已保存
LOGIN_TIMEOUT         # 扫码超时
LOGIN_FAILED          # 登录失败（异常）
```

**理由**：
- Agent 可以通过 `NEED_LOGIN:` 前缀立即识别并展示二维码
- 标准化状态码便于错误处理和重试逻辑

### 3. Cookie 验证策略

**决策**：每次 Agent 执行 scraper 前都调用 `--check-only` 验证 Cookie。

**理由**：
- 小红书 Cookie 可能随时失效（服务端强制登出）
- 提前验证可以避免搜索进行到一半才发现登录失效
- 验证操作相对轻量（仅访问首页检测元素）

**替代方案**：仅检查文件存在性
- **拒绝原因**：文件存在不代表 Cookie 有效，不可靠

### 4. Scraper 失败处理

**决策**：scraper 检测到未登录时输出错误信息并退出，由 Agent 捕获后重试登录流程。

**理由**：
- 保持 scraper 单一职责（只负责搜索）
- Agent 可以根据错误类型决定是否重试
- 避免 scraper 内部复杂的登录重试逻辑

### 5. 二维码文件路径

**决策**：固定保存到项目根目录 `xhs_qr_login.png`，脚本结束后自动清理。

**理由**：
- 简化路径处理，无需临时目录管理
- 统一路径便于 Agent 读取和展示

## 风险 / 权衡

| 风险 | 缓解措施 |
|:-----|:---------|
| Cookie 验证增加额外开销 | 验证只访问首页，不执行完整登录流程，开销 < 5 秒 |
| 登录超时后用户需重新发起搜索 | Agent 自动重试登录流程，无需用户手动操作 |
| 小红书改版导致选择器失效 | 与 `fetch_xhs.py` 共享 `selectors.py`，统一维护 |
| 并发调用可能导致 Cookie 覆盖 | 登录 skill 和 scraper 顺序执行，天然避免并发 |

## 迁移计划

1. **创建 `xiaohongshu-login` skill**
   - 实现 `login_xhs.py` 脚本
   - 编写 `SKILL.md` 文档

2. **修改 `xiaohongshu-scraper` skill**
   - 从 `fetch_xhs.py` 中移除 `_ensure_login` 和 `_qr_login` 方法
   - 更新 `SKILL.md` 添加步骤 1.5（确保已登录）

3. **更新 Agent 执行流程**
   - 澄清阶段后调用 `xiaohongshu-login --check-only`
   - 根据返回状态决定是否需要登录

4. **回滚策略**
   - 保留原有 `fetch_xhs.py` 的登录逻辑作为注释备份
   - 如新流程出现严重问题，可快速恢复原逻辑
