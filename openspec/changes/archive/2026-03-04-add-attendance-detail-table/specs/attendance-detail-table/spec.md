## 新增需求

### 需求:明细表输出

系统必须在达标预测表格之后、CSV 保存提示之前输出本月工作日明细表。

#### 场景:有工作日数据时输出明细表

- **当** 系统完成考勤数据获取且本月有已发生的工作日
- **那么** 系统必须输出包含以下信息的明细表：
  - 日期（MM-DD 格式）
  - 上班打卡时间（HH:MM 格式）
  - 下班打卡时间（HH:MM 格式）
  - 工时（保留 2 位小数，带 h 后缀）
  - 备注（迟到/早退/年假等，无备注时为空）

#### 场景:无工作日数据时不输出明细表

- **当** 本月没有已发生的工作日数据
- **那么** 系统不输出明细表

### 需求:数据过滤

明细表必须仅包含符合条件的工作日记录。

#### 场景:过滤条件应用

- **当** 生成明细表数据
- **那么** 系统必须仅包含满足以下条件的记录：
  - 月份等于查询月份
  - 日期小于当前日期（排除今天）
  - 假期 flag 等于 0（排除节假日）

### 需求:汇总信息

明细表底部必须显示汇总信息。

#### 场景:显示工作日天数和累计工时

- **当** 输出明细表
- **那么** 系统必须在表格底部显示：
  - 工作日天数
  - 累计工时（保留 2 位小数）

### 需求:输出格式规范

系统必须使用统一的 Markdown 表格格式。

#### 场景:明细表格式

- **当** 输出明细表
- **那么** 必须使用以下格式：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  本月明细
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

│  日期   │ 上班时间 │ 下班时间 │  工时   │  备注  │
│---------│----------│----------│---------│---------│
│ 03-02   │ 09:14    │ 20:45    │ 11.52h  │        │
│ 03-03   │ 09:14    │ 18:43    │  9.47h  │        │

共 2 个工作日，累计 21.00 小时
```
