## 修改需求

### 需求: 扫码登录
系统默认执行登录命令时，必须在单次浏览器会话内先检查当前登录状态，并仅在需要时继续扫码登录。

#### 场景: 首次登录输出时序
- **当** 用户未使用 `--check-only` 模式启动登录脚本
- **并且** Cookie 文件不存在或检测到未登录状态
- **那么** 系统必须先输出 `NEED_LOGIN:<qr_path>`
- **并且** `<qr_path>` 必须是二维码图片的绝对路径
- **然后** 才进入等待用户扫码完成阶段

#### 场景: Cookie 过期重新登录输出时序
- **当** 用户未使用 `--check-only` 模式启动登录脚本
- **并且** Cookie 文件存在但加载后检测到未登录状态
- **那么** 系统必须先输出 `NEED_LOGIN:<qr_path>`
- **并且** `<qr_path>` 必须是二维码图片的绝对路径
- **然后** 才进入等待用户扫码完成阶段
