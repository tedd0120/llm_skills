# 设计：修复小红书爬虫时间戳和 Emoji 使用问题

## 上下文

**当前状态**：
- SKILL.md 中目录格式示例为 `20260303_143000_北大口腔拔牙`，看起来像真实时间
- Agent 容易误将示例时间当作可复用的模板时间
- 报告模板中定义了章节 emoji（💬🎯💰⚠️🔍），但没有硬性要求使用
- 之前的 `improve-xhs-skill-ai-only-analysis` 变更移除了固定 emoji 映射规则，强调"自由化"

**约束**：
- 保持 emoji 使用的"自由化"精神（不强制固定映射）
- 但需要确保最低可读性标准
- 跨平台兼容（时间戳生成）

## 目标 / 非目标

**目标：**
- 防止 Agent 使用虚构/示例时间作为目录时间戳
- 确保报告使用 emoji 增强可读性
- 保持 Agent 在具体 emoji 选择上的自由度

**非目标：**
- 不强制使用固定的 emoji 映射规则
- 不限制 Agent 使用超出最低要求的 emoji

## 决策

### 决策 1：时间戳格式使用占位符

**选择**：将格式从 `yyyyMMdd_HHmmss` 改为 `YYYYMMDD_HHmmSS`

**理由**：
- 大写占位符 `YYYYMMDD` 更明确表示需要替换
- 避免看起来像真实时间的示例误导 Agent

**替代方案**：
- 保留小写格式但添加警告文字 → 仍可能被忽略
- 明确命令式（使用 `date` 命令）→ 跨平台兼容性问题

### 决策 2：Emoji 使用硬性最低要求

**选择**：要求报告标题和 8 个主要板块必须使用 emoji，具体 emoji 由 Agent 自行判断

**理由**：
- 平衡"自由化"精神和"多用 emoji"的用户期望
- 明确最低标准，同时保持灵活性

**必须使用 emoji 的位置**：
1. 报告标题
2. 搜索概览
3. 品牌/主题声量分析
4. 评论区高频共识
5. 场景化决策建议
6. 价格/规格参考
7. 避雷指南
8. 关键洞察

## 风险 / 权衡

| 风险 | 缓解措施 |
|-----|---------|
| Agent 可能仍忽略 emoji 要求 | 使用 `[核心要求]` 标签，违反则任务失败 |
| 占位符格式可能让 Agent 困惑 | 在 SKILL.md 中提供明确示例和说明 |
| 不同 Agent 的 emoji 选择差异大 | 这是预期行为，保持自由化 |
