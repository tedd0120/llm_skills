# 提案：修复小红书爬虫时间戳和 Emoji 使用问题

## 为什么

小红书爬虫 skill 在执行中发现了两个问题：

1. **目录时间戳使用虚构时间**：Agent 在创建搜索目录时使用了模板中的示例时间（143000）而非实际系统时间（182300），导致目录名时间与实际执行时间不一致
2. **报告 Emoji 使用不足**：生成的 `_index.md` 报告中仅使用了 1 个 emoji（💬），而模板中定义了 6 个章节 emoji，报告可读性不如预期

第一个问题导致数据管理混乱（目录名时间不反映实际抓取时间），第二个问题降低了报告的用户体验。

## 变更内容

1. **修改目录命名规则**：将格式从 `yyyyMMdd_HHmmss` 改为占位符 `YYYYMMDD_HHmmSS`，并明确禁止使用虚构/示例时间
2. **添加 Emoji 硬性要求**：要求 Agent 在报告标题和 8 个主要板块必须使用 emoji 增强可读性

## 功能 (Capabilities)

### 修改功能

- `xiaohongshu-scraper`:
  - 添加硬性约束：目录时间戳必须使用当前系统时间，禁止使用示例、模板或虚构时间
  - 添加硬性约束：报告标题和 8 个主要板块必须使用 emoji

## 影响

- **SKILL.md**：修改目录创建说明和报告生成要求
- **spec.md**：新增相关约束需求
- **Agent 行为**：强制使用实时时间，强制使用 emoji
