# 小红书爬虫时间戳和 Emoji 规范变更

## 新增需求

### 需求: 目录时间戳必须使用当前系统时间

Agent 在创建搜索目录时，时间戳必须使用当前系统时间，禁止使用示例、模板或虚构时间。

#### 场景: 目录创建时间要求

- **当** Agent 创建搜索目录
- **那么** 必须使用当前系统时间生成时间戳
- **并且** 禁止使用示例时间（如 `143000`）或任何虚构时间
- **示例**：`20260303_182300_北京粤菜推荐`（18:23 执行时）

### 需求: 报告必须使用 Emoji 增强可读性

Agent 在生成 `_index.md` 报告时，必须在报告标题和 8 个主要板块使用 emoji 增强可读性。

#### 场景: 报告标题 Emoji 要求

- **当** Agent 生成报告标题
- **那么** 必须使用适当的 emoji
- **并且** emoji 应当与报告主题相关

#### 场景: 报告板块 Emoji 要求

- **当** Agent 生成报告的 8 个主要板块
- **那么** 每个板块标题必须使用 emoji
- **并且** 具体使用的 emoji 由 Agent 根据内容上下文自行判断
- **禁止** 固定使用特定的 emoji 映射规则

#### 场景: 必须使用 Emoji 的 8 个板块

- **当** Agent 生成 `_index.md` 报告
- **那么** 以下位置必须使用 emoji：
  1. 报告标题
  2. 搜索概览
  3. 品牌/主题声量分析
  4. 评论区高频共识
  5. 场景化决策建议
  6. 价格/规格参考
  7. 避雷指南
  8. 关键洞察

## 修改需求

### 需求: 确认后必须立即创建搜索目录

用户确认关键词后，Agent 必须立即在调用 fetch_xhs.py 之前创建搜索目录。

#### 场景: 目录格式更新

- **当** Agent 创建搜索目录
- **那么** 目录格式必须为 `YYYYMMDD_HHmmSS_搜索主旨`
- **并且** 时间戳部分（`YYYYMMDD_HHmmSS`）必须替换为当前系统时间
- **禁止** 使用示例时间或虚构时间
