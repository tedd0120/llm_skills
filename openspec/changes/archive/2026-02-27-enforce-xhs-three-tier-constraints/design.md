## 上下文

当前 `xiaohongshu-scraper` 的 SKILL.md 使用非规范性的模糊语言（"建议"、"推荐"、"可以"），导致 Agent 在执行任务时可能跳过关键步骤，影响输出质量和一致性。

## 目标 / 非目标

**目标：**
- 在 SKILL.md 和 workflow.md 中引入清晰的三层约束标签系统
- 将所有模糊语言替换为符合优先级的明确表述
- 添加优先级定义和冲突处理规则
- 确保 Agent 能够严格区分核心要求（P0）和可选路径（P2）

**非目标：**
- 不修改任何 Python 脚本代码
- 不改变抓取脚本的功能行为
- 不调整报告生成逻辑

## 决策

### 1. 标签格式选择

**决策：** 使用文本标签 `[核心要求]`、`[标准操作]`、`[灵活选项]`、`[参考信息]`，不使用 Emoji

**理由：**
- 文本标签更正式，适合规范文档
- 避免视觉干扰，保持文档整洁
- 易于在纯文本环境下解析

**替代方案：** 使用 Emoji + 文本（如 🔴 [核心要求]）- 因视觉混乱被否决

### 2. 优先级定义

采用四层优先级体系：

| 优先级 | 标签 | 关键词 | 违反后果 |
|-------|-----|--------|---------|
| P0 | [核心要求] | 必须、禁止、绝对 | 任务失败 |
| P1 | [标准操作] | 应当、要求 | 质量下降 |
| P2 | [灵活选项] | 可、可以、可选 | 无（提供备选） |
| P3 | [参考信息] | 注意、提示、⏱️ | 无（仅提示） |

### 3. 冲突处理规则

```
优先级降序：P0 > P1 > P2 > P3
- P0 受阻时 → 必须停止，任务失败
- P1 受阻时 → 可降级到 P2（并标注原因）
- P2/P3 可忽略
```

### 4. 应用范围

**必须修改的文件：**
1. `.agent/skills/xiaohongshu-scraper/SKILL.md`
   - 第 25 行：Xvfb 推荐 → `[标准操作]`
   - 第 48 行：headless 降级 → `[灵活选项]`
   - 第 106-107 行：JSON 保存 → `[核心要求]`
   - 第 252-253 行：故障排除 → `[核心要求]`
   - 新增"优先级定义"章节

2. `.agent/workflows/xiaohongshu-scraper.md`
   - 步骤 4：添加标签前缀
   - 保持与 SKILL.md 一致

## 风险 / 权衡

**风险：** 过度使用 P0 标签可能导致 Agent 缺乏灵活性
**缓解措施：** 仅将真正关键步骤（如数据保存、报告生成）标记为 P0，其他降级为 P1

**风险：** 优先级冲突可能不明确
**缓解措施：** 在新增的"优先级定义"章节中提供清晰的冲突处理示例

**风险：** 现有 SKILL.md 内容量大，逐行修改容易遗漏
**缓解措施：** 在 tasks.md 中创建检查清单，逐段验证
