## 为什么

当前工时技能的输出格式为简单的纯文本，信息展示不够结构化，且缺乏对"剩余工作日需要多少工时才能达标"的预测性信息。用户无法快速了解本月工作进度和后续节奏规划。

## 变更内容

- **新增**：格式化统计输出，以表格形式展示基础统计信息
- **新增**：达标预测功能，基于目标平均工时（10/10.5/11小时）计算剩余工作日每日所需工时
- **修改**：CSV 输出改为默认行为（强制保存），确保基于完整数据进行计算

## 功能 (Capabilities)

### 新增功能
- `attendance-statistics`: 考勤统计输出功能，包含基础统计（已工作天数、累计工时、平均工时、剩余工作日）和达标预测（按目标工时计算剩余日均需工时）

### 修改功能
- 无（当前无现有规范）

## 影响

- **代码变更**：`.claude/skills/teams-attendance/scripts/fetch_attendance.py`
  - 新增 `print_statistics()` 函数用于格式化输出
  - 修改 `parsed_att_date()` 函数，强制保存 CSV
  - 修改 `main()` 函数，默认输出路径改为 `data/attendance_{YYYYMM}.csv`
- **用户体验**：CLI 输出从简单文本变为结构化表格，便于阅读
- **向后兼容**：函数返回值保持不变，不影响代码调用
