## 新增需求

### 需求:基础统计输出

系统必须以结构化格式输出考勤基础统计信息，包括当前日期、已工作天数、已累计工时、当前平均工时、剩余工作日。

#### 场景:正常月份统计输出

- **当** 系统完成考勤数据获取
- **那么** 系统必须输出以下信息：
  - 当前日期（YYYY-MM-DD 格式）
  - 已工作天数（不包含今天，仅计算 isrest=0 的日期）
  - 已累计工时（不包含今天的工时总和）
  - 当前平均工时（已累计工时 / 已工作天数，保留 2 位小数）
  - 剩余工作日（本月范围内，日期 >= today 且 isrest=0 的天数）

#### 场景:无考勤数据

- **当** API 未返回任何考勤数据
- **那么** 系统必须输出"暂无本月考勤数据"

### 需求:达标预测计算

系统必须基于硬编码的目标工时（10、10.5、11 小时）计算剩余工作日每日所需的平均工时。

#### 场景:有剩余工作日时预测

- **当** 剩余工作日大于 0
- **那么** 系统必须计算并输出：
  - 目标总工时 = 目标 × (已工作天数 + 剩余工作日)
  - 剩余需工时 = 目标总工时 - 已累计工时
  - 剩余日均需 = 剩余需工时 / 剩余工作日（保留 1 位小数）

#### 场景:本月已结束时处理

- **当** 本月剩余工作日为 0
- **那么** 系统必须输出"本月已结束"且不显示预测表格

### 需求:CSV 数据保存

系统必须将考勤明细保存为 CSV 文件，确保统计计算基于完整数据。

#### 场景:CLI 调用默认保存

- **当** 用户通过 CLI 调用且未指定 --output 参数
- **那么** 系统必须将 CSV 保存至 `data/attendance_{YYYYMM}.csv`

#### 场景:代码调用默认保存

- **当** 用户通过代码调用且未指定 output_path 参数
- **那么** 系统必须将 CSV 保存至 `data/attendance_{YYYYMM}.csv`

#### 场景:指定路径保存

- **当** 用户指定了输出路径
- **那么** 系统必须将 CSV 保存至指定路径

### 需求:输出格式规范

系统必须使用统一的表格格式输出统计信息。

#### 场景:基础统计表格格式

- **当** 输出基础统计
- **那么** 必须使用以下格式：

```
═══════════════════════════════════════════════════════════
  考勤统计 - 2026-03
═══════════════════════════════════════════════════════════
  当前日期         2026-03-04
  已工作天数       15 天
  已累计工时       120.0 小时
  当前平均工时     8.0 小时/天
  剩余工作日       10 天
```

#### 场景:达标预测表格格式

- **当** 输出达标预测
- **那么** 必须使用以下表格格式：

```
┌────────────┬──────────────┐
│  目标工时   │  剩余日均需  │
├────────────┼──────────────┤
│   10.0 h   │    8.5 h     │
│   10.5 h   │    9.2 h     │
│   11.0 h   │   10.1 h     │
└────────────┴──────────────┘
```

#### 场景:CSV保存提示

- **当** CSV 保存完成
- **那么** 系统必须输出 `考勤明细已保存至: {path}`
