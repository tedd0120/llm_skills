## 上下文

**当前状态**：
- 工时技能通过 360teams API 获取考勤数据
- 输出为简单文本行，格式：`当前日期 {date} 检查月份 {month} 有效工作日 {days} 有效工时 {hours} 平均工时 {avg}`
- CSV 输出是可选的（通过 `--output` 参数）

**约束**：
- 必须基于 API 返回的 `isrest` 字段判断工作日/假期
- 数据包含本月+下月，确保计算完整
- 目标工时硬编码为 [10, 10.5, 11]
- 向后兼容：函数返回值 `[avg_hours, att_df]` 保持不变

## 目标 / 非目标

**目标：**
- 提供结构化的统计输出，便于快速阅读
- 计算并显示"剩余工作日每日需多少工时才能达标"
- 强制保存 CSV 以确保数据完整性

**非目标：**
- 不改变函数 API 接口
- 不支持动态目标工时配置（硬编码即可）
- 不支持图形化界面

## 决策

| 决策 | 选择 | 理由 |
|-----|------|------|
| 输出格式 | 纯文本表格（ASCII） | 兼容所有终端，无需额外依赖 |
| 剩余工作日计算 | `日期 >= today AND isrest = 0` | 包含今天，符合用户预期 |
| 已工作天数计算 | `日期 < today AND isrest = 0` | 排除今天，避免数据不完整 |
| CSV 保存策略 | 强制保存，默认路径 | 确保数据可用于后续分析 |
| "本月已结束"判断 | 剩余工作日 = 0 | 本月范围内无剩余工作日 |

## 风险 / 权衡

| 风险 | 缓解措施 |
|-----|---------|
| 假期数据不完整（API 仅返回当月） | 使用本月+下月数据确保覆盖 |
| 剩余工作日计算偏差 | 严格基于 API 的 `isrest` 字段 |
| CSV 文件覆盖 | 文件名包含月份，自然避免覆盖 |
| 终端宽度限制 | 表格设计简洁，关键信息不换行 |
