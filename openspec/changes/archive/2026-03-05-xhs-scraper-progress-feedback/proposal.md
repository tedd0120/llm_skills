## 为什么

当前 xiaohongshu-scraper 在执行抓取时，用户需要等待数分钟却看不到任何进度反馈。Python 脚本虽然有进度输出，但由于 Bash 工具的阻塞调用机制，这些输出只能在脚本完成后一次性展示，导致用户体验不佳——长时间等待期间无法知道系统是否在工作。

## 变更内容

1. **Python 脚本输出实时化**：为 `fetch_xhs.py` 的所有进度输出添加 `flush=True`，确保输出不被缓冲
2. **Agent 调用模式变更**：修改 skill.md 指导 Agent 使用后台运行 + 轮询模式，在抓取过程中主动向用户同步进度
3. **进度信息格式化**：规范进度输出格式，便于 Agent 解析并展示给用户

## 功能 (Capabilities)

### 新增功能
- `xhs-scraper-progress-feedback`: 提供小红书爬虫执行过程中的实时进度反馈，包括当前搜索关键词、已抓取篇数/总篇数等信息

### 修改功能
- `xiaohongshu-scraper`: 修改执行模式，从阻塞调用改为后台运行 + 轮询进度，增强用户对抓取过程的可见性

## 影响

- **代码修改**：`fetch_xhs.py` 添加 `flush=True` 参数
- **文档修改**：`skill.md` 添加后台运行和进度轮询的执行指导
- **用户体验**：抓取过程中可见实时进度，不再需要"盲等"
