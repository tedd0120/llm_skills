## 修改需求

### 需求:SKILL.md 必须明确编排层职责边界

xiaohongshu-scraper 的 SKILL.md 必须仅包含编排层相关内容，子 skill 仅用一句话概括功能并引用其文档。

#### 场景:子 skill 概括格式
- **当** SKILL.md 描述内部架构
- **那么** 每个子 skill 必须使用一行描述：「`<skill-name>` - <一句话功能概括>（详见其 SKILL.md）」
- **并且** 禁止在 SKILL.md 中重复子 skill 的详细规范

#### 场景:编排层保留内容
- **当** 编写 SKILL.md 内容
- **那么** 必须保留：澄清阶段流程、执行阶段编排流程图、核心要求清单（编排层约束）、前置依赖配置、上下文传递规范
- **并且** 禁止包含：完整报告模板、引用格式详细规范、CLI 命令详细示例

### 需求:SKILL.md 必须消除章节重复

xiaohongshu-scraper 的 SKILL.md 必须消除文件内的重复章节。

#### 场景:核心要求与重要提醒合并
- **当** SKILL.md 包含「核心要求清单」章节
- **那么** 必须删除「重要提醒」章节
- **并且** 核心要求清单仅保留编排层相关的约束

#### 场景:删除 CLI 基础用法章节
- **当** SKILL.md 包含「CLI 基础用法」章节
- **那么** 必须删除该章节
- **并且** 在执行步骤中改为引用：「调用 xiaohongshu-fetch（详见其 SKILL.md）」

### 需求:SKILL.md 必须删除跨文件重复内容

xiaohongshu-scraper 的 SKILL.md 必须删除已在其他 skill 文档中定义的内容。

#### 场景:删除报告模板
- **当** SKILL.md 包含完整报告模板章节
- **那么** 必须删除该章节
- **并且** 改为引用：「调用 xiaohongshu-summarize（详见其 SKILL.md）生成报告」

#### 场景:删除引用格式规范
- **当** SKILL.md 包含引用格式规范的详细表格
- **那么** 必须删除该表格
- **并且** 该规范已在 xiaohongshu-summarize 中定义

### 需求:核心要求清单必须聚焦编排层约束

xiaohongshu-scraper 的核心要求清单必须仅包含编排层相关的约束，删除属于子 skill 的要求。

#### 场景:保留的编排层约束
- **当** 编写核心要求清单
- **那么** 必须保留：
  1. 必须完成澄清阶段后才能进入执行阶段
  2. 澄清阶段必须先执行模式选择
  3. 模式 A 必须衍生关键词并获得用户确认
  4. 模式 B 必须收集参数并展示配额预览
  5. 用户确认后必须立即创建搜索目录
  6. 目录时间戳必须使用当前系统时间
  7. 生成报告后必须将原文发送到用户对话框

#### 场景:删除的子 skill 约束
- **当** 审核核心要求清单
- **那么** 必须删除属于子 skill 的约束：
  - 发散模式使用 --seen-ids（属于 fetch skill）
  - 报告板块数量（属于 summarize skill）
  - 禁止调用外部脚本分析（属于 summarize skill）
