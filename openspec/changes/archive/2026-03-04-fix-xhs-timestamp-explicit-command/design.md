## 上下文

当前 `xiaohongshu-scraper` skill 的 SKILL.md 中，"步骤 1：创建搜索目录"部分使用了意图声明式的指令（"时间戳必须使用当前系统时间，禁止使用示例或虚构时间"），并配以具体的数字示例（`20260303_182300_北大口腔拔牙/`）。

在实际执行中发现，Agent 会从示例中"学习"时间格式，直接使用示例中的数字（如 `143000`）而不是主动调用系统时间获取真实值。这违反了规范中的核心要求。

## 目标 / 非目标

**目标：**
- 修改 SKILL.md 中的目录创建指令格式，使用显式的 Bash 命令模板
- 确保 Agent 执行时能够正确获取实时系统时间
- 移除可能诱导模仿的具体数字示例

**非目标：**
- 不修改目录命名格式规则
- 不修改时间戳的格式要求（仍为 `YYYYMMDD_HHmmSS`）
- 不修改其他步骤的指令风格

## 决策

### 指令格式从意图声明改为显式命令

**选择：** 将"目录格式 + 说明 + 示例"的格式改为"可执行命令模板"

**理由：**
- Agent 更倾向于复制-执行命令，而非从意图推断实现方式
- 显式命令（`$(date +%Y%m%d_%H%M%S)`）消除了歧义
- 示例中的具体数字本身就成为了一种"数据源"，诱导模仿

**替代方案：**
1. 保持意图声明，但使用占位符示例（如 `YYYYMMDD_HHmmSS`）—— 可能仍然导致 Agent 使用占位符字面值
2. 在脚本中自动生成时间戳 —— 需要修改 fetch_xhs.py，增加复杂度
3. 使用 Python 单行命令获取时间 —— 与现有 Bash 风格不一致

### 示例格式调整

**选择：** 命令执行示例 + 预期输出，而非具体目录示例

**理由：**
- 展示"输入命令 → 输出结果"的流程更清晰
- 避免提供可被直接复制的具体时间值

## 风险 / 权衡

- **风险**：如果 Agent 执行环境不支持 Bash `date` 命令（如 Windows），命令可能失败
  - **缓解措施**：SKILL.md 中已说明使用 Windows 环境，但 Git Bash 或 WSL 通常可用

- **权衡**：显式命令降低了 Agent 的"理解"需求，但减少了灵活性
  - 这是可接受的，因为获取当前时间是高度确定性的操作，不需要创造性

2. 验证修改后的指令在不同执行环境下的正确性
3. 无需代码迁移，仅文档更新
