# 提案: 强制小红书抓取输出格式规范

## 为什么

当前 `SKILL.md` 中的 Agent 操作流步骤 3-5 虽然描述了正确的输出结构（创建文件夹 → 每篇帖子生成 MD → 汇总索引），但语气过于柔和（"建议"、"模板建议"），导致 Agent 在实际执行时经常跳过中间步骤，直接生成一个合并的汇总文件。

**实际案例**：`广州75平扫拖机器人_20260227_152927/` 搜索中，Agent 完全跳过了每篇帖子的 MD 生成，只输出了一个排行榜式的 `_index.md` 和 `ranking.json`。

## 变更内容

修改 `.agent/skills/xiaohongshu-scraper/SKILL.md`，强化输出格式约束：

1. **新增「⛔ 强制输出规范」section** — 放在 Agent 操作流开头，用最醒目的方式列出三条不可违反的铁律
2. **强化步骤 3（创建文件夹）** — 从"建议"改为"必须"，添加禁止条款
3. **强化步骤 4（每篇帖子 MD）** — 添加明确的"绝对禁止"红线（禁止跳过、禁止合并、禁止只生成汇总）
4. **强化步骤 5（汇总索引）** — 强调必须在所有单篇 MD 完成后才能生成，必须引用每篇 MD 链接

## 影响

- **文件**: `.agent/skills/xiaohongshu-scraper/SKILL.md`
- **影响范围**: 仅影响 Agent 读取 SKILL.md 时的行为指导，不涉及脚本代码变更
- **风险**: 低 — 纯文档改动，不改变抓取逻辑
