# 设计: 强制小红书抓取输出格式

## 背景与目标

**背景**：目前小红书抓取 Agent 有时未能严格遵守输出格式，特别是跳过针对每篇帖子生成单独 Markdown 文件的步骤，直接输出一个聚合的 `_index.md`。

**目标**：通过修改 `SKILL.md`，使用更具约束力的指令和结构，迫使大型语言模型 (LLM) 在执行 Agent 任务时，严格按照「总目录 -> 单篇解析 -> 汇总索引」的步骤输出结果，不跳过任何中间环节。

## 决策

1.  **引入醒目的红线警告区域**：
    *   在 `SKILL.md` 的顶部或“Agent 标准操作流”的开头，增加一个专门的 `⛔ 强制输出规范` 段落。
    *   将之前散见于各步骤的建议性描述，提炼为三条绝对指令。
2.  **修改步骤描述词汇**：
    *   将建议性动词（例如“根据需要”、“建议”）替换为强制性动词（“必须”、“绝对禁止”）。
3.  **明确先后依赖关系**：
    *   在步骤 5 (`_index.md`) 的描述中，明确指出其必须在步骤 4 (所有单篇 MD) 完成后才能执行。这防止 LLM 提前进行总结。

## 风险 / 权衡

*   **风险**：过于严格的约束可能导致部分模型在某些特殊情况下（例如帖子被删或内容极短）强行生成低质量的单篇分析。
*   **缓解措施**：在单篇分析要求中保留对"无有效评论"等情况的妥协处理格式（如“暂无有价值评论”），但在生成独立 MD 文件的结构要求上绝不妥协。
