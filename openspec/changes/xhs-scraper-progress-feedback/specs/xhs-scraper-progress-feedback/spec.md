# 小红书爬虫进度反馈规范

## 目的

定义小红书爬虫执行过程中实时进度反馈的规范，确保用户能够及时了解抓取进度。

---

## 新增需求

### 需求: Python 脚本必须实时输出进度信息

`fetch_xhs.py` 脚本的所有进度输出必须使用 `flush=True` 强制刷新缓冲区。

#### 场景: 关键词搜索开始
- **当** 脚本开始搜索一个关键词
- **那么** 必须输出 `[*] 搜索关键词: {keyword} (配额 {quota}, 总剩余 {remaining})`
- **并且** 必须使用 `flush=True`

#### 场景: 帖子链接收集完成
- **当** 脚本完成帖子链接收集
- **那么** 必须输出 `找到 {count} 篇（去重后），抓前 {limit} 篇`
- **并且** 必须使用 `flush=True`

#### 场景: 单篇帖子抓取中
- **当** 脚本抓取单个帖子
- **那么** 必须输出 `[{current}/{total}] {url_preview}...`
- **并且** 必须使用 `flush=True`

#### 场景: 抓取完成
- **当** 脚本完成所有抓取
- **那么** 必须输出 `[✓] 共 {total} 篇 → {output_file}`
- **并且** 必须使用 `flush=True`

### 需求: Agent 必须使用后台运行 + 轮询模式

Agent 在调用 `fetch_xhs.py` 时必须使用后台运行模式，并轮询进度向用户展示。

#### 场景: 启动抓取
- **当** Agent 调用抓取脚本
- **那么** 必须使用 `run_in_background=true` 启动后台任务
- **并且** 必须保存返回的 task_id

#### 场景: 进度轮询
- **当** 后台任务正在运行
- **那么** Agent 必须每隔约 5 秒调用 `TaskOutput(task_id, block=false)` 获取最新输出
- **并且** 必须解析输出中的进度信息
- **并且** 必须向用户展示当前状态（关键词、进度）

#### 场景: 完成等待
- **当** Agent 检测到 `[✓] 共` 输出或任务状态为 completed
- **那么** 必须调用 `TaskOutput(task_id, block=true)` 获取完整输出
- **并且** 必须检查退出码是否为 0
- **并且** 非零退出码时必须报告错误

### 需求: 进度展示必须清晰可读

Agent 向用户展示的进度信息必须包含关键信息且格式友好。

#### 场景: 关键词级别进度
- **当** 展示关键词搜索进度
- **那么** 必须显示当前搜索的关键词
- **并且** 必须显示配额和总剩余数量

#### 场景: 帖子级别进度
- **当** 展示帖子抓取进度
- **那么** 必须显示当前进度（如 [3/10]）
- **并且** 必须显示抓取的帖子 URL 或标题

#### 场景: 完成通知
- **当** 抓取完成
- **那么** 必须显示总抓取篇数
- **并且** 必须显示输出文件路径
