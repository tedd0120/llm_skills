# 小红书爬虫执行规范（增量）

## 目的

此文档包含 `xiaohongshu-scraper` 功能的增量规范变更，用于实现实时进度反馈。

---

## 新增需求

### 需求: Agent 必须使用后台运行模式执行抓取

Agent 调用 `fetch_xhs.py` 时必须使用后台运行 + 轮询模式，而非阻塞模式。

#### 场景: 抓取脚本调用
- **当** Agent 进入执行阶段并准备调用 `fetch_xhs.py`
- **那么** 必须使用 `run_in_background=true` 参数启动后台任务
- **禁止** 使用阻塞模式调用

#### 场景: 进度轮询循环
- **当** 后台任务正在运行
- **那么** Agent 必须进入轮询循环，每约 5 秒获取一次输出
- **并且** 必须解析输出中的进度信息（关键词、篇数）
- **并且** 必须向用户展示当前进度

#### 场景: 抓取完成处理
- **当** 检测到抓取完成（输出包含 `[✓] 共` 或状态为 completed）
- **那么** 必须获取完整的任务输出
- **并且** 必须验证退出码为 0
- **并且** 退出码非 0 时必须向用户报告错误

---

## 修改需求

### 需求: 执行步骤 2 必须使用后台运行模式

原有步骤 2 为阻塞调用 `fetch_xhs.py`，现修改为后台运行 + 轮询模式。

#### 场景: 启动后台抓取任务
- **当** Agent 执行步骤 2（抓取脚本）
- **那么** 必须使用后台模式启动 `fetch_xhs.py`
- **并且** 命令格式为：
  ```bash
  python .claude/skills/xiaohongshu-scraper/scripts/fetch_xhs.py \
      --keywords "关键词1,关键词2,关键词3" \
      --max-posts 30 \
      --search-strategy '[...]' \
      --output "data/xiaohongshu/YYYYMMDD_HHmmSS_搜索主旨/raw.json"
  ```
- **并且** 必须使用 `run_in_background=true`

#### 场景: 进度展示
- **当** 获取到新的输出内容
- **那么** 必须向用户展示：
  - 当前搜索的关键词
  - 当前进度（已抓取/总目标）
  - 当前抓取的帖子信息

#### 场景: 错误处理
- **当** 脚本以非零退出码结束
- **那么** 必须检查输出中是否包含 `检测到未登录`
- **如果是** 则提示用户执行 `xiaohongshu-login` skill
- **否则** 展示错误信息并停止任务
