## 新增需求

### 需求: codex 文档必须清理与实际实现不一致的描述
系统必须清理 codex 相关文档中与真实文件结构或目标架构不一致的技能说明。

#### 场景: 清理失效 login 引用
- **当** codex 相关 README、SKILL.md 或设计说明提到独立 login skill
- **那么** 必须将其改写为 orchestrator 内部认证步骤或明确标记为旧体系说明
- **并且** 禁止在 codex 文档中继续把 login 描述为必经的独立 skill 入口

#### 场景: 清理旧架构误导
- **当** codex 文档描述小红书架构时
- **那么** 必须明确区分“旧版保留实现”和“codex 推荐架构”
- **并且** 禁止让读者误以为两套架构的边界完全相同

### 需求: codex 文档必须消除重复职责描述
系统必须删除或收敛重复描述，避免多个文档对同一职责进行平行定义。

#### 场景: 报告职责收敛
- **当** codex 文档描述报告阶段
- **那么** 必须将报告生成、格式增强和超链接渲染统一归入 report-codex 组件
- **并且** 禁止在多个 codex 文档中重复内嵌完整报告模板或重复说明相同后处理规则

#### 场景: 编排职责收敛
- **当** codex 文档描述 scraper-codex 入口职责
- **那么** 必须聚焦用户澄清、流程调度和最终输出
- **并且** 禁止在入口文档中重复定义本应由 fetch-codex 或 report-codex 独占的底层细节

### 需求: codex 脚本布局必须清理重复基础脚本
系统必须在 codex 版目录中清理重复基础脚本，并将共享实现收敛到 core-codex。

#### 场景: 选择器脚本唯一来源
- **当** codex 版抓取和认证逻辑都需要页面选择器
- **那么** 系统必须只保留一份共享的选择器脚本实现
- **并且** 禁止在多个 codex 技能目录中保留内容重复的 `xhs_selectors.py`

#### 场景: 共享脚本引用
- **当** codex 版组件引用共享基础脚本
- **那么** 必须从 core-codex 共享位置引用
- **并且** 禁止为规避导入而复制同名脚本到多个目录

## 修改需求

### 需求: README 中的小红书模块说明必须反映 codex 推荐入口
仓库 README 在描述小红书能力时必须同时说明旧版保留与 codex 推荐入口的关系。

#### 场景: README 入口说明
- **当** README 描述小红书技能入口
- **那么** 必须将 `xiaohongshu-scraper-codex` 标识为新架构推荐入口
- **并且** 必须说明旧版 skills 仍保留用于兼容参考

#### 场景: README 组件列表
- **当** README 列出 codex 相关内部组件
- **那么** 必须反映 `scraper-codex`、`fetch-codex`、`report-codex` 和 `core-codex` 的边界
- **并且** 禁止把 summarize/formatter 拆分链继续描述为 codex 版默认路径
