## 新增需求

### 需求: codex 小红书技能套件必须以单一入口对外暴露
系统必须新增一套带 `-codex` 后缀的小红书技能，并且仅允许 `xiaohongshu-scraper-codex` 作为面向用户的统一入口。

#### 场景: codex 入口命名
- **当** 创建新小红书技能目录
- **那么** 面向用户的入口目录必须命名为 `xiaohongshu-scraper-codex`
- **并且** 禁止覆盖现有的 `xiaohongshu-scraper` 目录

#### 场景: 入口职责
- **当** 用户调用 codex 版小红书能力
- **那么** 系统必须由 `xiaohongshu-scraper-codex` 负责澄清阶段、执行进度展示与最终报告发送
- **并且** 禁止要求用户手动调用 codex 版内部组件完成完整流程

### 需求: codex 架构必须包含内部 orchestrator 层
系统必须在 codex 架构中提供内部 orchestrator 层，统一处理目录初始化、登录、模式调度、任务状态与结果收敛。

#### 场景: 固定模式编排
- **当** 用户在 codex 入口选择固定关键词模式并完成确认
- **那么** orchestrator 必须创建输出目录、确保登录、调用抓取组件、调用报告组件并完成最终校验

#### 场景: 发散模式编排
- **当** 用户在 codex 入口选择发散模式并完成参数确认
- **那么** orchestrator 必须按轮次驱动抓取、记录发散路径、合并轮次结果并调用报告组件生成最终报告

### 需求: codex 架构必须提供共享基础层
系统必须新增共享基础层，用于承载多个 codex 技能共同依赖的基础脚本和数据契约。

#### 场景: 共享选择器
- **当** codex 抓取或登录逻辑需要使用页面选择器
- **那么** 系统必须从共享基础层读取统一的选择器定义
- **并且** 禁止在多个 codex 技能目录中保留重复的选择器实现

#### 场景: 共享浏览器与会话配置
- **当** codex 技能需要启动浏览器或读取认证状态
- **那么** 系统必须复用共享基础层中的浏览器配置与认证状态解析逻辑
- **并且** 禁止在多个 codex 技能中复制平台判断和默认 auth 路径逻辑

### 需求: 旧技能必须保留为兼容参考
系统必须允许原有小红书 skills 与 codex 版 skills 并存。

#### 场景: 新旧并存
- **当** 引入 codex 版技能套件
- **那么** 系统必须保留现有 `xiaohongshu-scraper`、`xiaohongshu-fetch`、`xiaohongshu-summarize` 和 `xiaohongshu-formatter` 目录
- **并且** 禁止通过创建 codex 版目录覆盖旧版实现

## 修改需求

### 需求: 登录边界说明必须与 codex 架构一致
新 codex 架构中的登录能力必须作为编排层内部能力存在，而不是新的独立 skill 边界。

#### 场景: codex 登录说明
- **当** 编写 codex 版入口或架构文档
- **那么** 必须将登录描述为 orchestrator 内部的认证步骤
- **并且** 禁止要求用户先调用独立的 `xiaohongshu-login-codex` 才能继续完整流程

#### 场景: 旧版引用隔离
- **当** 旧规范或旧文档仍然提及独立 login skill
- **那么** codex 版文档必须明确这些描述仅属于旧体系或历史材料
- **并且** 禁止把旧版 login skill 描述继承为 codex 版默认架构
