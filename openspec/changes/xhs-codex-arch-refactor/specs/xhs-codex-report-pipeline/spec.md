## 新增需求

### 需求: codex 架构必须提供统一的报告流水线能力
系统必须新增 `xiaohongshu-report-codex`，统一承担报告生成、超链接渲染和格式化职责，并输出最终版 `_index.md`。

#### 场景: 统一报告输出
- **当** codex 报告组件接收已抓取的数据目录
- **那么** 组件必须读取 `raw.json` 并输出最终版 `_index.md`
- **并且** 禁止要求额外调用独立的格式化组件才能得到可发送结果

#### 场景: 统一组件内部的 draft/finalizer 协议
- **当** codex 报告组件执行报告阶段
- **那么** 组件可以先由 LLM 基于 `raw.json` 生成中间产物 `report_draft.md`
- **并且** 允许同一组件内的脚本 finalizer 再将 `report_draft.md` 渲染为最终 `_index.md`
- **并且** 编排层仍然只能面向单一 `xiaohongshu-report-codex` 组件，而不是重新拆回 summarize 与 formatter

#### 场景: 发散模式报告
- **当** `raw.json` 包含 `divergence_path` 字段
- **那么** 报告组件必须在最终报告中包含“搜索发散路径”板块
- **并且** 必须保留发散轮次、关键词、实际篇数和发散理由的可追溯信息

### 需求: codex 报告流水线必须整合超链接渲染
系统必须将链接占位符处理与最终 Markdown 渲染收敛到同一组件内。

#### 场景: 启用超链接
- **当** 抓取结果目录中存在 `id_url_map.json`
- **那么** 报告组件必须将帖子引用渲染为可点击链接
- **并且** 禁止依赖独立 formatter 组件完成链接替换

#### 场景: 未启用超链接
- **当** 抓取结果目录中不存在 `id_url_map.json`
- **那么** 报告组件必须生成纯文本引用格式
- **并且** 不得因为缺少映射文件导致报告生成失败

### 需求: codex 报告流水线必须保留 AI 分析约束
系统必须继续以 LLM 自然语言理解作为报告分析的唯一语义分析手段。

#### 场景: 内容分析
- **当** codex 报告组件分析帖子与评论内容
- **那么** 必须使用 LLM 自然语言理解能力生成结论
- **并且** 禁止调用外部语义分析脚本、词频统计脚本或实体识别脚本替代分析

#### 场景: finalizer 后处理
- **当** codex 报告组件执行脚本 finalizer
- **那么** finalizer 仅可执行最终 Markdown 渲染、链接替换、标题/分割线整理、评论清理与固定元信息落盘
- **并且** 不得在 finalizer 中新增主题提炼、优缺点总结、情绪判断、价格/规格抽取等语义分析逻辑

#### 场景: 可追溯输出
- **当** codex 报告组件生成结论、优缺点或建议
- **那么** 必须为关键结论提供可追溯的帖子标题或评论引用
- **并且** 在样本不足时必须明确说明样本有限

## 修改需求

### 需求: 报告阶段边界必须从双组件改为单组件
新 codex 架构中的报告阶段必须使用单一组件完成生成与格式增强，而不是先 summarize 再 formatter。

#### 场景: 编排层调用
- **当** orchestrator 完成抓取阶段并准备生成报告
- **那么** 必须只调用 `xiaohongshu-report-codex`
- **并且** 禁止继续要求编排层串行调用独立的 summarize 和 formatter 组件

#### 场景: 编排层 handoff
- **当** orchestrator 完成抓取但尚未拿到 `report_draft.md`
- **那么** 必须明确输出 handoff 信息，要求先由 LLM 按报告组件协议生成草稿
- **并且** 在草稿存在后再调用同一组件内的 finalizer 生成 `_index.md`

#### 场景: 文档职责说明
- **当** 编写 codex 版报告组件文档
- **那么** 必须同时说明报告结构生成、emoji 增强、超链接渲染和评论清理属于同一职责边界
- **并且** 必须说明该单组件边界内部允许存在 `report_draft.md` 中间产物与脚本 finalizer
- **并且** 禁止把这些能力拆散描述为多个必须串联的 codex 子 skill

## 移除需求

### 需求: 报告最终渲染依赖独立 formatter 组件
**Reason**: codex 架构将报告生成与格式增强合并为单一流水线，减少 `_index.md` 半成品协议和跨组件耦合。
**Migration**: 新编排层直接调用 `xiaohongshu-report-codex` 完成“LLM 草稿 + finalizer 定稿”并生成最终 `_index.md`；旧版 formatter 仅作为兼容参考保留。

#### 场景: 旧行为（已移除）
- ~~当~~ 报告草稿 `_index.md` 生成完成
- ~~那么~~ 必须再调用独立 formatter 组件才能渲染超链接和最终样式
