# 小红书报告生成规范

## 目的

定义小红书数据分析报告生成组件的执行规范。此组件可独立调用，处理已有的 raw.json 数据。

---

## 新增需求

### 需求: 可独立调用

summarize 组件必须可由用户单独调用，用于分析已有数据。

#### 场景: 独立调用入口

- **当** 用户已有 raw.json 数据
- **那么** 用户必须能够通过 `/xiaohongshu-summarize --dir <目录路径>` 调用
- **并且** 组件必须读取指定目录下的 raw.json
- **并且** 组件必须生成 _index.md 报告

### 需求: 读取 raw.json 生成报告

summarize 组件必须读取 raw.json 数据并生成结构化报告。

#### 场景: 数据读取

- **当** summarize 组件执行
- **那么** 必须读取 raw.json 中的以下字段：
  - `search_time`: 执行时间
  - `posts`: 所有帖子数据
  - `search_strategy`: 搜索策略（如存在）
  - `divergence_path`: 发散路径（如存在）
- **并且** 必须累加所有帖子的 `comments_count` 计算总评论数

#### 场景: 报告板块生成

- **当** summarize 组件生成报告
- **那么** 必须包含以下板块：
  - 搜索概览
  - 品牌/主题声量分析（如适用）
  - 评论区高频共识
  - 场景化决策建议（如适用）
  - 价格/规格参考（如适用）
  - 避雷指南
  - 关键洞察
  - 数据来源说明

#### 场景: 发散模式特殊板块

- **当** raw.json 包含 `divergence_path` 字段
- **那么** 报告必须在搜索概览后插入"搜索发散路径"板块
- **并且** 板块总数为 9 个（比固定模式多 1 个）

### 需求: 禁止使用语义分析脚本

summarize 组件必须使用 LLM 自然语言理解能力进行内容分析，禁止使用外部脚本。

#### 场景: 内容理解

- **当** summarize 组件分析帖子内容
- **那么** 必须使用 LLM 自然语言理解能力
- **并且** 禁止调用 Python `re` 正则匹配、`collections.Counter` 词频统计等脚本
- **并且** 允许基本数值统计（如帖子数量、评论总数）

#### 场景: 品牌识别

- **当** summarize 组件识别品牌或主题
- **那么** 必须基于 LLM 对帖子标题和内容的理解
- **并且** 禁止使用实体识别脚本

### 需求: 报告必须可追溯

summarize 组件生成的报告必须所有结论可追溯到具体帖子或评论。

#### 场景: 引用格式

- **当** summarize 组件引用帖子标题
- **那么** 必须使用 `*《标题》*` 格式（斜体）
- **当** summarize 组件引用用户评论
- **那么** 必须使用 `_"评论内容"_` 格式（斜体引号）
- **当** summarize 组件标注来源
- **那么** 必须使用 `> 来源：*《帖子标题》*` 格式

#### 场景: 样式区分

- **当** summarize 组件区分 AI 总结与原始引用
- **那么** AI 总结内容使用正常字体（粗体强调重点）
- **并且** 原始引用内容使用斜体样式

### 需求: Emoji 增强

summarize 组件生成的报告必须使用 emoji 增强可读性。

#### 场景: 必须使用 Emoji 的位置

- **当** summarize 组件生成报告
- **那么** 报告标题必须使用 emoji
- **并且** 以下板块标题必须使用 emoji：
  - 搜索概览
  - 搜索发散路径（如适用）
  - 品牌/主题声量分析
  - 评论区高频共识
  - 场景化决策建议
  - 价格/规格参考
  - 避雷指南
  - 关键洞察
  - 数据来源说明

#### 场景: Emoji 自由选择

- **当** summarize 组件选择具体 emoji
- **那么** 必须根据内容上下文自行判断
- **并且** 禁止使用固定映射规则

### 需求: 数据来源说明模板

summarize 组件必须在报告末尾包含固定的数据来源说明模板。

#### 场景: 固定模板

- **当** summarize 组件生成报告
- **那么** 必须在报告末尾使用以下模板：

```markdown
---

## 📍 数据来源说明

> 本报告由 AI 自动生成，数据来源于小红书平台用户评论。报告中所有品牌/店铺推荐、避雷评价、价格信息均来自用户真实反馈，AI 仅负责归纳整理。内容仅供参考，不构成任何消费建议。对于因使用本报告内容造成的任何损失，AI 不承担责任。

- 报告生成时间：[YYYY-MM-DD HH:MM]
- 工具版本：xiaohongshu-scraper v[X.X]
```

- **并且** 禁止修改免责声明措辞或省略字段
- **并且** 报告生成时间必须使用 raw.json 中的 `search_time` 字段

### 需求: 输出文件

summarize 组件必须将报告输出到指定目录。

#### 场景: 输出位置

- **当** summarize 组件完成报告生成
- **那么** 必须输出到输入目录下的 `_index.md`
- **并且** 必须将报告原文直接发送到用户对话框
